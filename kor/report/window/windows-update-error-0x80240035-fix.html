Windows 0x80240035 WU_E_UPDATE_NOT_PROCESSED 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 업데이트가 처리되지 않았을 때 발생하는 문제입니다. 업데이트 큐 문제, 처리 프로세스 중단, 또는 시스템 리소스 부족이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240035 (WU_E_UPDATE_NOT_PROCESSED)는 Windows Update 시스템에서 업데이트가 처리되지 않았을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 업데이트 처리 큐에서 업데이트가 건너뛰어짐
  • Windows Update 서비스 처리 프로세스 중단
  • 업데이트 메타데이터 처리 실패
  • 시스템 리소스 부족으로 인한 처리 지연
  • 업데이트 우선순위 설정 문제
  • Windows Update 큐 관리 오류
  • 동시 업데이트 처리 충돌
  • 업데이트 종속성 해결 실패
  • 시스템 정책에 의한 업데이트 차단
  • 손상된 업데이트 처리 엔진

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 처리 상태 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: Windows Update 큐 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 큐 및 처리 데이터 삭제:
  • 다음 명령어들을 실행하여 큐 데이터 삭제:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Windows\WER\*
del /f /s /q %windir%\Prefetch\*
3 서비스 재시작 및 큐 재초기화:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow
wuauclt /updatenow

방법 3: 업데이트 처리 우선순위 재설정

1 업데이트 처리 순서 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 대기 중인 업데이트 확인:
Get-WindowsUpdate -Verbose
dism /online /get-packages | findstr "Install Pending"
Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 10
2 업데이트 우선순위 재설정:
  • 다음 명령어로 업데이트 처리 순서 재설정:
usoclient ScanInstallWait
usoclient StartScan
usoclient StartDownload
usoclient StartInstall
usoclient RefreshSettings

방법 4: 시스템 리소스 최적화

1 메모리 및 CPU 사용량 최적화:
  • 작업 관리자에서 리소스 사용량 확인
  • 불필요한 프로그램 및 서비스 종료
  • 다음 명령어로 시스템 리소스 확인:
wmic process get ProcessId,PageFileUsage,WorkingSetSize
wmic OS get TotalVisibleMemorySize,FreePhysicalMemory
tasklist /svc | findstr svchost
powercfg /requests
2 디스크 공간 및 성능 최적화:
  • 다음 명령어로 디스크 정리 및 최적화:
cleanmgr /sagerun:1
dism /online /cleanup-image /startcomponentcleanup /resetbase
vssadmin delete shadows /all /quiet
defrag C: /O
chkdsk C: /f /r

방법 6: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
DISM /Online /Cleanup-Image /AnalyzeComponentStore
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
sfc /scanfile=C:\Windows\System32\wucltux.dll
sfc /scanfile=C:\Windows\System32\wuwebv.dll
  • 스캔 완료 후 시스템 재부팅

방법 7: 업데이트 정책 및 설정 재구성

1 Windows Update 정책 확인:
  • 그룹 정책 편집기 실행 (gpedit.msc)
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들을 "구성되지 않음"으로 설정:
  • - "자동 업데이트 구성"
  • - "업데이트 설치 일정 지정"
  • - "자동 다시 시작 지연"
2 레지스트리를 통한 정책 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallDay /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallTime /t REG_DWORD /d 3 /f
gpupdate /force

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
Set-WUSettings -AcceptTrustedPublisherCerts
2 업데이트 처리 강제 실행:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -IgnoreReboot -Verbose
Get-WindowsUpdateLog -LogPath C:\WindowsUpdate.log
Get-WUHistory | Where-Object {$_.Result -eq "Failed"}
Reset-WUComponents -Verbose

⚠️ 주의사항

업데이트 처리되지 않음 오류 해결 시 주의점

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 업데이트 처리 중 시스템을 강제 종료하지 마세요
  • 충분한 시스템 리소스를 확보하세요
  • 네트워크 연결이 안정적인 상태에서 작업하세요
  • 여러 업데이트를 동시에 처리하지 마세요
  • 정책 변경 시 기업 환경에서는 IT 관리자와 상의하세요
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 업데이트 처리 엔진이 지속적으로 실패하는 경우
  • 기업 환경에서 복잡한 정책이 적용된 경우
  • 하드웨어 리소스 부족으로 인한 지속적인 문제
  • 악성 소프트웨어에 의한 시스템 손상이 의심되는 경우
  • 시스템 전체 재설치가 필요한 경우

✅ 예방 방법

  • 정기적인 시스템 리소스 모니터링
  • 충분한 메모리 및 디스크 공간 확보
  • Windows Update 자동 설치 설정
  • 정기적인 시스템 재부팅
  • 불필요한 백그라운드 프로그램 정리
  • 시스템 복구 지점 정기 생성
  • 안정적인 인터넷 연결 환경 유지
  • 정기적인 시스템 파일 검사 (월 1회)
  • 업데이트 처리 상태 정기 확인

💡 추가 팁

  • 업데이트 처리되지 않음 오류는 주로 큐 관리 문제로 발생합니다
  • 시스템 리소스 부족 시 업데이트 처리가 지연될 수 있습니다
  • 업데이트 우선순위를 올바르게 설정하는 것이 중요합니다
  • 정기적인 시스템 재부팅으로 처리 큐를 초기화하세요
  • 한 번에 하나의 업데이트만 처리하는 것이 안전합니다
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 디스크 정리 및 최적화를 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

🔍 관련 오류 코드

유사한 Windows Update 처리 관련 오류들

  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x8024001D: 잘못된 업데이트 (WU_E_INVALID_UPDATE)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x8024001B: 자체 업데이트 진행 중 (WU_E_SELFUPDATE_IN_PROGRESS)
  • 0x80240036: 업데이트 처리 시간 초과 (WU_E_UPDATE_TIMEOUT)
  • 0x80240037: 업데이트 큐 오버플로우 (WU_E_QUEUE_OVERFLOW)
  • 0x80070005: 액세스 거부됨

🛠️ 고급 해결 방법

업데이트 처리 상태 모니터링 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 업데이트 처리 상태 실시간 모니터링:
while ($true) {
    Clear-Host
    Write-Host "Windows Update 처리 상태 모니터링" -ForegroundColor Green
    Write-Host "시간: $(Get-Date)" -ForegroundColor Yellow
    Write-Host "---" -ForegroundColor White
    $Updates = Get-WindowsUpdate -Verbose
    if ($Updates) {
        Write-Host "대기 중인 업데이트: $($Updates.Count)개" -ForegroundColor Cyan
        $Updates | Select-Object Title, Size | Format-Table
    } else {
        Write-Host "처리할 업데이트가 없습니다." -ForegroundColor Green
    }
    Start-Sleep -Seconds 30
}

업데이트 큐 분석 스크립트

  • 업데이트 큐 상태 분석:
$QueuePath = "C:\Windows\SoftwareDistribution\Download"
if (Test-Path $QueuePath) {
    $QueueFiles = Get-ChildItem $QueuePath -Recurse
    Write-Host "큐에 있는 파일 수: $($QueueFiles.Count)" -ForegroundColor Yellow
    $TotalSize = ($QueueFiles | Measure-Object -Property Length -Sum).Sum
    Write-Host "총 큐 크기: $([math]::Round($TotalSize/1MB, 2)) MB" -ForegroundColor Cyan
    $EmptyFiles = $QueueFiles | Where-Object {$_.Length -eq 0}
    if ($EmptyFiles) {
        Write-Host "손상된 파일: $($EmptyFiles.Count)개" -ForegroundColor Red
    }
} else {
    Write-Host "업데이트 큐 폴더가 존재하지 않습니다." -ForegroundColor Red
}

🔄 업데이트 처리 프로세스 이해

Windows Update 처리 단계

  1. 스캔: 사용 가능한 업데이트 검색
  2. 큐 생성: 업데이트를 처리 큐에 추가
  3. 우선순위 설정: 업데이트 처리 순서 결정
  4. 다운로드: 업데이트 파일 다운로드
  5. 검증: 다운로드된 파일 무결성 확인
  6. 설치 준비: 설치 전 시스템 준비
  7. 설치: 업데이트 실제 설치
  8. 완료: 설치 완료 및 큐에서 제거

처리되지 않음 오류가 발생하는 단계

  • 큐 생성 단계: 업데이트가 큐에 추가되지 않음
  • 우선순위 설정: 처리 순서에서 제외됨
  • 리소스 할당: 시스템 리소스 부족으로 건너뛰어짐
  • 종속성 해결: 필수 구성 요소 누락으로 처리 중단

문제 유형 증상 원인 해결 방법

🔧 업데이트 처리 최적화 도구

Windows 내장 도구

  • usoclient.exe: 업데이트 세션 오케스트레이터 클라이언트
  • wuauclt.exe: Windows Update 클라이언트 도구
  • UsoSvc: 업데이트 오케스트레이터 서비스
  • WaaSMedicSvc: Windows Update Medic 서비스

PowerShell 모듈

  • PSWindowsUpdate: 고급 Windows Update 관리
  • Get-WindowsUpdate: 업데이트 상태 확인
  • Install-WindowsUpdate: 업데이트 강제 설치
  • Reset-WUComponents: Windows Update 구성 요소 재설정

처리 최적화 방법

  • 충분한 시스템 리소스 확보

  • 백그라운드 프로세스 최소화
  • 정기적인 큐 정리 및 재설정
