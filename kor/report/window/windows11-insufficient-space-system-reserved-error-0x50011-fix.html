Windows 11 0x50011 INSUFFICIENT_SPACE_SYSTEM_RESERVED 해결 가이드

⚠️ 주의사항: 이 오류는 시스템 예약 파티션(System Reserved Partition) 또는 EFI 시스템 파티션의 공간이 부족할 때 발생하는 오류입니다. 부팅 관련 중요한 파티션이므로 신중하게 작업해야 합니다.

🔍 오류 개요

0x50011 INSUFFICIENT_SPACE_SYSTEM_RESERVED는 Windows 11에서 시스템 예약 파티션이나 EFI 시스템 파티션의 여유 공간이 부족할 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 시스템 예약 파티션(100MB~500MB) 공간 부족
  • EFI 시스템 파티션(ESP) 용량 부족
  • 부팅 파일 및 복구 환경 파일 누적
  • Windows Update 파일이 시스템 파티션에 저장됨
  • 언어 팩 설치로 인한 공간 부족
  • 복구 이미지 파일 크기 증가
  • 부팅 로그 파일 누적
  • 타사 부팅 관리자 파일 축적

🛠️ 해결 방법

방법 1: 시스템 파티션 상태 확인

1 디스크 관리에서 파티션 확인:
  • Windows + X 키를 누르고 "디스크 관리" 선택
  • 시스템 예약 파티션 또는 EFI 시스템 파티션 확인
  • 파티션 크기 및 여유 공간 확인
  • 일반적으로 다음과 같은 크기:
    • 시스템 예약 파티션: 100MB~500MB
    • EFI 시스템 파티션: 100MB~260MB
2 명령 프롬프트로 파티션 정보 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 파티션 상태 확인:
diskpart
list disk
select disk 0
list partition
list volume

방법 2: 시스템 파티션 정리

1 EFI 시스템 파티션 마운트:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 EFI 파티션 마운트:
diskpart
list volume
select volume [EFI 볼륨 번호]
assign letter=Z
exit
2 불필요한 파일 삭제:
  • 마운트된 EFI 파티션에서 다음 파일들 확인 및 정리:
dir Z:\ /s
cd Z:\EFI\Microsoft\Boot\Fonts
dir *.ttf
del /f /q *.ttf (불필요한 폰트 파일 삭제)
cd Z:\EFI\Microsoft\Boot\Resources
dir
(사용하지 않는 언어 폴더 삭제)
3 부팅 로그 파일 정리:
  • 다음 명령어로 부팅 관련 로그 파일 정리:
del /f /q Z:\bootmgr.efi.log
del /f /q Z:\EFI\Microsoft\Boot\*.log
del /f /q Z:\EFI\Boot\*.log

방법 3: Windows RE(복구 환경) 관리

1 Windows RE 상태 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 복구 환경 상태 확인:
reagentc /info
2 Windows RE 임시 비활성화:
  • 공간 확보를 위해 임시로 복구 환경 비활성화:
reagentc /disable
  • 복구 이미지 파일(winre.wim) 삭제 또는 이동
  • 작업 완료 후 다시 활성화:
reagentc /enable

방법 4: 시스템 파티션 크기 확장

1 인접한 파티션에서 공간 확보:
  • 디스크 관리에서 시스템 파티션 옆의 파티션 확인
  • C: 드라이브에서 공간을 축소하여 여유 공간 생성
  • 다음 명령어로 파티션 축소:
diskpart
list volume
select volume C
shrink desired=1024 (1GB 축소)
exit
2 시스템 파티션 확장:
  • 타사 파티션 관리 도구 사용 권장 (예: GParted, AOMEI Partition Assistant)
  • 또는 다음 명령어로 확장 시도:
diskpart
list volume
select volume [시스템 예약 볼륨 번호]
extend size=1024
exit

방법 5: 부팅 파일 정리 및 최적화

1 BCD 저장소 정리:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 부팅 구성 데이터 정리:
bcdedit /enum all
bcdedit /cleanup
bcdedit /deletevalue {bootmgr} displayorder
bcdedit /set {bootmgr} displayorder {current}
2 불필요한 부팅 항목 제거:
  • 시스템 구성 유틸리티 실행:
msconfig
  • "부팅" 탭에서 불필요한 부팅 항목 삭제
  • 기본 부팅 항목만 유지

방법 6: 언어 팩 및 기능 정리

1 불필요한 언어 팩 제거:
  • 설정 → 시간 및 언어 → 언어
  • 사용하지 않는 언어 팩 제거
  • 또는 PowerShell로 언어 팩 확인 및 제거:
Get-WindowsPackage -Online | Where-Object {$_.PackageName -like "*Language*"}
Remove-WindowsPackage -Online -PackageName [패키지명]
2 선택적 기능 제거:
  • 제어판 → 프로그램 → Windows 기능 켜기/끄기
  • 사용하지 않는 기능 비활성화
  • 특히 다음 기능들 확인:
    • Internet Explorer 11
    • Windows Media Player
    • XPS 뷰어
    • Fax 및 스캔

방법 7: 시스템 파일 압축

1 시스템 파티션 파일 압축:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 시스템 파일 압축:
compact /c /s:Z:\ /i /a
compact /c /s:Z:\EFI /i /a
2 압축 효과 확인:
  • 다음 명령어로 압축 결과 확인:
compact /q /s:Z:\
dir Z:\ /s

방법 8: 새로운 시스템 파티션 생성 (고급)

1 백업 및 준비:
  • 중요한 데이터 전체 백업
  • Windows 설치 미디어 준비
  • 시스템 복구 디스크 생성
2 파티션 재구성:
  • 타사 파티션 관리 도구 사용
  • 더 큰 크기의 EFI 시스템 파티션 생성 (512MB 권장)
  • 부팅 파일 복사 및 BCD 재구성
  • 부팅 테스트 및 검증

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 공간 확보량 위험도
시스템 파티션 정리 보통 30-45분 10-50MB 낮음
Windows RE 관리 보통 20-30분 50-200MB 보통
파티션 크기 확장 어려움 60-120분 500MB-2GB 높음
부팅 파일 정리 보통 20-40분 5-20MB 보통
언어 팩 제거 쉬움 15-30분 20-100MB 낮음
시스템 파일 압축 보통 30-60분 10-30MB 낮음
새로운 파티션 생성 매우 어려움 180-300분 대량 매우 높음

🔧 주요 명령어 참조

파티션 관리 명령어

  • diskpart: 디스크 파티션 관리 도구
  • list volume: 볼륨 목록 확인
  • assign letter=Z: 드라이브 문자 할당
  • shrink desired=1024: 파티션 크기 축소

부팅 관리 명령어

  • reagentc /info: Windows RE 상태 확인
  • bcdedit /enum all: 부팅 구성 데이터 확인
  • compact /c /s:Z:\ /i: 파일 압축
  • bootrec /fixboot: 부팅 섹터 복구

⚠️ 주의사항

시스템 파티션 작업 시 주의점

  • 시스템 파티션 작업 전 반드시 전체 시스템 백업을 생성하세요
  • EFI 시스템 파티션의 중요한 파일을 삭제하지 마세요
  • 파티션 크기 변경 시 데이터 손실 위험이 있습니다
  • 부팅 파일 수정 시 시스템이 부팅되지 않을 수 있습니다
  • Windows 복구 디스크를 미리 준비해두세요
  • 작업 중 전원이 차단되지 않도록 주의하세요

언제 전문가의 도움이 필요한가?

  • 파티션 테이블이 손상된 경우
  • 부팅이 완전히 불가능한 경우
  • 중요한 업무용 시스템인 경우
  • RAID 구성이나 복잡한 디스크 구조인 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 시스템 파티션 공간 모니터링
  • 불필요한 언어 팩 설치 자제
  • Windows RE 크기 정기적 확인
  • 부팅 로그 파일 정기적 정리
  • 시스템 설치 시 충분한 크기의 EFI 파티션 생성
  • 정기적인 부팅 구성 데이터 정리
  • 타사 부팅 관리자 사용 시 주의

💡 추가 팁

  • EFI 시스템 파티션은 최소 260MB 이상 권장
  • 시스템 예약 파티션은 500MB 이상 권장
  • 언어 팩 설치 전 시스템 파티션 공간 확인
  • Windows 업데이트 후 시스템 파티션 공간 확인
  • 파티션 관리 도구 사용 시 신뢰할 수 있는 도구 선택
  • 정기적인 시스템 백업으로 데이터 보호

🔍 관련 오류 코드

유사한 시스템 파티션 관련 오류들

  • 0x80070070: 디스크 공간이 부족합니다
  • 0x80070002: 지정된 파일을 찾을 수 없습니다
  • 0x80070003: 지정된 경로를 찾을 수 없습니다
  • 0xC0000225: 부팅 구성 데이터 누락
  • 0xC000000F: 부팅 선택 실패
  • 0x80300024: 파티션 생성 실패

🛠️ 고급 진단 도구

파티션 분석 도구

  • 디스크 관리: diskmgmt.msc - 파티션 상태 확인
  • DiskPart: diskpart - 고급 파티션 관리
  • 시스템 정보: msinfo32 - 시스템 구성 확인
  • 이벤트 뷰어: eventvwr.msc - 시스템 오류 로그

타사 파티션 도구

  • GParted: 무료 파티션 관리 도구
  • AOMEI Partition Assistant: 파티션 크기 조정
  • MiniTool Partition Wizard: 종합 파티션 관리
  • EaseUS Partition Master: 사용자 친화적 인터페이스

파티션 유형 최소 크기 권장 크기 용도
