🚀 단계별 듀얼부팅 설정 가이드

1단계

사전 준비 및 백업

🔒 필수 백업 작업: 듀얼부팅 설정 전 시스템 전체 백업은 필수입니다. 파티션 작업 중 데이터 손실이 발생할 수 있습니다.

백업해야 할 항목들:

  • 시스템 이미지: Windows 백업 및 복원 도구 사용
  • 중요 파일: 개인 문서, 사진, 동영상 등
  • 프로그램 설정: 브라우저 북마크, 게임 세이브 파일
  • 드라이버: 하드웨어 제조사 드라이버 파일
  • 라이선스 키: Windows 및 소프트웨어 제품키

필요한 도구 준비:

  1. Windows 11 설치 미디어: 공식 ISO 파일 및 USB
  2. 파티션 관리 도구: GParted, EaseUS Partition Master
  3. 부트 복구 도구: Boot Repair Disk, Windows PE
  4. 외장 저장소: 백업용 외장 하드 또는 클라우드

✅ 사전 준비 체크리스트





# PowerShell로 시스템 정보 백업
Get-ComputerInfo > system_info.txt
Get-WmiObject -Class Win32_Product | Export-Csv installed_programs.csv
wmic product get name,version /format:csv > software_list.csv
2단계

디스크 파티션 계획 및 설정

💡 파티션 계획: 듀얼부팅을 위해서는 각 운영체제가 독립적인 파티션에 설치되어야 합니다. 적절한 크기 배분이 중요합니다.

권장 파티션 구성:

1TB SSD 기준 파티션 예시
┌─────────────────────────────────────────────────────┐
│ EFI System (100MB) │ MSR (16MB) │ Windows 10 (400GB) │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│ Windows 11 (400GB) │ Shared Data (150GB) │ Linux (50GB) │
└─────────────────────────────────────────────────────┘

파티션 크기 가이드라인:

운영체제 최소 크기 권장 크기 용도
Windows 11 64GB 250GB 이상 시스템 + 프로그램
Windows 10 32GB 200GB 이상 시스템 + 프로그램
Ubuntu Linux 25GB 50GB 이상 시스템 + 개발 도구
공유 데이터 - 100GB 이상 문서, 미디어 파일
Swap (Linux) 2GB RAM 크기와 동일 가상 메모리

디스크 관리 도구로 파티션 생성:

  1. Windows + X → "디스크 관리" 선택
  2. 기존 파티션 축소 (C: 드라이브 우클릭 → "볼륨 축소")
  3. 할당되지 않은 공간에서 새 파티션 생성
  4. 각 OS용 파티션을 NTFS로 포맷
# DiskPart로 파티션 생성
diskpart
list disk
select disk 0
list partition
create partition primary size=250000
format fs=ntfs quick label="Windows11"
assign letter=D
exit

✅ 파티션 설정 체크리스트





3단계

Windows 11 설치 (듀얼부팅 모드)

⚠️ 설치 시 주의사항: Windows 11 설치 시 기존 부트로더를 덮어쓸 수 있습니다. 설치 후 부트 메뉴 복구가 필요할 수 있습니다.

Windows 11 설치 과정:

  1. 부팅 순서 변경: BIOS에서 USB 우선 부팅 설정
  2. 설치 시작: Windows 11 설치 미디어로 부팅
  3. 파티션 선택: 미리 준비한 파티션 선택 (포맷하지 않음)
  4. 사용자 지정 설치: "사용자 지정" 옵션 선택
  5. 설치 진행: 자동 설치 과정 대기

설치 중 중요한 선택사항:

파티션 선택 화면에서:
  • 기존 Windows가 설치된 파티션은 건드리지 않기
  • 새로 만든 파티션만 선택하여 설치
  • "드라이버 로드" 필요시에만 사용
  • 시스템 파티션(EFI) 건드리지 않기

설치 후 확인사항:

  • 부팅 메뉴: 재시작 시 OS 선택 메뉴 확인
  • 드라이버: 하드웨어 정상 작동 확인
  • 네트워크: 인터넷 연결 상태 확인
  • 활성화: Windows 정품 인증 상태

✅ Windows 11 설치 체크리스트





4단계

부트로더 설정 및 관리

💡 부트로더란: 여러 운영체제 중 어느 것을 부팅할지 선택할 수 있게 해주는 프로그램입니다. Windows는 BCDEdit, Linux는 GRUB을 주로 사용합니다.

Windows 부트 관리자 설정:

  1. 관리자 권한으로 명령 프롬프트 실행
  2. 현재 부트 옵션 확인
  3. 부트 메뉴 시간 조정
  4. 기본 OS 설정
# BCDEdit 명령어로 부트 관리
# 현재 부트 설정 확인
bcdedit

# 부트 메뉴 표시 시간 설정 (30초)
bcdedit /timeout 30

# 기본 부팅 OS 설정
bcdedit /default {current}

# 부트 메뉴 항목 이름 변경
bcdedit /set {current} description "Windows 11 Pro"

Linux와 듀얼부팅 시 GRUB 설정:

설정 항목 파일 위치 주요 옵션 설명
GRUB 설정 /etc/default/grub GRUB_TIMEOUT 부트 메뉴 대기 시간
메뉴 항목 /boot/grub/grub.cfg menuentry 부팅 옵션 목록
기본 OS /etc/default/grub GRUB_DEFAULT 기본 부팅 시스템
Windows 감지 os-prober 자동 감지 Windows 자동 인식
# Linux에서 GRUB 설정 업데이트
sudo nano /etc/default/grub
# GRUB_TIMEOUT=10 (10초 대기)
# GRUB_DEFAULT=0 (첫 번째 항목 기본)

# Windows 자동 감지 활성화
sudo os-prober

# GRUB 설정 업데이트
sudo update-grub

✅ 부트로더 설정 체크리스트





5단계

Linux 설치 (Ubuntu 기준)

⚠️ Linux 설치 주의사항: Linux 설치 시 파티션을 잘못 선택하면 Windows 데이터가 삭제될 수 있습니다. 수동 파티션 설정을 권장합니다.

Ubuntu 설치 과정:

  1. Live USB 생성: Rufus 또는 Etcher로 부팅 USB 생성
  2. Try Ubuntu: 먼저 Live 모드로 테스트
  3. 설치 시작: "Install Ubuntu" 선택
  4. 설치 유형: "기타" 또는 "Something else" 선택
  5. 파티션 설정: 수동으로 파티션 할당

Ubuntu 파티션 설정:

마운트 포인트 파일 시스템 크기 용도
/ ext4 30-50GB 루트 파티션 (시스템)
/home ext4 50GB+ 사용자 데이터
swap swap RAM 크기 가상 메모리
/boot/efi FAT32 기존 사용 EFI 시스템 파티션

GRUB 부트로더 설치:

부트로더 설치 위치:
  • UEFI 시스템: /boot/efi 파티션에 설치
  • Legacy BIOS: /dev/sda (전체 디스크)에 설치
  • 주의: Windows Boot Manager를 덮어쓰지 않도록 주의

듀얼부팅 최적화 및 관리

시간 동기화 문제 해결:

💡 시간 문제: Windows와 Linux는 시간을 다르게 처리하여 듀얼부팅 시 시간이 맞지 않는 문제가 발생할 수 있습니다.
# Windows에서 UTC 시간 사용하도록 설정
reg add "HKEY_LOCAL_MACHINE\System\CurrentControlSet\Control\TimeZoneInformation" /v RealTimeIsUniversal /d 1 /t REG_DWORD /f

# 또는 Linux에서 로컬 시간 사용
sudo timedatectl set-local-rtc 1 --adjust-system-clock

공유 파티션 설정:

  1. NTFS 파티션 생성: 양쪽 OS에서 읽기/쓰기 가능
  2. Linux에서 NTFS 지원: ntfs-3g 패키지 설치
  3. 자동 마운트: /etc/fstab에 설정 추가
  4. 권한 설정: 사용자 권한 적절히 설정

🔧 듀얼부팅 문제 해결

일반적인 문제와 해결책

문제 증상 원인 해결 방법
부팅 메뉴 없음 한 OS만 부팅됨 부트로더 설정 오류 BCDEdit 또는 GRUB 재설정
Windows 부팅 실패 부팅 오류 메시지 BCD 손상 Windows 복구 환경 사용
Linux 부팅 실패 GRUB 오류 GRUB 설정 문제 Live USB로 GRUB 복구
시간 불일치 OS 전환 시 시간 변경 시간 기준 차이 UTC 시간 통일

부트 복구 명령어

# Windows 부트 복구
bootrec /fixmbr
bootrec /fixboot
bootrec /scanos
bootrec /rebuildbcd

# Linux GRUB 복구
sudo mount /dev/sda1 /mnt
sudo grub-install --root-directory=/mnt /dev/sda
sudo update-grub
