Windows 0x80240012 WU_E_REG_VALUE_INVALID 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 레지스트리 값이 유효하지 않거나 손상되었을 때 발생하는 문제입니다. 레지스트리 손상이나 잘못된 설정 값이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240012 (WU_E_REG_VALUE_INVALID)는 Windows Update 시스템에서 레지스트리 값이 유효하지 않거나 손상되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 관련 레지스트리 값 손상
  • 잘못된 형식의 레지스트리 데이터
  • 누락된 필수 레지스트리 키 또는 값
  • 레지스트리 권한 설정 문제
  • 타사 소프트웨어에 의한 레지스트리 변경
  • 시스템 파일 손상으로 인한 레지스트리 오류
  • 악성 소프트웨어에 의한 레지스트리 손상
  • 부적절한 시스템 종료로 인한 레지스트리 손상
  • Windows Update 정책 설정 오류
  • 그룹 정책에 의한 레지스트리 값 충돌

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 레지스트리 검증:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: Windows Update 레지스트리 완전 재설정

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 관련 레지스트리 키 삭제:
  • 다음 경로들의 키를 삭제:
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Orchestrator
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Reporting
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Validation
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\wuauserv\Parameters
3 Windows Update 정책 레지스트리 재설정:
  • 다음 경로의 키들을 삭제:
HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
HKEY_CURRENT_USER\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\PolicyManager\current\device\Update
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\OSUpgrade
HKEY_LOCAL_MACHINE\SOFTWARE\WOW6432Node\Microsoft\Windows\CurrentVersion\WindowsUpdate
  • 레지스트리 편집기 종료 후 재부팅

방법 3: Windows Update 서비스 재구축

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 캐시 및 데이터 완전 삭제:
  • 다음 명령어들을 실행하여 모든 관련 데이터 삭제:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
rd /s /q %windir%\WindowsUpdate.log
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Windows\WER\*
3 Windows Update 서비스 재시작 및 레지스트리 재생성:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc

wuauclt /updatenow

방법 4: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
DISM /Online /Cleanup-Image /AnalyzeComponentStore
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:

sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
sfc /scanfile=C:\Windows\System32\wucltux.dll
  • 스캔 완료 후 시스템 재부팅

방법 6: 레지스트리 권한 및 소유권 복구

1 레지스트리 권한 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행하여 레지스트리 권한 복구:

gpupdate /force
sfc /purgecache
2 Windows Update 레지스트리 소유권 복구:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
$acl = Get-Acl "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate"
$acl.SetOwner([System.Security.Principal.NTAccount]"NT AUTHORITY\SYSTEM")
Set-Acl "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" $acl
$rule = New-Object System.Security.AccessControl.RegistryAccessRule("Everyone","FullControl","Allow")
$acl.SetAccessRule($rule)
Set-Acl "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" $acl

방법 7: 그룹 정책 및 Windows Update 정책 재설정

1 로컬 그룹 정책 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:

gpupdate /force
secedit /configure /cfg %windir%\inf\defltbase.inf /db defltbase.sdb /verbose
2 Windows Update 정책 재설정:
  • gpedit.msc 실행 (Windows Pro/Enterprise)
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 모든 정책을 "구성되지 않음"으로 설정
  • 또는 명령 프롬프트에서 다음 명령어 실행:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f

gpupdate /force

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 레지스트리 편집 시 정확한 경로와 값을 확인하세요
  • 잘못된 레지스트리 수정은 시스템 부팅 실패를 야기할 수 있습니다
  • 관리자 권한으로 모든 작업을 수행하세요
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다
  • 그룹 정책 변경 시 기업 환경에서는 IT 관리자와 상의하세요

🔍 관련 오류 코드

유사한 Windows Update 레지스트리 관련 오류들

  • 0x80240011: 잘못된 관계 (WU_E_INVALID_RELATIONSHIP)
  • 0x80240010: 관계 깊이 초과 (WU_E_TOO_DEEP_RELATION)
  • 0x8024000F: 순환 종속성 감지 (WU_E_CYCLE_DETECTED)
  • 0x80240013: 중복 업데이트 (WU_E_DUPLICATE_ITEM)
  • 0x80240014: 업데이트 ID 무효 (WU_E_INVALID_UPDATE_ID)
  • 0x80070005: 액세스 거부됨
  • 0x80070002: 시스템이 지정된 파일을 찾을 수 없음

🛠️ 고급 해결 방법

레지스트리 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 레지스트리 값 검증:
$WUPath = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate"
if (Test-Path $WUPath) {
    Get-ItemProperty $WUPath | Format-List
} else {
    Write-Host "Windows Update 레지스트리 키가 존재하지 않습니다."
}
Get-Service wuauserv | Format-List

레지스트리 복구 스크립트

  • 손상된 레지스트리 값 복구:
$RegPath = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update"
if (!(Test-Path $RegPath)) {
    New-Item -Path $RegPath -Force
}
Set-ItemProperty -Path $RegPath -Name "AUOptions" -Value 4 -Type DWord
Set-ItemProperty -Path $RegPath -Name "ScheduledInstallDay" -Value 0 -Type DWord
Set-ItemProperty -Path $RegPath -Name "ScheduledInstallTime" -Value 3 -Type DWord

🔄 레지스트리 구조 이해

Windows Update 주요 레지스트리 키

  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate: 기본 설정
  • HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update: 자동 업데이트 설정
  • HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate: 정책 설정
  • HKLM\SYSTEM\CurrentControlSet\Services\wuauserv: 서비스 설정

중요한 레지스트리 값

  • AUOptions: 자동 업데이트 옵션 (DWORD)
  • ScheduledInstallDay: 예약 설치 요일 (DWORD)
  • ScheduledInstallTime: 예약 설치 시간 (DWORD)
  • UseWUServer: WSUS 서버 사용 여부 (DWORD)

📊 레지스트리 값 유형 및 형식

레지스트리 값 이름 데이터 형식 유효한 값 설명
AUOptions REG_DWORD 1, 2, 3, 4, 5 자동 업데이트 동작 설정
ScheduledInstallDay REG_DWORD 0-7 예약 설치 요일 (0=매일)
ScheduledInstallTime REG_DWORD 0-23 예약 설치 시간 (24시간 형식)
UseWUServer REG_DWORD 0, 1 WSUS 서버 사용 여부
WUServer REG_SZ URL 형식 WSUS 서버 주소
