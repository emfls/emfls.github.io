Windows 0x80240032 WU_E_INVALID_CRITERIA 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 잘못된 검색 기준이나 필터 조건이 사용되었을 때 발생하는 문제입니다. 업데이트 검색 매개변수 오류, API 호출 문제, 또는 시스템 호환성 기준 불일치가 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240032 (WU_E_INVALID_CRITERIA)는 Windows Update 시스템에서 잘못된 검색 기준이나 필터 조건이 사용되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 잘못된 업데이트 검색 기준 설정
  • Windows Update API 호출 매개변수 오류
  • 손상된 업데이트 필터 설정
  • 시스템 호환성 기준 불일치
  • 업데이트 카테고리 필터 오류
  • 날짜 범위 검색 조건 문제
  • 업데이트 우선순위 기준 오류
  • 타사 소프트웨어의 잘못된 API 사용
  • 손상된 Windows Update 구성 요소
  • 레지스트리의 검색 기준 설정 손상

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 기준 검증:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

1 모든 Windows Update 관련 서비스 중지:

방법 4: 레지스트리 검색 기준 설정 복구

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 검색 기준 관련 레지스트리 키 재설정:
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
3 검색 기준 필터 설정 복구:
  • 다음 명령어들을 실행:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallDay /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallTime /t REG_DWORD /d 3 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /v AcceptTrustedPublisherCerts /t REG_DWORD /d 1 /f
gpupdate /force

방법 5: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth

2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:

방법 6: 업데이트 검색 필터 수동 설정

1 PowerShell을 이용한 검색 기준 확인:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어로 현재 검색 기준 확인:
$UpdateSession = New-Object -ComObject Microsoft.Update.Session
$UpdateSearcher = $UpdateSession.CreateUpdateSearcher()
$UpdateSearcher.Online = $true
$UpdateSearcher.IncludePotentiallySupersededUpdates = $false
Write-Host "검색 기준 설정:"
Write-Host "온라인 검색: $($UpdateSearcher.Online)"
Write-Host "대체된 업데이트 포함: $($UpdateSearcher.IncludePotentiallySupersededUpdates)"
2 기본 검색 기준으로 업데이트 검색:
  • 다음 명령어로 기본 기준으로 검색:
try {
    $SearchResult = $UpdateSearcher.Search("IsInstalled=0")
    Write-Host "검색 성공: $($SearchResult.Updates.Count)개 업데이트 발견"
    foreach ($Update in $SearchResult.Updates) {
        Write-Host "- $($Update.Title)"
    }
} catch {
    Write-Host "검색 실패: $($_.Exception.Message)" -ForegroundColor Red
}

  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x8024001D: 잘못된 업데이트 (WU_E_INVALID_UPDATE)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x80240033: 잘못된 인덱스 (WU_E_INVALID_INDEX)
  • 0x80240034: 잘못된 작업 (WU_E_INVALID_OPERATION)
  • 0x80240035: 잘못된 매개변수 (WU_E_INVALID_PARAMETER)
  • 0x80070057: 매개변수가 올바르지 않음

🛠️ 고급 해결 방법

Windows Update API 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • API 기능 검증:
try {
    $UpdateSession = New-Object -ComObject Microsoft.Update.Session
    Write-Host "Windows Update 세션 생성 성공" -ForegroundColor Green
    $UpdateSearcher = $UpdateSession.CreateUpdateSearcher()
    Write-Host "업데이트 검색기 생성 성공" -ForegroundColor Green
    $SearchCriteria = @(
        "IsInstalled=0",
        "IsHidden=0",
        "Type='Software'"
    )
    foreach ($Criteria in $SearchCriteria) {
        try {
            $Result = $UpdateSearcher.Search($Criteria)
            Write-Host "검색 기준 '$Criteria': 성공 ($($Result.Updates.Count)개)" -ForegroundColor Green
        } catch {
            Write-Host "검색 기준 '$Criteria': 실패 - $($_.Exception.Message)" -ForegroundColor Red
        }
    }
} catch {
    Write-Host "Windows Update API 초기화 실패: $($_.Exception.Message)" -ForegroundColor Red
}

검색 기준 복구 스크립트

  • 손상된 검색 기준 자동 복구:
$DefaultCriteria = @{
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" = @{
        "AUOptions" = 4
        "ScheduledInstallDay" = 0
        "ScheduledInstallTime" = 3
    }
}
foreach ($Path in $DefaultCriteria.Keys) {
    if (!(Test-Path $Path)) {
        New-Item -Path $Path -Force | Out-Null
    }
    foreach ($Name in $DefaultCriteria[$Path].Keys) {
        Set-ItemProperty -Path $Path -Name $Name -Value $DefaultCriteria[$Path][$Name] -Type DWord
        Write-Host "설정됨: $Path\$Name = $($DefaultCriteria[$Path][$Name])" -ForegroundColor Green
    }
}

🔄 검색 기준 이해 가이드

Windows Update 검색 기준 문법

기준 설명 예시 용도
IsInstalled 설치 상태 IsInstalled=0 미설치 업데이트 검색
IsHidden 숨김 상태 IsHidden=0 숨겨지지 않은 업데이트
Type 업데이트 유형 Type='Software' 소프트웨어 업데이트만
CategoryIDs 카테고리 ID CategoryIDs contains '{GUID}' 특정 카테고리 업데이트
UpdateID 업데이트 ID UpdateID='{GUID}' 특정 업데이트 검색

검색 기준 조합 예시

  • 기본 검색: IsInstalled=0
