Windows 0x80240042 WU_E_UNKNOWN_SERVICE 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 알 수 없는 서비스가 감지되었을 때 발생하는 문제입니다. 서비스 등록 오류, 손상된 서비스 구성, 또는 타사 소프트웨어 간섭이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240042 (WU_E_UNKNOWN_SERVICE)는 Windows Update 시스템에서 알 수 없거나 등록되지 않은 서비스를 감지했을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 서비스 등록 정보 손상
  • 알 수 없는 업데이트 서비스 제공자
  • 손상된 서비스 구성 파일
  • 타사 업데이트 관리 소프트웨어 간섭
  • 잘못된 WSUS 서버 설정
  • 레지스트리의 서비스 정보 손상
  • Windows Update Agent 구성 오류
  • 악성 소프트웨어에 의한 서비스 변조
  • 시스템 파일 손상으로 인한 서비스 인식 실패
  • 업데이트 서비스 목록 불일치

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 서비스 검증:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: Windows Update 서비스 등록 상태 확인

1 등록된 Windows Update 서비스 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 서비스 목록 확인:
Get-Service | Where-Object {$_.Name -like "*update*" -or $_.Name -like "*wu*"}
sc query type= service state= all | findstr /i "update"
wmic service where "name like '%update%'" get name,state,startmode
2 Windows Update 서비스 제공자 확인:
  • 다음 명령어로 서비스 제공자 목록 확인:
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services"
reg query "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update\Results"
powershell "Get-WmiObject -Class Win32_Service | Where-Object {$_.Name -like '*update*'}"

방법 3: Windows Update 서비스 재등록

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 구성 요소 재등록:
  • 다음 명령어들을 순서대로 실행:
regsvr32 /s wuapi.dll
regsvr32 /s wuaueng.dll
regsvr32 /s wuaueng1.dll
regsvr32 /s wucltui.dll
regsvr32 /s wups.dll
regsvr32 /s wups2.dll
regsvr32 /s wuweb.dll
regsvr32 /s qmgr.dll
regsvr32 /s qmgrprxy.dll
regsvr32 /s wucltux.dll
regsvr32 /s muweb.dll
regsvr32 /s wuwebv.dll
regsvr32 /s wudriver.dll
regsvr32 /s wuapp.exe
3 서비스 재시작 및 서비스 목록 재구축:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 4: 레지스트리 서비스 정보 복구

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 서비스 레지스트리 키 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services" /f
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
3 기본 Windows Update 서비스 등록:
  • 다음 명령어로 기본 서비스 등록:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services\7971F918-A847-4430-9279-4A52D1EFE18D" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services\7971F918-A847-4430-9279-4A52D1EFE18D" /v RegisteredWithAU /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services\7971F918-A847-4430-9279-4A52D1EFE18D" /v IsDefaultAUService /t REG_DWORD /d 1 /f

방법 5: 타사 업데이트 소프트웨어 확인 및 제거

1 설치된 업데이트 관리 소프트웨어 확인:
  • 제어판 → 프로그램 및 기능에서 다음 프로그램 확인:
  • - Windows Update 관리 도구
  • - 타사 업데이트 매니저
  • - 시스템 최적화 도구
  • - 보안 소프트웨어의 업데이트 모듈
2 간섭하는 소프트웨어 임시 비활성화:
  • 다음 프로그램들을 임시 비활성화:
  • - 타사 보안 소프트웨어
  • - 시스템 최적화 도구
  • - 업데이트 관리 프로그램
  • - 방화벽 소프트웨어
3 시작 프로그램 정리:
  • 작업 관리자 → 시작 프로그램 탭
  • 업데이트 관련 불필요한 프로그램 비활성화
  • 또는 명령어로 확인:
wmic startup get caption,command
msconfig

방법 6: WSUS 서버 설정 확인 및 재설정

1 WSUS 서버 설정 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WSUS 설정 확인:
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUStatusServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer
2 WSUS 설정 재설정 (필요시):
  • WSUS 서버 사용 중지:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUServer /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUStatusServer /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer /t REG_DWORD /d 0 /f
gpupdate /force

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
sfc /scanfile=C:\Windows\System32\wucltux.dll
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 서비스 관리자 재설정 및 업데이트:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Remove-WUServiceManager -ServiceID * -Confirm:$false
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -IgnoreReboot -Verbose
Reset-WUComponents -Verbose

  • reg add "키경로" /v 값이름 /t 데이터형식 /d 값: 레지스트리 값 추가

  • 중요한 데이터는 백업 후 작업하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요

  • 서비스 재등록 시 순서를 정확히 따르세요
  • 관리자 권한으로 모든 작업을 수행하세요

💡 추가 팁

  • 알 수 없는 서비스 오류는 주로 서비스 등록 문제로 발생합니다
  • 타사 소프트웨어가 Windows Update 서비스를 간섭할 수 있습니다
  • WSUS 서버 설정이 잘못되면 서비스 인식 오류가 발생할 수 있습니다
  • 서비스 재등록은 순서가 중요합니다
  • 레지스트리 수정 시 정확한 경로와 값을 확인하세요
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 서비스 상태 모니터링을 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

🔍 관련 오류 코드

유사한 Windows Update 서비스 관련 오류들

  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x8024001D: 잘못된 업데이트 (WU_E_INVALID_UPDATE)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x8024001B: 자체 업데이트 진행 중 (WU_E_SELFUPDATE_IN_PROGRESS)
  • 0x80240043: 서비스 시작 실패 (WU_E_SERVICE_START_FAILED)
  • 0x80240044: 서비스 등록 실패 (WU_E_SERVICE_REGISTRATION_FAILED)
  • 0x80070005: 액세스 거부됨

🛠️ 고급 해결 방법

서비스 등록 상태 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • Windows Update 서비스 등록 상태 확인:
$WUServices = @("wuauserv", "cryptSvc", "bits", "msiserver", "UsoSvc")
Write-Host "Windows Update 서비스 등록 상태 확인" -ForegroundColor Green
foreach ($Service in $WUServices) {
    try {
        $ServiceObj = Get-Service -Name $Service -ErrorAction Stop
        Write-Host "$Service : 등록됨 ($($ServiceObj.Status))" -ForegroundColor Green
    } catch {
        Write-Host "$Service : 등록되지 않음" -ForegroundColor Red
    }
}
$WUProviders = Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services\*" -ErrorAction SilentlyContinue
Write-Host "등록된 업데이트 서비스 제공자: $($WUProviders.Count)개" -ForegroundColor Yellow

알 수 없는 서비스 자동 제거 스크립트

  • 알 수 없는 서비스 제거 및 기본 서비스 복원:
$DefaultServiceID = "7971F918-A847-4430-9279-4A52D1EFE18D"
$ServicesPath = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Services"
$RegisteredServices = Get-ChildItem $ServicesPath -ErrorAction SilentlyContinue
foreach ($Service in $RegisteredServices) {
    $ServiceID = $Service.PSChildName
    if ($ServiceID -ne $DefaultServiceID) {
        Write-Host "알 수 없는 서비스 제거: $ServiceID" -ForegroundColor Yellow
        Remove-Item $Service.PSPath -Recurse -Force
    }
}
Write-Host "기본 Windows Update 서비스만 유지되었습니다." -ForegroundColor Green

🔧 서비스 관리 도구

Windows 기본 도구

  • services.msc: 서비스 관리 콘솔
  • sc.exe: 서비스 제어 명령줄 도구
  • regedit.exe: 레지스트리 편집기
  • msconfig.exe: 시스템 구성 유틸리티

PowerShell 모듈

  • PSWindowsUpdate: Windows Update 관리 모듈
  • Get-Service: 서비스 상태 확인
  • Start-Service/Stop-Service: 서비스 제어
  • Get-WmiObject: WMI 객체 조회

진단 도구

  • 이벤트 뷰어: 시스템 로그 확인
  • 작업 관리자: 프로세스 및 서비스 모니터링
  • 리소스 모니터: 시스템 리소스 사용량
  • 성능 모니터: 성능 카운터 추적

🔒 보안 고려사항

서비스 보안 검증

  • 디지털 서명 확인: 서비스 파일의 Microsoft 서명 검증
  • 서비스 경로 확인: 시스템 폴더 외부 서비스 주의
  • 권한 설정 검토: 과도한 권한 부여 방지
  • 네트워크 연결 모니터링: 의심스러운 외부 연결 차단

악성 소프트웨어 방지

  • 신뢰할 수 있는 소프트웨어만 설치
  • 정기적인 전체 시스템 검사
  • 의심스러운 서비스 즉시 조사
  • 시스템 파일 무결성 정기 검사
