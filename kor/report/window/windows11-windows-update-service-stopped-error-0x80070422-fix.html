Windows 11 0x80070422 WINDOWS_UPDATE_SERVICE_STOPPED 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 서비스가 중지되어 있거나 시작할 수 없을 때 발생하는 오류입니다. 서비스 의존성 문제나 시스템 설정 오류가 주요 원인입니다.

🔍 오류 개요

0x80070422 WINDOWS_UPDATE_SERVICE_STOPPED는 Windows 11에서 Windows Update 서비스가 중지되어 있거나 시작되지 않을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 서비스가 수동으로 비활성화됨
  • 서비스 의존성 문제로 인한 시작 실패
  • 그룹 정책에 의한 Windows Update 서비스 제한
  • 레지스트리 손상으로 인한 서비스 구성 오류
  • 시스템 파일 손상으로 인한 서비스 실행 불가
  • 타사 소프트웨어에 의한 서비스 간섭
  • 바이러스나 악성 코드에 의한 서비스 비활성화
  • 부적절한 시스템 최적화 도구 사용

🛠️ 해결 방법

방법 1: Windows Update 서비스 수동 시작

1 서비스 관리자 실행:
  • Windows + R 키를 누르고 services.msc 입력
  • 서비스 목록에서 다음 서비스들 찾기:
    • Windows Update (wuauserv)
    • Background Intelligent Transfer Service (BITS)
    • Cryptographic Services (CryptSvc)
    • Windows Installer (msiserver)
2 서비스 시작 유형 변경:
  • 각 서비스를 더블클릭하여 속성 창 열기
  • "시작 유형"을 다음과 같이 설정:
    • Windows Update: "자동 (지연된 시작)"
    • BITS: "자동 (지연된 시작)"
    • Cryptographic Services: "자동"
    • Windows Installer: "수동"
  • "서비스 상태"에서 "시작" 버튼 클릭
  • "확인" 클릭하여 설정 저장

방법 2: 명령 프롬프트를 통한 서비스 시작

1 관리자 권한으로 명령 프롬프트 실행:
  • Windows + X 키를 누르고 "Windows Terminal(관리자)" 선택
  • 다음 명령어들을 순서대로 실행:
sc config wuauserv start= auto
sc config bits start= auto
sc config cryptsvc start= auto
sc config msiserver start= demand
2 서비스 시작:
  • 다음 명령어들을 순서대로 실행:
net start cryptsvc
net start bits
net start wuauserv
net start msiserver
  • 각 명령어 실행 후 "서비스가 시작되었습니다" 메시지 확인

방법 3: 서비스 의존성 문제 해결

1 서비스 의존성 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 서비스 의존성 확인:
sc qc wuauserv
sc query wuauserv
  • 의존 서비스들이 모두 실행 중인지 확인
2 의존 서비스 시작:
  • Windows Update 서비스가 의존하는 서비스들을 먼저 시작:
net start rpcss
net start eventlog
net start cryptsvc
net start bits
net start wuauserv

방법 4: 레지스트리 수정을 통한 서비스 복구

1 레지스트리 편집기 실행:
  • Windows + R 키를 누르고 regedit 입력
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\Services\wuauserv
2 서비스 시작 유형 수정:
  • "Start" 값을 더블클릭
  • 값 데이터를 다음과 같이 설정:
    • 2: 자동
    • 3: 수동
    • 4: 사용 안 함
  • Windows Update의 경우 "2" 또는 "3"으로 설정
  • 다른 관련 서비스들도 동일하게 확인 및 수정

방법 5: 그룹 정책 확인 및 수정

1 로컬 그룹 정책 편집기 실행:
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 다음 경로로 이동:
컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
2 Windows Update 정책 확인:
  • 다음 정책들을 확인하고 필요시 수정:
    • "자동 업데이트 구성": "구성되지 않음" 또는 "사용"
    • "자동 업데이트 사용 안 함": "구성되지 않음" 또는 "사용 안 함"
    • "Windows Update 서비스에 대한 액세스 제거": "구성되지 않음"
  • 제한적인 정책이 있다면 "구성되지 않음"으로 변경
  • 변경 후 gpupdate /force 명령어 실행

방법 6: Windows Update 문제 해결사 실행

1 자동 문제 해결사 실행:
  • 설정 → 시스템 → 문제 해결 → 기타 문제 해결사
  • "Windows Update" 항목에서 "실행" 클릭
  • 문제 해결사가 서비스 관련 문제 자동 진단
  • 발견된 문제에 대한 수정 사항 적용
2 고급 문제 해결사 실행:
  • Windows + R 키를 누르고 다음 명령어 입력:
msdt.exe -id WindowsUpdateDiagnostic
  • "고급" 옵션 선택
  • "관리자 권한으로 실행" 체크
  • "자동으로 복구 적용" 체크
  • 모든 권장 수정 사항 적용

방법 7: 시스템 파일 검사 및 복구

1 SFC 스캔 실행:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
sfc /scannow
  • 스캔 완료까지 기다리기 (30-90분 소요)
  • 손상된 파일 복구 완료 후 재부팅
2 DISM 명령어 실행:
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth

방법 8: 안전 모드에서 서비스 복구

1 안전 모드 진입:
  • 설정 → 업데이트 및 보안 → 복구
  • "고급 시작" 섹션에서 "지금 다시 시작" 클릭
  • "문제 해결" → "고급 옵션" → "시작 설정"
  • "다시 시작" 클릭 후 F5 키로 "네트워킹을 사용하는 안전 모드" 선택
2 안전 모드에서 서비스 복구:
  • 안전 모드에서 관리자 권한으로 명령 프롬프트 실행
  • 앞서 언급한 서비스 시작 명령어들 실행
  • 서비스 시작 유형을 자동으로 설정
  • 정상 모드로 재부팅 후 서비스 상태 확인

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
서비스 수동 시작 쉬움 10-15분 매우 높음 아니오
명령 프롬프트 서비스 시작 보통 10-20분 매우 높음 아니오
서비스 의존성 해결 보통 15-25분 높음 아니오
레지스트리 수정 어려움 15-30분 높음 예
그룹 정책 수정 어려움 15-25분 보통 예
Windows Update 문제 해결사 쉬움 15-25분 높음 예
시스템 파일 검사 쉬움 60-120분 보통 예
안전 모드 서비스 복구 보통 30-45분 높음 예

🔧 주요 명령어 참조

서비스 관리 명령어

  • net start wuauserv: Windows Update 서비스 시작
  • net stop wuauserv: Windows Update 서비스 중지
  • sc config wuauserv start= auto: 서비스 자동 시작 설정
  • sc query wuauserv: 서비스 상태 확인

서비스 진단 명령어

  • sc qc wuauserv: 서비스 구성 정보 확인
  • sc queryex wuauserv: 서비스 상세 정보 확인
  • services.msc: 서비스 관리 콘솔
  • gpupdate /force: 그룹 정책 강제 업데이트

⚠️ 주의사항

서비스 수정 시 주의점

  • 모든 작업은 관리자 권한으로 수행하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 그룹 정책 변경 시 조직의 IT 정책을 확인하세요
  • 서비스 시작 유형 변경 시 시스템 성능에 미치는 영향을 고려하세요
  • 타사 보안 프로그램이 서비스를 차단하지 않는지 확인하세요
  • 시스템 복원 지점을 생성한 후 작업하세요

언제 전문가의 도움이 필요한가?

  • 기업 환경에서 그룹 정책 문제가 복잡한 경우
  • 시스템 파일이 심각하게 손상된 경우
  • 바이러스 감염이 의심되는 경우
  • 여러 시스템 서비스가 동시에 문제를 보이는 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • Windows Update 자동 설정 유지
  • 정기적인 시스템 재시작 (주 1회 이상)
  • 시스템 최적화 도구 사용 시 주의
  • 신뢰할 수 있는 보안 프로그램만 사용
  • 정기적인 시스템 복원 지점 생성
  • 레지스트리 정리 도구 신중하게 사용
  • 시스템 서비스 임의 변경 금지

💡 추가 팁

  • 서비스 중지 오류 발생 시 이벤트 뷰어에서 상세 로그 확인
  • 서비스 의존성 관계를 이해하고 순서대로 시작
  • 그룹 정책 설정이 서비스에 미치는 영향 파악
  • 타사 프로그램이 Windows 서비스를 비활성화하지 않도록 주의
  • 정기적인 서비스 상태 점검으로 문제 조기 발견
  • 시스템 구성 유틸리티(msconfig)로 서비스 관리

🔍 관련 오류 코드

유사한 Windows Update 서비스 관련 오류들

  • 0x80240001: WU_E_NO_SERVICE
  • 0x80070005: 액세스가 거부되었습니다
  • 0x80070424: 서비스가 존재하지 않습니다
  • 0x8007042C: 서비스 데이터베이스가 잠겨 있습니다
  • 0x80070426: 서비스가 잘못된 상태입니다
  • 0x8007041D: 서비스 로그온에 실패했습니다

🛠️ 고급 진단 도구

서비스 진단 도구

  • 서비스 관리자: services.msc - 서비스 상태 및 설정 관리
  • 이벤트 뷰어: eventvwr.msc - 서비스 관련 오류 로그
  • 시스템 구성: msconfig - 서비스 시작 관리
  • 그룹 정책 편집기: gpedit.msc - 정책 설정 확인

PowerShell 진단 명령어

  • Get-Service wuauserv | Select-Object *: 서비스 상세 정보
  • Get-WinEvent -LogName System | Where-Object {$_.Id -eq 7036}: 서비스 시작/중지 로그
  • Test-NetConnection update.microsoft.com -Port 443: 네트워크 연결 테스트
  • Get-WindowsUpdateLog: Windows Update 로그 생성

📊 서비스 상태 확인 체크리스트

Windows Update 관련 주요 서비스 상태

서비스명 표시명 권장 시작 유형 정상 상태
wuauserv Windows Update 자동 (지연된 시작) 실행 중
BITS Background Intelligent Transfer Service 자동 (지연된 시작) 실행 중
CryptSvc Cryptographic Services 자동 실행 중
msiserver Windows Installer 수동 중지됨 (정상)
dosvc Delivery Optimization 자동 (지연된 시작) 실행 중

🔄 자동화 스크립트

Windows Update 서비스 복구 스크립트

@echo off
echo Windows Update 서비스 복구 시작...
echo.
echo 1단계: 서비스 중지 중...
net stop wuauserv
net stop cryptSvc
net stop bits
net stop dosvc
echo.
echo 2단계: 서비스 시작 유형 설정 중...
sc config wuauserv start= auto
sc config cryptsvc start= auto
sc config bits start= auto
sc config dosvc start= auto
echo.
echo 3단계: 서비스 시작 중...
net start cryptsvc
net start bits
net start wuauserv
net start dosvc
echo.
echo 4단계: 서비스 상태 확인 중...
sc query wuauserv
sc query bits
sc query cryptsvc
echo.
echo 서비스 복구 완료!
echo 이제 Windows Update를 확인하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 Windows Update 상태 확인

📈 서비스 관리 최적화 가이드

효율적인 서비스 관리 방법

  • 정기 점검: 월 1회 주요 서비스 상태 확인
  • 자동화: 서비스 모니터링 스크립트 활용
  • 문서화: 서비스 설정 변경 사항 기록
  • 백업: 서비스 관련 레지스트리 키 백업
  • 교육: 서비스 의존성 관계 이해
  • 도구 활용: 서비스 관리 도구 적극 활용
