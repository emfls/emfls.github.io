Windows 11 0x80240008 WU_E_ITEMNOTFOUND 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 시스템에서 요청된 업데이트 항목을 찾을 수 없을 때 발생하는 오류입니다. 업데이트 카탈로그 불일치나 파일 참조 실패가 주요 원인입니다.

🔍 오류 개요

0x80240008 WU_E_ITEMNOTFOUND는 Windows 11에서 업데이트 시스템이 특정 업데이트 항목이나 파일을 찾을 수 없을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 업데이트 카탈로그와 로컬 데이터베이스 불일치
  • 손상되거나 누락된 업데이트 파일
  • Windows Update 서버와의 동기화 실패
  • 부분적으로 다운로드된 업데이트 파일
  • 업데이트 메타데이터 손상
  • 시스템 시간 및 날짜 불일치
  • 네트워크 연결 중단으로 인한 파일 손실
  • Windows Update 캐시 손상

🛠️ 해결 방법

방법 1: Windows Update 캐시 완전 재설정

1 Windows Update 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop dosvc
net stop appidsvc
2 업데이트 캐시 폴더 완전 삭제:
  • 다음 명령어로 모든 캐시 폴더 제거:
rd /s /q %windir%\SoftwareDistribution
rd /s /q %windir%\System32\catroot2
del /f /q %windir%\WindowsUpdate.log
rd /s /q "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
3 새로운 캐시 폴더 생성:
  • 다음 명령어로 새 폴더 생성:
md %windir%\SoftwareDistribution
md %windir%\System32\catroot2
md "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"

방법 2: 시스템 시간 동기화 및 확인

1 시간 서비스 재시작:
  • 다음 명령어로 시간 서비스 재설정:
net stop w32time
w32tm /unregister
w32tm /register
net start w32time
2 시간 강제 동기화:
  • 다음 명령어로 정확한 시간 설정:
w32tm /config /manualpeerlist:"time.windows.com,0x1" /syncfromflags:manual /reliable:yes
w32tm /resync /force
w32tm /query /status
3 수동 시간 설정 확인:
  • 설정 → 시간 및 언어 → 날짜 및 시간
  • "시간 자동 설정" 활성화
  • "표준 시간대 자동 설정" 활성화
  • "지금 동기화" 클릭하여 즉시 동기화

방법 3: Windows Update 서버 재연결

1 업데이트 서버 연결 재설정:
  • 다음 명령어로 서버 연결 초기화:
wuauclt /resetauthorization
wuauclt /detectnow
wuauclt /updatenow
usoclient startscan
usoclient refreshsettings
2 서버 연결 테스트:
  • 다음 명령어로 Microsoft 서버 연결 확인:
ping update.microsoft.com
nslookup update.microsoft.com
telnet update.microsoft.com 80

방법 4: Windows Update 구성 요소 재등록

1 기존 구성 요소 등록 해제:
  • 다음 명령어들을 실행하여 기존 등록 해제:
regsvr32 /u wuapi.dll
regsvr32 /u wuaueng.dll
regsvr32 /u wuaueng1.dll
regsvr32 /u wucltui.dll
regsvr32 /u wups.dll
regsvr32 /u wups2.dll
regsvr32 /u wuweb.dll
regsvr32 /u qmgr.dll
regsvr32 /u qmgrprxy.dll
regsvr32 /u wucltux.dll
regsvr32 /u muweb.dll
regsvr32 /u wuwebv.dll
2 구성 요소 재등록:
  • 다음 명령어들을 실행하여 구성 요소 재등록:
regsvr32 /s wuapi.dll
regsvr32 /s wuaueng.dll
regsvr32 /s wuaueng1.dll
regsvr32 /s wucltui.dll
regsvr32 /s wups.dll
regsvr32 /s wups2.dll
regsvr32 /s wuweb.dll
regsvr32 /s qmgr.dll
regsvr32 /s qmgrprxy.dll
regsvr32 /s wucltux.dll
regsvr32 /s muweb.dll
regsvr32 /s wuwebv.dll

방법 5: 네트워크 연결 최적화

1 네트워크 스택 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew
netsh winsock reset
netsh int ip reset
netsh advfirewall reset
2 DNS 서버 변경:
  • 설정 → 네트워크 및 인터넷 → 이더넷 (또는 Wi-Fi)
  • "DNS 서버 할당" → "수동" 선택
  • 다음 DNS 서버 설정:
기본 DNS: 8.8.8.8
보조 DNS: 8.8.4.4
또는
기본 DNS: 1.1.1.1
보조 DNS: 1.0.0.1

방법 6: Windows Update 문제 해결사 실행

1 자동 문제 해결사 실행:
  • 설정 → 시스템 → 문제 해결 → 기타 문제 해결사
  • "Windows Update" 항목에서 "실행" 클릭
  • 문제 해결사가 누락된 항목 관련 문제 자동 진단
  • 발견된 문제에 대한 수정 사항 적용
2 고급 문제 해결사 실행:
  • Windows + R 키를 누르고 다음 명령어 입력:
msdt.exe -id WindowsUpdateDiagnostic
  • "고급" 옵션 선택
  • "관리자 권한으로 실행" 체크
  • "자동으로 복구 적용" 체크
  • 모든 권장 수정 사항 적용

방법 7: 시스템 파일 검사 및 복구

1 SFC 스캔 실행:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
sfc /scannow
  • 스캔 완료까지 기다리기 (30-90분 소요)
  • 손상된 파일 복구 완료 후 재부팅
2 DISM 명령어 실행:
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth
DISM /Online /Cleanup-Image /StartComponentCleanup

방법 8: 수동 업데이트 다운로드 및 설치

1 Microsoft Update 카탈로그 사용:
  • 웹 브라우저에서 Microsoft Update 카탈로그 접속
  • 실패한 업데이트의 KB 번호 검색
  • 시스템 아키텍처에 맞는 업데이트 다운로드
  • 다운로드한 파일을 관리자 권한으로 실행
2 Windows Update Assistant 사용:
  • Microsoft 공식 사이트에서 Windows 11 Update Assistant 다운로드
  • 관리자 권한으로 프로그램 실행
  • "지금 업데이트" 클릭하여 최신 업데이트 설치
  • 업데이트 완료 후 시스템 재시작
3 PowerShell을 통한 업데이트:
  • 관리자 권한으로 PowerShell 실행
  • PSWindowsUpdate 모듈 설치 및 사용:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-WindowsUpdate
Install-WindowsUpdate -AcceptAll -AutoReboot

🛠️ 고급 진단 도구

업데이트 항목 진단 도구

  • 이벤트 뷰어: eventvwr.msc - Windows Update 관련 오류 로그
  • 시스템 정보: msinfo32 - 시스템 상태 및 시간 확인
  • 서비스 관리자: services.msc - Windows Update 서비스 상태
  • 레지스트리 편집기: regedit - Windows Update 설정 확인

PowerShell 진단 명령어

  • Get-WindowsUpdateLog: Windows Update 로그 생성
  • Get-WUHistory: 업데이트 히스토리 확인
  • Get-WUList: 사용 가능한 업데이트 목록
  • Test-NetConnection update.microsoft.com -Port 443: 서버 연결 테스트

📊 업데이트 항목 누락 진단 체크리스트

단계별 진단 방법

  1. 오류 발생 시점 확인: 특정 업데이트에서만 발생하는지 확인
  2. 시스템 시간 확인: 날짜/시간이 정확한지 확인
  3. 네트워크 연결 상태: 인터넷 연결 및 Microsoft 서버 접근 가능 여부
  4. 캐시 상태 점검: Windows Update 캐시 폴더 내용 확인
  5. 서비스 상태 확인: Windows Update 관련 서비스 실행 상태
  6. 로그 파일 분석: 상세한 오류 정보 및 누락된 항목 확인
