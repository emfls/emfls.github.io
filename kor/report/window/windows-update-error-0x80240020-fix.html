Windows 0x80240020 WU_E_NO_INTERACTIVE_USER 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 대화형 사용자가 없을 때 발생하는 문제입니다. 사용자 세션 문제, 서비스 계정 설정, 또는 원격 데스크톱 환경에서의 업데이트 시도가 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240020 (WU_E_NO_INTERACTIVE_USER)는 Windows Update 시스템에서 대화형 사용자 세션이 없을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 원격 데스크톱 세션에서 Windows Update 실행
  • 서비스 계정으로 실행되는 업데이트 프로세스
  • 사용자 로그오프 상태에서의 자동 업데이트
  • Windows 서비스로 실행되는 업데이트 작업
  • 터미널 서비스 환경에서의 업데이트 문제
  • 사용자 세션 토큰 문제
  • 백그라운드에서 실행되는 업데이트 스크립트
  • 시스템 계정으로 실행되는 자동화된 업데이트
  • Fast User Switching 환경에서의 문제
  • Windows Update 서비스 권한 설정 문제

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 사용자 세션 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 대화형 사용자 세션에서 업데이트 실행

1 로컬 콘솔 세션에서 로그인:
  • 원격 데스크톱 연결을 종료
  • 컴퓨터에 직접 로그인
  • 관리자 계정으로 로그인
  • Windows Update 실행
2 현재 사용자 세션 상태 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 세션 상태 확인:
query user
query session
whoami
whoami /logonid
echo %SESSIONNAME%
3 대화형 세션 강제 생성:
  • 다음 명령어로 대화형 세션 확인:
tscon 1 /dest:console
logoff
shutdown /l

방법 3: Windows Update 서비스 계정 설정 수정

1 Windows Update 서비스 계정 확인:
  • services.msc 실행
  • "Windows Update" 서비스 찾기
  • 우클릭 → 속성 → 로그온 탭
  • 현재 서비스 계정 설정 확인
2 서비스 계정 설정 수정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 서비스 계정 설정:
sc config wuauserv obj= "NT AUTHORITY\LocalService"
sc config wuauserv type= interact
sc config bits obj= "NT AUTHORITY\LocalService"
sc config cryptSvc obj= "NT AUTHORITY\NetworkService"
3 서비스 권한 설정:
  • 다음 명령어로 서비스 권한 설정:
sc privs wuauserv SeAssignPrimaryTokenPrivilege/SeIncreaseQuotaPrivilege/SeAuditPrivilege/SeChangeNotifyPrivilege/SeCreateGlobalPrivilege/SeImpersonatePrivilege
sc config wuauserv start= auto
net start wuauserv

방법 4: 사용자 계정 컨트롤 (UAC) 설정 조정

1 UAC 설정 확인 및 조정:
  • 제어판 → 사용자 계정 → 사용자 계정 제어 설정 변경
  • 슬라이더를 "기본값" 또는 "알리지 않음"으로 설정
  • 또는 레지스트리를 통한 설정:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v EnableLUA /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v ConsentPromptBehaviorAdmin /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v PromptOnSecureDesktop /t REG_DWORD /d 0 /f
2 관리자 권한 자동 승격 설정:
  • 다음 명령어로 자동 승격 설정:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v FilterAdministratorToken /t REG_DWORD /d 0 /f
gpupdate /force

방법 5: Windows Update 서비스 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 캐시 및 설정 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작 및 권한 재설정:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 6: 레지스트리 권한 설정 복구

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 레지스트리 권한 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallDay /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallTime /t REG_DWORD /d 3 /f
3 사용자 세션 관련 레지스트리 설정:
  • 다음 명령어들을 실행:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /v AllowOSUpgrade /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /v DisableOSUpgrade /t REG_DWORD /d 0 /f
reg add "HKLM\SYSTEM\CurrentControlSet\Services\wuauserv" /v ServiceSidType /t REG_DWORD /d 1 /f

방법 7: 작업 스케줄러를 통한 업데이트 실행

1 작업 스케줄러에서 Windows Update 작업 생성:
  • taskschd.msc 실행
  • "작업 만들기" 클릭
  • 일반 탭에서 "가장 높은 수준의 권한으로 실행" 체크
  • "사용자가 로그온했는지 여부에 관계없이 실행" 선택
2 명령어를 통한 작업 생성:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 작업 생성:
schtasks /create /tn "WindowsUpdateFix" /tr "wuauclt /detectnow" /sc once /st 12:00 /ru SYSTEM
schtasks /create /tn "WindowsUpdateCheck" /tr "usoclient startscan" /sc daily /st 10:00 /ru SYSTEM
schtasks /run /tn "WindowsUpdateFix"

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 사용자 세션 확인 및 업데이트 강제 실행:
  • 다음 명령어들을 실행:
$CurrentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
Write-Host "현재 사용자: $($CurrentUser.Name)"
Write-Host "인증 유형: $($CurrentUser.AuthenticationType)"
Write-Host "대화형 여부: $($CurrentUser.IsAuthenticated)"
Get-WUServiceManager
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -Verbose
Reset-WUComponents -Verbose

🔧 주요 명령어 참조

사용자 세션 관련 명령어

  • query user: 현재 로그인된 사용자 확인
  • query session: 활성 세션 확인
  • whoami: 현재 사용자 정보 확인
  • whoami /logonid: 로그온 세션 ID 확인

서비스 계정 관련 명령어

  • sc config [서비스명] obj= [계정]: 서비스 계정 설정
  • sc config [서비스명] type= interact: 대화형 서비스 설정
  • sc privs [서비스명] [권한]: 서비스 권한 설정
  • sc qc [서비스명]: 서비스 구성 확인

작업 스케줄러 관련 명령어

  • schtasks /create: 새 작업 생성
  • schtasks /run /tn [작업명]: 작업 즉시 실행
  • schtasks /query: 작업 목록 확인
  • schtasks /delete /tn [작업명]: 작업 삭제

  • 중요한 데이터는 백업 후 작업하세요
  • UAC 설정 변경 시 보안 위험을 고려하세요
  • 서비스 계정 변경 시 시스템 안정성을 확인하세요
  • 원격 데스크톱 환경에서는 로컬 콘솔 접근을 권장합니다
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 관리자 권한으로 모든 작업을 수행하세요
  • 서버 환경에서는 특별한 주의가 필요합니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 터미널 서비스 환경에서 복잡한 설정이 필요한 경우
  • 기업 환경에서 도메인 정책과 충돌하는 경우
  • 서버 환경에서 서비스 계정 설정이 복잡한 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 보안 정책으로 인한 제한이 있는 경우
  • 시스템 전체 재설치가 필요한 경우

  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요

🔍 관련 오류 코드

유사한 Windows Update 사용자 세션 관련 오류들

  • 0x8024001F: 연결 없음 (WU_E_NO_CONNECTION)
  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x80240018: 사용자 토큰 없음 (WU_E_NO_USERTOKEN)
  • 0x80240021: 업데이트 설치 관리자 없음 (WU_E_NO_UPDATE_INSTALLER)
  • 0x80240022: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)
  • 0x80070005: 액세스 거부됨
  • 0x80070057: 매개 변수가 올바르지 않음

🛠️ 고급 해결 방법

사용자 세션 상태 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 사용자 세션 상태 분석:
$CurrentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
$Principal = New-Object Security.Principal.WindowsPrincipal($CurrentUser)
Write-Host "=== 사용자 세션 정보 ===" -ForegroundColor Green
Write-Host "사용자 이름: $($CurrentUser.Name)"
Write-Host "인증 유형: $($CurrentUser.AuthenticationType)"
Write-Host "인증 여부: $($CurrentUser.IsAuthenticated)"
Write-Host "익명 여부: $($CurrentUser.IsAnonymous)"
Write-Host "게스트 여부: $($CurrentUser.IsGuest)"
Write-Host "시스템 여부: $($CurrentUser.IsSystem)"
Write-Host "관리자 권한: $($Principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator))"
Write-Host "세션 이름: $env:SESSIONNAME"
Write-Host "로그온 서버: $env:LOGONSERVER"

Windows Update 서비스 권한 복구 스크립트

  • 서비스 권한 자동 복구:
$Services = @("wuauserv", "cryptSvc", "bits", "msiserver")
foreach ($Service in $Services) {
    Write-Host "서비스 복구 중: $Service" -ForegroundColor Yellow
    Stop-Service -Name $Service -Force -ErrorAction SilentlyContinue
    Start-Sleep -Seconds 2
    Set-Service -Name $Service -StartupType Automatic
    Start-Service -Name $Service -ErrorAction SilentlyContinue
    $Status = (Get-Service -Name $Service).Status
    Write-Host "서비스 $Service 상태: $Status" -ForegroundColor Green
}

🔧 원격 환경 최적화

원격 데스크톱 환경에서의 Windows Update

  • 콘솔 세션 사용: mstsc /console 또는 /admin 옵션
  • 세션 0 격리: Windows Vista 이후 세션 0에서 서비스 실행
  • 대화형 서비스 감지: 대화형 서비스 감지 서비스 활성화
  • 터미널 서비스: 다중 사용자 환경에서의 특별 고려사항

서버 환경에서의 고려사항

  • Windows Server에서는 기본적으로 자동 업데이트 비활성화
  • WSUS 또는 Windows Update for Business 사용 권장
  • 서버 코어 설치에서는 PowerShell 또는 sconfig 사용
  • 클러스터 환경에서는 CAU (Cluster-Aware Updating) 사용

🏢 기업 환경 특별 고려사항

도메인 환경에서의 추가 확인사항

  • 그룹 정책: Windows Update 관련 도메인 정책
  • WSUS 서버: 기업 업데이트 서버 설정
  • 터미널 서비스: RDS 환경에서의 업데이트 정책
  • 보안 정책: 기업 보안 정책 준수
  • 서비스 계정: 도메인 서비스 계정 사용

IT 관리자와 상의해야 할 사항

  • 도메인 그룹 정책 설정
  • WSUS 서버 설정 및 승인 정책
  • 터미널 서비스 라이선스 정책
  • 보안 정책 및 권한 설정
  • 서비스 계정 권한 및 정책
