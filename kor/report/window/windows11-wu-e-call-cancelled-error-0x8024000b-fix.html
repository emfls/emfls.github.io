Windows 11 0x8024000B WU_E_CALL_CANCELLED 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 시스템에서 업데이트 호출이 사용자나 시스템에 의해 취소되었을 때 발생하는 오류입니다. 의도적 취소나 시스템 간섭이 주요 원인입니다.

🔍 오류 개요

0x8024000B WU_E_CALL_CANCELLED는 Windows 11에서 업데이트 시스템의 API 호출이 사용자나 시스템에 의해 취소되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 사용자가 업데이트 프로세스를 수동으로 취소
  • 시스템 종료 또는 재시작으로 인한 호출 중단
  • 타사 소프트웨어에 의한 업데이트 프로세스 간섭
  • 보안 프로그램의 Windows Update 차단
  • 네트워크 연결 중단으로 인한 호출 취소
  • 시스템 리소스 부족으로 인한 자동 취소
  • 그룹 정책에 의한 업데이트 제한
  • Windows Update 서비스 타임아웃

🛠️ 해결 방법

방법 1: 업데이트 재시도 및 완료까지 대기

1 업데이트 상태 확인:
  • 설정 → 업데이트 및 보안 → Windows Update
  • 현재 진행 중이거나 대기 중인 업데이트 확인
  • 이전에 취소된 업데이트가 있는지 확인
  • "업데이트 확인" 클릭하여 새로운 검색 시작
2 업데이트 완료까지 중단 없이 대기:
  • 업데이트 시작 후 임의로 취소하지 않기
  • 시스템 종료나 재시작 피하기
  • 다른 무거운 프로그램 실행 자제
  • 네트워크 연결 안정성 확인
  • 전원 공급 안정성 확보

방법 2: Windows Update 서비스 재시작

1 Windows Update 관련 서비스 재시작:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop dosvc
2 서비스 재시작:
  • 잠시 대기 후 다음 명령어들을 실행:
net start cryptSvc
net start bits
net start msiserver
net start wuauserv
net start dosvc
3 업데이트 권한 재설정:
  • 다음 명령어로 업데이트 권한 재설정:
wuauclt /resetauthorization
wuauclt /detectnow
usoclient startscan

방법 3: 보안 프로그램 및 방화벽 확인

1 타사 보안 프로그램 확인:
  • 설치된 백신 프로그램의 실시간 보호 기능 임시 비활성화
  • 방화벽 설정에서 Windows Update 관련 프로세스 허용
  • 보안 프로그램의 업데이트 차단 기능 확인
  • 다음 프로세스들이 차단되지 않았는지 확인:
    • wuauclt.exe
    • usoclient.exe
    • svchost.exe (BITS)
    • TiWorker.exe
2 Windows Defender 설정 확인:
  • Windows 보안 → 바이러스 및 위협 방지
  • 실시간 보호 설정 확인
  • 제외 항목에 Windows Update 관련 폴더 추가 (필요시):
    • C:\Windows\SoftwareDistribution
    • C:\Windows\System32\catroot2

방법 4: 네트워크 연결 안정화

1 네트워크 연결 상태 확인:
  • 인터넷 연결 안정성 테스트
  • Wi-Fi 사용 시 유선 연결로 변경 (가능한 경우)
  • 네트워크 속도 및 대역폭 확인
  • 라우터/모뎀 재시작
2 네트워크 설정 최적화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
ipconfig /flushdns
ipconfig /registerdns
netsh winsock reset
netsh int ip reset

방법 5: 시스템 리소스 최적화

1 불필요한 프로그램 종료:
  • 작업 관리자에서 CPU 및 메모리 사용량이 높은 프로그램 확인
  • 업데이트와 관련 없는 무거운 프로그램 종료
  • 백그라운드에서 실행 중인 불필요한 서비스 중지
  • 시작 프로그램 최소화
2 디스크 공간 확보:
  • 디스크 정리 도구 실행:
cleanmgr
  • 다음 항목들을 체크하여 삭제:
    • 임시 파일
    • 시스템 오류 메모리 덤프 파일
    • Windows Update 정리
    • 휴지통

방법 6: 그룹 정책 확인 및 수정

1 로컬 그룹 정책 편집기 실행:
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 다음 경로로 이동:
컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
2 Windows Update 정책 확인:
  • 다음 정책들이 "사용 안 함" 또는 "구성되지 않음"으로 설정되어 있는지 확인:
    • "자동 업데이트 구성"
    • "인트라넷 Microsoft 업데이트 서비스 위치 지정"
    • "자동 업데이트에서 즉시 설치 허용 안 함"
  • 제한적인 정책이 있다면 "구성되지 않음"으로 변경

방법 7: Windows Update 문제 해결사 실행

1 자동 문제 해결사 실행:
  • 설정 → 시스템 → 문제 해결 → 기타 문제 해결사
  • "Windows Update" 항목에서 "실행" 클릭
  • 문제 해결사가 호출 취소 관련 문제 자동 진단
  • 발견된 문제에 대한 수정 사항 적용
2 고급 문제 해결사 실행:
  • Windows + R 키를 누르고 다음 명령어 입력:
msdt.exe -id WindowsUpdateDiagnostic
  • "고급" 옵션 선택
  • "관리자 권한으로 실행" 체크
  • "자동으로 복구 적용" 체크
  • 모든 권장 수정 사항 적용

방법 8: 수동 업데이트 설치

1 Microsoft Update 카탈로그 사용:
  • 웹 브라우저에서 Microsoft Update 카탈로그 접속
  • 취소된 업데이트의 KB 번호 검색
  • 시스템 아키텍처에 맞는 업데이트 다운로드
  • 다운로드한 파일을 관리자 권한으로 실행
2 PowerShell을 통한 업데이트:
  • 관리자 권한으로 PowerShell 실행
  • PSWindowsUpdate 모듈 설치 및 사용:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-WindowsUpdate
Install-WindowsUpdate -AcceptAll -IgnoreReboot

🔧 주요 명령어 참조

서비스 관리 명령어

  • net start wuauserv: Windows Update 서비스 시작
  • net stop wuauserv: Windows Update 서비스 중지
  • sc query wuauserv: Windows Update 서비스 상태 확인
  • wuauclt /resetauthorization: 업데이트 권한 재설정

네트워크 관련 명령어

  • ipconfig /flushdns: DNS 캐시 삭제
  • netsh winsock reset: Winsock 재설정
  • ping update.microsoft.com: Microsoft 서버 연결 테스트
  • nslookup update.microsoft.com: DNS 조회 테스트

⚠️ 주의사항

업데이트 취소 방지 주의점

  • 업데이트 진행 중에는 시스템을 강제로 종료하지 마세요
  • 보안 프로그램 비활성화는 임시로만 사용하세요
  • 그룹 정책 변경 시 조직의 IT 정책을 확인하세요
  • 네트워크 설정 변경 전 현재 설정을 기록해두세요
  • 업데이트 중에는 다른 무거운 작업을 피하세요
  • 전원 공급이 불안정한 환경에서는 UPS 사용을 권장합니다

언제 전문가의 도움이 필요한가?

  • 기업 환경에서 그룹 정책 문제가 복잡한 경우
  • 네트워크 인프라 문제가 의심되는 경우
  • 보안 정책으로 인해 업데이트가 제한되는 경우
  • 시스템이 반복적으로 업데이트를 취소하는 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 충분한 디스크 공간 확보

💡 추가 팁

  • 업데이트 시간대를 시스템 사용량이 적은 시간으로 설정
  • 대용량 업데이트는 네트워크 트래픽이 적은 시간에 실행
  • 업데이트 완료 후 시스템 복원 지점 생성
  • 정기적인 Windows Update 로그 확인으로 패턴 파악

🛠️ 고급 진단 도구

호출 취소 진단 도구

  • 이벤트 뷰어: eventvwr.msc - Windows Update 관련 취소 로그
  • 작업 관리자: taskmgr - 시스템 리소스 사용량 모니터링
  • 리소스 모니터: resmon - 상세한 리소스 분석
  • 성능 모니터: perfmon - 시스템 성능 추적

PowerShell 진단 명령어

  • Get-WindowsUpdateLog: Windows Update 로그 생성
  • Get-WUHistory: 업데이트 히스토리 확인
  • Get-WUList: 사용 가능한 업데이트 목록
  • Test-NetConnection update.microsoft.com -Port 443: 서버 연결 테스트

📊 업데이트 호출 취소 진단 체크리스트

단계별 진단 방법

  1. 취소 원인 확인: 사용자 취소인지 시스템 취소인지 확인
  2. 시스템 리소스 확인: CPU, 메모리, 디스크 사용률 점검
  3. 네트워크 상태 확인: 연결 안정성 및 대역폭 확인
  4. 보안 프로그램 확인: 업데이트 차단 여부 확인
  5. 그룹 정책 확인: 업데이트 제한 정책 존재 여부
  6. 로그 파일 분석: 상세한 취소 원인 및 시점 확인

🔄 자동화 스크립트

Windows Update 호출 취소 문제 해결 스크립트

@echo off
echo Windows Update 호출 취소 문제 해결 시작...
echo 현재 Windows Update 서비스 상태 확인 중...
sc query wuauserv
echo Windows Update 서비스 재시작 중...
net stop wuauserv
net stop cryptSvc
net stop bits
net stop dosvc
timeout /t 5
net start cryptSvc
net start bits
net start wuauserv
net start dosvc
echo 업데이트 권한 재설정 중...
wuauclt /resetauthorization
wuauclt /detectnow
echo 네트워크 설정 최적화 중...
ipconfig /flushdns
netsh winsock reset
echo 임시 파일 정리 중...
del /f /s /q %temp%\*.* 2>nul
del /f /s /q %windir%\Temp\*.* 2>nul
echo 문제 해결 완료!
echo 이제 Windows Update를 다시 시도하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 Windows Update 재시도
