Windows 11 0xC1900208 INCOMPATIBLE_APP_BLOCKING 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 업그레이드나 업데이트 과정에서 호환되지 않는 애플리케이션이 설치되어 있어 진행이 차단될 때 발생하는 오류입니다. 문제가 되는 앱을 제거하거나 업데이트해야 합니다.

🔍 오류 개요

0xC1900208 INCOMPATIBLE_APP_BLOCKING는 Windows 11 업그레이드나 주요 업데이트 과정에서 호환되지 않는 애플리케이션이 감지되어 설치가 차단될 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 오래된 버전의 백신 프로그램
  • 호환되지 않는 드라이버 소프트웨어
  • 구버전 시스템 유틸리티 프로그램
  • Windows 11과 호환되지 않는 타사 소프트웨어
  • 오래된 게임 런처나 게임 소프트웨어
  • 호환되지 않는 가상화 소프트웨어
  • 구버전 개발 도구 및 IDE
  • Windows 11에서 지원 중단된 레거시 애플리케이션

🛠️ 해결 방법

방법 1: 호환되지 않는 앱 식별 및 제거

1 Windows 11 호환성 검사 도구 실행:
  • Microsoft 공식 사이트에서 PC 상태 검사 앱 다운로드
  • 프로그램 실행 후 "지금 확인" 클릭
  • 호환되지 않는 앱 목록 확인
  • 각 앱의 호환성 문제 상세 내용 검토
2 문제가 되는 앱 제거:
  • 설정 → 앱 → 앱 및 기능
  • 호환되지 않는 앱을 검색하여 선택
  • "제거" 버튼 클릭
  • 제거 과정 완료 후 시스템 재시작
  • 또는 제어판 → 프로그램 및 기능에서 제거

방법 2: 일반적으로 문제가 되는 소프트웨어 확인

1 백신 프로그램 확인:
  • 다음과 같은 구버전 백신 프로그램들을 확인:
    • Norton Antivirus (2018 이전 버전)
    • McAfee Antivirus (구버전)
    • Kaspersky (구버전)
    • AVG Antivirus (구버전)
    • Avast Antivirus (구버전)
  • 최신 버전으로 업데이트하거나 임시 제거
2 시스템 유틸리티 프로그램 확인:
  • 다음과 같은 프로그램들을 확인:
    • CCleaner (구버전)
    • IObit 제품군 (Advanced SystemCare, Driver Booster 등)
    • Wise Care 365
    • PC Cleaner Pro
    • Registry Cleaner 류 프로그램
  • 최신 버전으로 업데이트하거나 제거

방법 3: 드라이버 관련 소프트웨어 업데이트

1 그래픽 드라이버 소프트웨어 확인:
  • NVIDIA GeForce Experience 최신 버전 확인
  • AMD Radeon Software 최신 버전 확인
  • Intel Graphics Command Center 업데이트
  • 구버전 드라이버 제거 후 최신 버전 설치
2 기타 하드웨어 관련 소프트웨어:
  • 사운드 카드 드라이버 및 소프트웨어
  • 네트워크 어댑터 관리 소프트웨어
  • 프린터 및 스캐너 소프트웨어
  • 웹캠 및 기타 USB 장치 소프트웨어
  • 제조사 공식 사이트에서 Windows 11 호환 버전 다운로드

방법 4: 가상화 및 개발 도구 업데이트

1 가상화 소프트웨어 확인:
  • VMware Workstation/Player 최신 버전 확인
  • Oracle VirtualBox 최신 버전 확인
  • Parallels Desktop (Mac용이지만 Boot Camp 관련)
  • 구버전 제거 후 Windows 11 호환 버전 설치
2 개발 도구 및 IDE 확인:
  • Visual Studio (구버전) → 최신 버전으로 업데이트
  • Android Studio 최신 버전 확인
  • Eclipse IDE 최신 버전 확인
  • Git 클라이언트 프로그램 업데이트
  • Docker Desktop Windows 11 호환 버전 설치

방법 5: 레지스트리를 통한 강제 업그레이드 (고급)

1 레지스트리 편집기 실행:
  • Windows + R 키를 누르고 regedit 입력
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SYSTEM\Setup\MoSetup
  • 우클릭 → 새로 만들기 → DWORD (32비트) 값
  • 이름: AllowUpgradesWithUnsupportedTPMOrCPU
  • 값: 1
2 호환성 검사 우회 (주의 필요):
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SYSTEM\Setup\LabConfig
  • 다음 DWORD 값들을 생성:
    • BypassTPMCheck: 1
    • BypassRAMCheck: 1
    • BypassSecureBootCheck: 1
    • BypassStorageCheck: 1

방법 6: Windows 11 설치 미디어를 이용한 업그레이드

1 Windows 11 설치 미디어 준비:
  • Microsoft 공식 사이트에서 Windows 11 ISO 다운로드
  • Media Creation Tool 사용하여 USB 설치 미디어 생성
  • 또는 ISO 파일을 직접 마운트
2 호환성 검사 우회하여 설치:
  • 설치 미디어에서 setup.exe 실행
  • 설치 과정에서 Shift + F10 키로 명령 프롬프트 열기
  • 다음 명령어 실행:
reg add "HKLM\SYSTEM\Setup\MoSetup" /v "AllowUpgradesWithUnsupportedTPMOrCPU" /t REG_DWORD /d 1 /f
  • 설치 계속 진행

방법 7: 임시 앱 비활성화

1 시작 프로그램 비활성화:
  • 작업 관리자 실행 (Ctrl + Shift + Esc)
  • "시작 프로그램" 탭 클릭
  • 호환되지 않는 앱들을 "사용 안 함"으로 설정
  • 시스템 재시작 후 업그레이드 재시도
2 서비스 임시 중지:
  • Windows + R 키를 누르고 services.msc 입력
  • 문제가 되는 앱의 서비스 찾기
  • 서비스 우클릭 → "중지"
  • 시작 유형을 "수동"으로 변경
  • 업그레이드 완료 후 다시 활성화

방법 8: 클린 부팅 상태에서 업그레이드

1 시스템 구성 유틸리티 실행:
  • Windows + R 키를 누르고 msconfig 입력
  • "일반" 탭에서 "선택적 시작" 선택
  • "시스템 서비스 로드" 및 "시작 항목 로드" 체크 해제
  • "서비스" 탭에서 "Microsoft 서비스 모두 숨기기" 체크
  • "모두 사용 안 함" 클릭
2 클린 부팅 후 업그레이드:
  • 시스템 재시작
  • 최소한의 서비스만 실행되는 상태에서 Windows 11 업그레이드 시도
  • 업그레이드 완료 후 시스템 구성을 "일반 시작"으로 복원

📋 문제 해결 체크리스트

가상화/개발 도구 업데이트 보통 60-180분 보통 예

클린 부팅 업그레이드 보통 180-360분 높음 예

🔧 주요 명령어 참조

앱 제거 관련 명령어

  • appwiz.cpl: 프로그램 및 기능 열기
  • ms-settings:appsfeatures: 앱 및 기능 설정 열기
  • wmic product get name: 설치된 프로그램 목록 확인
  • Get-AppxPackage | Select Name: UWP 앱 목록 확인

시스템 구성 관련 명령어

  • msconfig: 시스템 구성 유틸리티
  • services.msc: 서비스 관리 콘솔
  • taskmgr: 작업 관리자
  • regedit: 레지스트리 편집기

⚠️ 주의사항

앱 제거 및 업그레이드 시 주의점

  • 중요한 데이터는 미리 백업하세요
  • 라이선스가 있는 소프트웨어의 경우 재설치 방법을 미리 확인하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 호환성 검사 우회는 시스템 안정성에 영향을 줄 수 있습니다
  • 업그레이드 과정 중 전원이 차단되지 않도록 주의하세요
  • 네트워크 연결이 안정적인 환경에서 진행하세요

언제 전문가의 도움이 필요한가?

  • 기업용 특수 소프트웨어가 설치된 경우
  • 중요한 업무용 시스템에서 문제가 발생한 경우
  • 하드웨어 호환성 문제가 의심되는 경우
  • 여러 번 시도했지만 계속 실패하는 경우
  • 시스템이 부팅되지 않는 상황이 발생한 경우

✅ 예방 방법

  • 정기적으로 설치된 프로그램의 Windows 11 호환성 확인
  • 소프트웨어 업데이트를 최신 상태로 유지
  • 불필요한 프로그램 정기적 제거
  • Windows 업그레이드 전 호환성 검사 도구 실행
  • 중요한 데이터 정기적 백업
  • 제조사 공식 사이트에서만 소프트웨어 다운로드
  • 시스템 복원 지점 정기적 생성

💡 추가 팁

  • 업그레이드 전 PC 상태 검사 앱으로 미리 호환성 확인
  • 문제가 되는 앱의 대체 프로그램 미리 조사
  • 업그레이드 후 필요한 소프트웨어 재설치 계획 수립
  • 제조사별 Windows 11 호환성 가이드 참조
  • 베타 버전이나 개발자 버전 소프트웨어 사용 자제
  • 가상 머신에서 먼저 테스트해보기

🔍 관련 오류 코드

유사한 Windows 11 업그레이드 관련 오류들

  • 0xC1900200: 시스템이 최소 요구 사항을 충족하지 않음
  • 0xC1900201: 호환되지 않는 앱이 발견됨
  • 0xC1900204: 마이그레이션 선택이 취소됨
  • 0xC1900209: 호환되지 않는 앱으로 인한 업그레이드 차단
  • 0xC1900210: 디스크 공간이 부족함
  • 0x80070002: 파일을 찾을 수 없음

🛠️ 고급 진단 도구

호환성 진단 도구

  • PC 상태 검사: Microsoft 공식 호환성 검사 도구
  • Windows 11 호환성 검사: 하드웨어 및 소프트웨어 호환성 확인
  • 이벤트 뷰어: eventvwr.msc - 업그레이드 관련 오류 로그
  • 시스템 정보: msinfo32 - 시스템 구성 정보

PowerShell 진단 명령어

  • Get-AppxPackage | Where-Object {$_.Name -like "*앱이름*"}: 특정 UWP 앱 확인
  • Get-WmiObject -Class Win32_Product: 설치된 프로그램 목록
  • Get-Service | Where-Object {$_.Status -eq "Running"}: 실행 중인 서비스
  • Get-Process | Sort-Object CPU -Descending: CPU 사용률 높은 프로세스

🔄 자동화 스크립트

호환되지 않는 앱 확인 스크립트

@echo off
echo 호환되지 않는 앱 확인 시작...
echo.
echo 설치된 프로그램 목록을 확인합니다...
wmic product get name,version > installed_programs.txt
echo.
echo UWP 앱 목록을 확인합니다...
powershell "Get-AppxPackage | Select Name,Version | Out-File uwp_apps.txt"
echo.
echo 실행 중인 서비스를 확인합니다...
sc query state= all > services_list.txt
echo.
echo 시작 프로그램을 확인합니다...
wmic startup get caption,command > startup_programs.txt
echo.
echo 확인 완료! 생성된 파일들을 검토하세요:
echo - installed_programs.txt
echo - uwp_apps.txt
echo - services_list.txt
echo - startup_programs.txt
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 생성된 파일들을 검토하여 호환성 문제 소프트웨어 확인

📈 업그레이드 성공률 향상 가이드

단계별 준비 과정

  1. 사전 준비: 중요 데이터 백업 및 복원 지점 생성
  2. 호환성 검사: PC 상태 검사 앱으로 사전 확인
  3. 소프트웨어 정리: 불필요한 프로그램 제거
  4. 업데이트: 필수 소프트웨어 최신 버전으로 업데이트
  5. 시스템 정리: 임시 파일 정리 및 디스크 정리
  6. 업그레이드 실행: 안정적인 환경에서 업그레이드 진행
  7. 사후 확인: 업그레이드 후 시스템 상태 점검

이 가이드를 통해 호환되지 않는 앱 차단 문제가 해결되기를 바랍니다. Windows 11 업그레이드 관련 문제가 지속되면 Microsoft 지원팀에 문의하시기 바랍니다.
