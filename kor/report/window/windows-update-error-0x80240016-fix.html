Windows 0x80240016 WU_E_INSTALL_NOT_ALLOWED 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 업데이트 설치가 허용되지 않을 때 발생하는 문제입니다. 시스템 정책, 권한 설정, 또는 보안 제한으로 인해 업데이트 설치가 차단될 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240016 (WU_E_INSTALL_NOT_ALLOWED)는 Windows Update 시스템에서 업데이트 설치가 허용되지 않을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 그룹 정책에 의한 Windows Update 설치 제한
  • 사용자 계정 권한 부족
  • Windows Update 서비스 권한 설정 문제
  • 보안 소프트웨어에 의한 업데이트 차단
  • 시스템 관리자에 의한 업데이트 제한 설정
  • WSUS 서버 정책에 의한 설치 제한
  • 레지스트리 권한 설정 문제
  • Windows Update 구성 요소 손상
  • 시스템 파일 권한 설정 오류
  • 기업 환경의 도메인 정책 제한

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 권한 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 사용자 계정 권한 확인 및 수정

1 관리자 권한 확인:
  • 현재 사용자 계정이 관리자 권한을 가지고 있는지 확인
  • 제어판 → 사용자 계정 → 사용자 계정 관리
  • 계정 유형이 "관리자"인지 확인
  • 필요시 관리자 계정으로 로그인
2 UAC 설정 확인 및 조정:
  • 제어판 → 사용자 계정 → 사용자 계정 제어 설정 변경
  • UAC 슬라이더를 "알리지 않음" 또는 "기본값"으로 설정
  • 또는 관리자 권한으로 명령 프롬프트에서 다음 명령어 실행:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v EnableLUA /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v ConsentPromptBehaviorAdmin /t REG_DWORD /d 0 /f

방법 3: 그룹 정책 설정 확인 및 수정

1 로컬 그룹 정책 편집기 실행:
  • Windows + R 키를 누르고 gpedit.msc 입력 (Pro/Enterprise 버전)
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들을 "구성되지 않음"으로 설정:
  • - "자동 업데이트 구성"
  • - "인트라넷 Microsoft 업데이트 서비스 위치 지정"
  • - "자동 업데이트를 즉시 설치할 수 있도록 허용"
2 레지스트리를 통한 정책 재설정 (Home 버전):
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKLM\SOFTWARE\Microsoft\PolicyManager\current\device\Update" /f
gpupdate /force

방법 4: Windows Update 서비스 권한 복구

1 Windows Update 서비스 중지 및 권한 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 서비스 권한 및 설정 복구:
  • 다음 명령어들을 실행하여 서비스 권한 복구:
sc config wuauserv start= auto
sc config cryptSvc start= auto
sc config bits start= auto
sc config msiserver start= demand
sc config appidsvc start= demand
sc config dosvc start= auto
sc config UsoSvc start= auto
sc config WaaSMedicSvc start= demand
sc config TrustedInstaller start= demand
3 서비스 재시작:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc

방법 5: 보안 소프트웨어 설정 확인

1 Windows Defender 설정 확인:
  • Windows 보안 → 바이러스 및 위협 방지
  • "바이러스 및 위협 방지 설정" → "설정 관리"
  • "실시간 보호" 임시 비활성화
  • "클라우드 제공 보호" 확인
2 타사 보안 소프트웨어 설정:
  • 설치된 보안 소프트웨어의 Windows Update 차단 설정 확인
  • 방화벽에서 Windows Update 관련 프로세스 허용
  • 필요시 보안 소프트웨어 임시 비활성화
  • 업데이트 완료 후 보안 소프트웨어 재활성화

방법 6: Windows Update 캐시 및 데이터 재구축

1 Windows Update 캐시 완전 삭제:
  • 앞서 중지한 서비스 상태에서 다음 명령어 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
rd /s /q %windir%\WindowsUpdate.log
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
2 권한 재설정 및 서비스 재시작:
  • 다음 명령어로 권한 재설정:
icacls C:\Windows\SoftwareDistribution /grant "NT AUTHORITY\SYSTEM:(OI)(CI)F" /T
icacls C:\Windows\System32\catroot2 /grant "NT AUTHORITY\SYSTEM:(OI)(CI)F" /T
wuauclt /resetauthorization /detectnow
wuauclt /updatenow

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 권한 확인 및 업데이트 강제 설치:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -Verbose
Get-WindowsUpdateLog -LogPath C:\WindowsUpdate.log
Reset-WUComponents -Verbose

설치 허용되지 않음 오류 해결 시 주의점

  • 중요한 데이터는 백업 후 작업하세요
  • 그룹 정책 변경 시 기업 환경에서는 IT 관리자와 상의하세요

  • 관리자 권한으로 모든 작업을 수행하세요
  • UAC 설정 변경 시 보안 위험을 고려하세요
  • 서비스 권한 변경 시 시스템 안정성을 확인하세요
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 기업 환경에서 도메인 정책으로 인한 제한이 있는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 보안 정책 변경이 다른 시스템 기능에 영향을 주는 경우
  • WSUS 서버 설정 문제로 인한 업데이트 제한
  • 하드웨어 호환성 문제로 인한 업데이트 실패
  • 시스템 전체 재설치가 필요한 경우

  • 정기적인 권한 설정 검토를 수행하세요

🔍 관련 오류 코드

유사한 Windows Update 권한 관련 오류들

  • 0x80070005: 액세스 거부됨
  • 0x80240017: 업데이트가 설치되지 않음 (WU_E_NOT_INSTALLED)
  • 0x80240015: 업데이트 인덱스 무효 (WU_E_INVALID_UPDATE_INDEX)
  • 0x80240014: 업데이트 ID 무효 (WU_E_INVALID_UPDATE_ID)
  • 0x80240013: 중복 항목 (WU_E_DUPLICATE_ITEM)
  • 0x80240012: 레지스트리 값 무효 (WU_E_REG_VALUE_INVALID)
  • 0x8024402C: 업데이트 서버에 연결할 수 없음

🛠️ 고급 해결 방법

권한 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 사용자 권한 확인:
$currentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
$principal = New-Object Security.Principal.WindowsPrincipal($currentUser)
$isAdmin = $principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
Write-Host "관리자 권한: $isAdmin"
Get-Service wuauserv | Select-Object Name, Status, StartType
Get-WmiObject -Class Win32_OperatingSystem | Select-Object Caption, Version

그룹 정책 확인 스크립트

  • Windows Update 관련 정책 확인:
$WUPolicies = @(
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate",
    "HKCU:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate"
)
foreach ($Policy in $WUPolicies) {
    if (Test-Path $Policy) {
        Write-Host "정책 발견: $Policy"
        Get-ItemProperty $Policy
    }
}

  • 오류 발생 시점 및 상황 설명
  • 시스템 정보 (Windows 버전, 빌드 번호)
  • 사용자 계정 유형 및 권한 정보
  • 그룹 정책 설정 정보
  • 설치된 보안 소프트웨어 정보
  • 이벤트 뷰어의 오류 로그
  • 네트워크 환경 정보 (기업 환경의 경우)
