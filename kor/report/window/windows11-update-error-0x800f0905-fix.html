Windows 11 0x800f0905 Windows Update Error 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 업데이트 다운로드 중 발생하는 네트워크 관련 문제입니다. 아래 해결 방법을 순서대로 시도하시기 바랍니다.

🔍 오류 개요

0x800f0905 Windows Update Error는 Windows 11에서 업데이트 다운로드 실패 시 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 네트워크 연결 불안정
  • Windows Update 서버 연결 문제
  • 방화벽 또는 보안 프로그램 차단
  • 프록시 서버 설정 문제
  • DNS 서버 문제
  • 업데이트 서비스 오류
  • 시스템 날짜/시간 불일치
  • 대역폭 제한 설정

🛠️ 해결 방법

방법 1: 네트워크 연결 확인 및 재설정

1 인터넷 연결 확인:
  • 웹 브라우저에서 다른 사이트 접속 테스트
  • 네트워크 케이블 또는 Wi-Fi 연결 상태 확인
  • 라우터/모뎀 재시작 (전원 끄고 30초 후 다시 켜기)
  • 다른 네트워크(모바일 핫스팟 등)로 테스트
2 네트워크 어댑터 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 입력:
ipconfig /release
ipconfig /flushdns
ipconfig /renew
netsh winsock reset
netsh int ip reset

방법 2: DNS 서버 변경

1 DNS 설정 변경:
  • 설정 → 네트워크 및 인터넷 → 이더넷 (또는 Wi-Fi)
  • 연결된 네트워크 클릭
  • "DNS 서버 할당" → "수동" 선택
  • IPv4 DNS 서버 설정:
기본 DNS: 8.8.8.8
보조 DNS: 8.8.4.4
또는
기본 DNS: 1.1.1.1
보조 DNS: 1.0.0.1
2 DNS 캐시 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 입력:
ipconfig /flushdns

방법 3: Windows Update 서비스 재시작

1 업데이트 서비스 중지:
  • Windows + X 키를 누르고 "Windows PowerShell(관리자)" 선택
  • 다음 명령어들을 순서대로 입력:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop dosvc
2 임시 파일 삭제:
  • 다음 명령어로 업데이트 캐시 삭제:
del /f /s /q %windir%\SoftwareDistribution\Download\*.*
ren %windir%\SoftwareDistribution SoftwareDistribution.old
3 서비스 재시작:
  • 다음 명령어들을 순서대로 입력:
net start wuauserv
net start cryptSvc
net start bits
net start dosvc

방법 4: 방화벽 및 보안 프로그램 확인

1 Windows Defender 방화벽 확인:
  • 설정 → 업데이트 및 보안 → Windows 보안
  • "방화벽 및 네트워크 보호" 클릭
  • "앱이 방화벽을 통과하도록 허용" 클릭
  • "Windows Update" 항목이 체크되어 있는지 확인
2 타사 보안 프로그램 확인:
  • 백신 프로그램의 방화벽 설정 확인
  • 임시로 실시간 보호 기능 비활성화 후 업데이트 시도
  • Windows Update 관련 프로세스를 예외 목록에 추가

방법 5: 프록시 설정 확인

1 프록시 설정 해제:
  • 설정 → 네트워크 및 인터넷 → 프록시
  • "자동으로 프록시 설정 검색" 끄기
  • "설정 스크립트 사용" 끄기
  • "수동 프록시 설정" 끄기
2 Internet Explorer 프록시 설정:
  • Internet Explorer 실행
  • 도구 → 인터넷 옵션 → 연결 탭
  • "LAN 설정" 클릭
  • 모든 프록시 설정 해제

방법 6: 시스템 날짜 및 시간 확인

1 날짜/시간 동기화:
  • 설정 → 시간 및 언어 → 날짜 및 시간
  • "시간 자동 설정" 활성화
  • "표준 시간대 자동 설정" 활성화
  • "지금 동기화" 클릭
2 수동 시간 동기화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 입력:
w32tm /resync
w32tm /config /manualpeerlist:"time.windows.com" /syncfromflags:manual

방법 7: 배달 최적화 재설정

1 배달 최적화 설정:
  • 설정 → 업데이트 및 보안 → 배달 최적화
  • "다른 PC에서 다운로드 허용" 끄기
  • "고급 옵션" 클릭
  • 다운로드 설정을 "무제한"으로 변경
2 배달 최적화 캐시 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 입력:
net stop dosvc
del /f /s /q %windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache\*.*
net start dosvc

방법 8: Microsoft Update 서버 직접 연결

1 업데이트 서버 연결 테스트:
  • 웹 브라우저에서 다음 URL 접속 테스트:
https://update.microsoft.com
https://windowsupdate.microsoft.com
https://download.windowsupdate.com
2 Windows Update 카탈로그 사용:
  • Microsoft Update 카탈로그 웹사이트 접속
  • 실패한 업데이트의 KB 번호 검색
  • 해당 업데이트 수동 다운로드 및 설치

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
네트워크 연결 확인 쉬움 10-15분 높음 예
DNS 서버 변경 쉬움 5-10분 매우 높음 아니오
Windows Update 서비스 재시작 보통 15-20분 높음 예
방화벽 설정 확인 보통 10-15분 보통 아니오
프록시 설정 해제 쉬움 5-10분 보통 아니오
시스템 시간 동기화 쉬움 5분 보통 아니오
배달 최적화 재설정 보통 10-15분 높음 예
수동 업데이트 다운로드 보통 20-40분 매우 높음 예

🔧 주요 명령어 참조

네트워크 관련 명령어

  • ipconfig /flushdns: DNS 캐시 삭제
  • netsh winsock reset: Winsock 재설정
  • netsh int ip reset: IP 설정 재설정
  • ping 8.8.8.8: 네트워크 연결 테스트
  • nslookup update.microsoft.com: DNS 조회 테스트

Windows Update 관련 명령어

  • net stop wuauserv: Windows Update 서비스 중지
  • net start wuauserv: Windows Update 서비스 시작
  • net stop dosvc: 배달 최적화 서비스 중지
  • w32tm /resync: 시간 동기화

⚠️ 주의사항

네트워크 설정 변경 시 주의점

  • DNS 서버 변경 전 기존 설정을 기록해두세요
  • 회사 네트워크에서는 IT 관리자와 상의하세요
  • 방화벽 설정 변경 시 보안에 주의하세요
  • 프록시 설정은 네트워크 환경에 따라 필요할 수 있습니다
  • 업데이트 중에는 네트워크 연결을 끊지 마세요

언제 전문가의 도움이 필요한가?

  • 기업 네트워크 환경에서 문제가 발생한 경우
  • 네트워크 하드웨어 문제가 의심되는 경우
  • ISP(인터넷 서비스 제공업체) 문제가 의심되는 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 안정적인 인터넷 연결 환경 유지
  • 정기적인 네트워크 설정 점검
  • 방화벽 및 보안 프로그램 설정 최적화
  • 시스템 시간 자동 동기화 설정 유지
  • 프록시 설정이 필요한 경우에만 사용
  • 네트워크 드라이버 최신 상태 유지

💡 추가 팁

  • 업데이트 다운로드 시 트래픽이 많은 시간대를 피하세요
  • VPN 사용 시 업데이트 전에 연결을 해제하세요
  • 모바일 핫스팟을 이용한 임시 해결 방법을 고려하세요
  • 네트워크 속도가 느린 경우 배달 최적화 설정을 조정하세요
  • 업데이트 실패 시 Windows Update 로그를 확인하세요

🔍 관련 오류 코드

유사한 네트워크 관련 Windows Update 오류들

  • 0x80072EE2: 네트워크 연결 시간 초과
  • 0x80072EFE: 네트워크 연결 실패
  • 0x80072F8F: 인증서 오류
  • 0x80070422: Windows Update 서비스 중지
  • 0x8024402C: 프록시 서버 문제
  • 0x80244019: 업데이트 서버 연결 실패

🌐 네트워크 진단 도구

Windows 내장 네트워크 진단 도구

  • 네트워크 문제 해결사: 설정 → 네트워크 및 인터넷 → 상태 → 네트워크 문제 해결사
  • 네트워크 재설정: 설정 → 네트워크 및 인터넷 → 상태 → 네트워크 재설정
  • Windows Update 문제 해결사: 설정 → 업데이트 및 보안 → 문제 해결

이 가이드를 통해 Windows Update 다운로드 문제가 해결되기를 바랍니다. 네트워크 관련 문제가 지속되면 ISP나 네트워크 관리자에게 문의하시기 바랍니다.
