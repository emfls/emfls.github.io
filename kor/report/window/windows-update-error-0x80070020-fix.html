Windows 0x80070020 InstallFileLocked 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 설치나 업데이트 과정에서 필요한 파일이 다른 프로세스에 의해 잠겨있을 때 발생하는 문제입니다. 파일 잠금, 프로세스 충돌, 또는 권한 문제가 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80070020 (InstallFileLocked)는 Windows 설치나 업데이트 과정에서 필요한 파일이 다른 프로세스에 의해 사용 중이거나 잠겨있을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 다른 프로세스가 설치 파일을 사용 중
  • 바이러스 백신 소프트웨어에 의한 파일 잠금
  • Windows 서비스가 파일을 사용 중
  • 백그라운드에서 실행 중인 프로그램의 간섭
  • 파일 시스템 권한 문제
  • 손상된 파일 핸들
  • 시스템 파일 보호 기능에 의한 차단
  • Windows Update 서비스 충돌
  • TrustedInstaller 프로세스에 의한 파일 잠금
  • 디스크 I/O 오류로 인한 파일 접근 실패

🛠️ 해결 방법

방법 1: 시스템 재부팅 및 안전 모드 사용

1 시스템 재부팅:
  • 모든 프로그램을 종료
  • 시스템을 완전히 재부팅
  • 재부팅 후 즉시 설치/업데이트 시도
  • 다른 프로그램 실행 전에 작업 수행
2 안전 모드에서 설치 시도:
  • 시작 메뉴 → 전원 → Shift 키를 누른 상태에서 "다시 시작" 클릭
  • 문제 해결 → 고급 옵션 → 시작 설정
  • "다시 시작" 클릭 후 F4 키 눌러 안전 모드 진입
  • 또는 명령어 사용:
bcdedit /set {default} safeboot minimal
shutdown /r /t 0
  • 안전 모드에서 설치/업데이트 시도
  • 완료 후 정상 모드로 복귀:
bcdedit /deletevalue {default} safeboot
shutdown /r /t 0

방법 2: 파일 잠금 프로세스 확인 및 종료

1 파일을 사용 중인 프로세스 확인:
  • 작업 관리자 (Ctrl + Shift + Esc) 실행
  • 프로세스 탭에서 높은 디스크 사용량을 보이는 프로세스 확인
  • 관리자 권한으로 명령 프롬프트에서 확인:
tasklist /svc
wmic process get ProcessId,Name,CommandLine
handle.exe [파일경로] (Sysinternals 도구 사용 시)
openfiles /query
2 잠금 프로세스 안전하게 종료:
  • Windows Update 관련 프로세스 종료:
taskkill /f /im wuauclt.exe
taskkill /f /im UsoClient.exe
taskkill /f /im usocoreworker.exe
taskkill /f /im WaaSMedicAgent.exe
taskkill /f /im svchost.exe /fi "SERVICES eq wuauserv"
  • ⚠️ 주의: TrustedInstaller.exe는 강제 종료하지 마세요

방법 3: Windows Update 서비스 중지 및 재시작

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 잠긴 파일 및 캐시 삭제:
  • 서비스 중지 상태에서 다음 명령어 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 4: 보안 소프트웨어 및 실시간 보호 비활성화

1 Windows Defender 실시간 보호 임시 비활성화:
  • Windows 보안 → 바이러스 및 위협 방지
  • "바이러스 및 위협 방지 설정" → "설정 관리"
  • "실시간 보호" 임시 비활성화
  • 또는 명령어 사용:
powershell -Command "Set-MpPreference -DisableRealtimeMonitoring $true"
sc config WinDefend start= disabled
net stop WinDefend
2 타사 보안 소프트웨어 설정:
  • 설치된 바이러스 백신 소프트웨어 임시 비활성화
  • 실시간 스캔 기능 비활성화
  • 파일 보호 기능 임시 해제
  • 설치/업데이트 완료 후 보안 기능 재활성화

방법 5: 파일 권한 및 소유권 수정

1 관리자 권한으로 파일 소유권 변경:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 소유권 변경:
takeown /f C:\Windows\System32 /r /d y
takeown /f C:\Windows\SoftwareDistribution /r /d y
icacls C:\Windows\System32 /grant administrators:F /t
icacls C:\Windows\SoftwareDistribution /grant administrators:F /t
2 PowerShell을 통한 권한 설정:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
$acl = Get-Acl "C:\Windows\System32"
$accessRule = New-Object System.Security.AccessControl.FileSystemAccessRule("Everyone","FullControl","Allow")
$acl.SetAccessRule($accessRule)
Set-Acl "C:\Windows\System32" $acl

방법 6: 디스크 검사 및 복구

1 디스크 오류 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
chkdsk C: /f /r /x
sfc /scannow
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth
2 파일 시스템 복구:
  • 시스템 재부팅 시 디스크 검사 실행
  • 손상된 파일 자동 복구
  • 복구 완료 후 설치/업데이트 재시도

방법 7: 클린 부팅 상태에서 설치

1 클린 부팅 설정:
  • Windows + R 키를 누르고 msconfig 입력
  • "일반" 탭에서 "선택적 시작" 선택
  • "시스템 서비스 로드" 체크
  • "시작 항목 로드" 체크 해제
  • "서비스" 탭에서 "Microsoft 서비스 모두 숨기기" 체크
  • "모두 사용 안 함" 클릭
  • 시스템 재부팅
2 클린 부팅 상태에서 설치:
  • 최소한의 서비스만 실행되는 상태에서 설치/업데이트 시도
  • 설치 완료 후 msconfig에서 "일반 시작" 선택
  • 시스템 재부팅하여 정상 상태로 복귀

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 사용:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Reset-WUComponents -Verbose
2 파일 잠금 해제 및 강제 설치:
  • 다음 명령어들을 실행:
Get-Process | Where-Object {$_.ProcessName -like "*wuau*" -or $_.ProcessName -like "*uso*"} | Stop-Process -Force
Remove-Item "C:\Windows\SoftwareDistribution" -Recurse -Force -ErrorAction SilentlyContinue
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -Verbose

🔧 주요 명령어 참조

프로세스 관리 관련 명령어

  • tasklist /svc: 실행 중인 서비스 및 프로세스 확인
  • taskkill /f /im [프로세스명]: 프로세스 강제 종료

  • wmic process get ProcessId,Name: 프로세스 정보 확인

서비스 관리 관련 명령어

  • net stop wuauserv: Windows Update 서비스 중지
  • net start wuauserv: Windows Update 서비스 시작
  • sc config [서비스명] start= disabled: 서비스 비활성화
  • services.msc: 서비스 관리 콘솔 실행

파일 권한 관련 명령어

  • takeown /f [경로] /r /d y: 파일 소유권 변경
  • icacls [경로] /grant administrators:F: 관리자 권한 부여
  • attrib -r -s -h [파일명]: 파일 속성 변경
  • cacls [경로] /e /g everyone:f: 모든 사용자 권한 부여

  • 파일 시스템 손상이 심각한 경우

🛠️ 고급 해결 방법

파일 잠금 상태 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 파일 잠금 상태 분석:
$LockedFiles = @()
$Processes = Get-Process
foreach ($Process in $Processes) {
    try {
        $Files = $Process.Modules | Where-Object {$_.FileName -like "*Windows*"}
        if ($Files) {
            $LockedFiles += [PSCustomObject]@{
                ProcessName = $Process.ProcessName
                ProcessId = $Process.Id
                FileName = $Files.FileName
            }
        }
    } catch { }
}
$LockedFiles | Format-Table -AutoSize

자동 파일 잠금 해제 스크립트

  • 파일 잠금 자동 해제:
Write-Host "Windows Update 관련 프로세스 종료 중..." -ForegroundColor Yellow
$WUProcesses = @("wuauclt", "UsoClient", "usocoreworker", "WaaSMedicAgent")
foreach ($ProcessName in $WUProcesses) {
    $Processes = Get-Process -Name $ProcessName -ErrorAction SilentlyContinue
    if ($Processes) {
        $Processes | Stop-Process -Force
        Write-Host "$ProcessName 프로세스 종료됨" -ForegroundColor Green
    }
}
Write-Host "파일 잠금 해제 완료" -ForegroundColor Green

🔄 파일 잠금 해제 도구

Windows 내장 도구

  • 작업 관리자: 프로세스 확인 및 종료
  • 리소스 모니터: 파일 핸들 확인
  • 서비스 관리자: 서비스 중지 및 시작
  • 이벤트 뷰어: 파일 잠금 관련 로그 확인

명령줄 도구

  • tasklist: 실행 중인 프로세스 목록
  • taskkill: 프로세스 강제 종료
  • openfiles: 열린 파일 목록 확인
  • handle.exe: Sysinternals 파일 핸들 도구

PowerShell 명령어

  • Get-Process: 프로세스 정보 확인
  • Stop-Process: 프로세스 종료
  • Get-Service: 서비스 상태 확인
  • Stop-Service: 서비스 중지
