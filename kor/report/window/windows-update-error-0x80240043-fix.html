Windows 0x80240043 WU_E_NO_UI_SUPPORT 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 사용자 인터페이스(UI) 지원이 없을 때 발생하는 문제입니다. 서버 환경, 최소 설치 모드, 또는 UI 구성 요소 손상이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240043 (WU_E_NO_UI_SUPPORT)는 Windows Update 시스템에서 사용자 인터페이스 지원이 없을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Server Core 환경에서의 업데이트 시도
  • Windows PE (Preinstallation Environment) 환경
  • 최소 설치 모드에서의 Windows Update 실행
  • 손상된 Windows Update UI 구성 요소
  • GUI 없는 서버 환경에서의 업데이트
  • 원격 서버 관리 환경에서의 문제
  • Windows Update 클라이언트 UI 모듈 손상
  • 시스템 파일 손상으로 인한 UI 지원 실패
  • 터미널 서비스 환경에서의 UI 제한
  • 자동화된 스크립트 환경에서의 UI 요구

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 UI 구성 요소 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 명령줄을 통한 Windows Update 실행

1 Windows Update 클라이언트 명령줄 도구 사용:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 업데이트 확인 및 설치:
wuauclt /detectnow
wuauclt /updatenow
usoclient ScanInstallWait
usoclient StartDownload
usoclient StartInstall
2 UsoClient를 이용한 업데이트:
  • 다음 명령어들을 순서대로 실행:
usoclient RefreshSettings
usoclient StartScan
usoclient ScanInstallWait
usoclient RestartDevice

방법 3: PowerShell을 이용한 Windows Update

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 PowerShell을 통한 업데이트 설치:
  • 다음 명령어들을 실행:
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -AutoReboot -Verbose
Get-WUHistory
Get-WURebootStatus

방법 5: Windows Update 서비스 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:

  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution

del /f /s /q %windir%\Logs\WindowsUpdate\*

3 서비스 재시작 및 UI 지원 활성화:

방법 6: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
DISM /Online /Get-Features | findstr "GUI"
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wucltui.dll
sfc /scanfile=C:\Windows\System32\wucltux.dll
sfc /scanfile=C:\Windows\System32\wuapp.exe
  • 스캔 완료 후 시스템 재부팅

방법 7: Windows 기능 및 GUI 구성 요소 확인

1 Windows 에디션 및 GUI 지원 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 시스템 정보 확인:
dism /online /get-currentedition
dism /online /get-targeteditions
systeminfo | findstr /C:"OS Name" /C:"OS Version"
wmic os get Caption,InstallationType,ProductType
2 GUI 기능 설치 (Server Core인 경우):
  • Server Core 환경에서 GUI 기능 추가:
dism /online /enable-feature /featurename:Server-Gui-Mgmt-Infra /all
dism /online /enable-feature /featurename:Server-Gui-Shell /all
dism /online /enable-feature /featurename:ServerCore-FullServer /all
shutdown /r /t 0

방법 8: 수동 업데이트 다운로드 및 설치

1 Microsoft Update Catalog 사용:
  • 다른 컴퓨터에서 Microsoft Update Catalog 접속
  • 시스템에 필요한 업데이트 검색 및 다운로드
  • USB 또는 네트워크를 통해 파일 전송
  • 대상 시스템에서 수동 설치
2 명령줄을 통한 수동 설치:
  • 다운로드한 업데이트 파일을 다음 명령어로 설치:
wusa.exe [업데이트파일명].msu /quiet /norestart
dism /online /add-package /packagepath:[업데이트파일명].msu
pkgmgr /ip /m:[업데이트파일명].xml
shutdown /r /t 60

  • GUI 기능 설치 후 시스템 성능에 영향을 줄 수 있습니다

  • 하드웨어 호환성 문제로 인한 업데이트 실패
  • 시스템 전체 재설치가 필요한 경우

🔍 관련 오류 코드

유사한 Windows Update UI 관련 오류들

  • 0x80240020: 대화형 사용자 없음 (WU_E_NO_INTERACTIVE_USER)
  • 0x8024001F: 연결 없음 (WU_E_NO_CONNECTION)
  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x80240044: UI 모드 지원 없음 (WU_E_NO_UI_MODE_SUPPORT)
  • 0x80240045: 사용자 인터페이스 없음 (WU_E_NO_USER_INTERFACE)
  • 0x80240046: 대화형 모드 없음 (WU_E_NO_INTERACTIVE_MODE)
  • 0x80070005: 액세스 거부됨

🛠️ 고급 해결 방법

PowerShell 자동화 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 자동 업데이트 스크립트 생성:
# Windows Update 자동화 스크립트
Import-Module PSWindowsUpdate
Write-Host "Windows Update 검색 중..." -ForegroundColor Green
$Updates = Get-WindowsUpdate -Verbose
if ($Updates) {
    Write-Host "발견된 업데이트: $($Updates.Count)개" -ForegroundColor Yellow
    Install-WindowsUpdate -AcceptAll -AutoReboot -Verbose
} else {
    Write-Host "설치할 업데이트가 없습니다." -ForegroundColor Green
}
Get-WUHistory | Select-Object -First 5

UI 구성 요소 상태 확인 스크립트

  • UI 지원 상태 확인:
# UI 구성 요소 상태 확인
$OSInfo = Get-WmiObject -Class Win32_OperatingSystem
Write-Host "OS: $($OSInfo.Caption)"
Write-Host "설치 유형: $($OSInfo.InstallationType)"
$GUIFeatures = dism /online /get-features | findstr "GUI"
if ($GUIFeatures) {
    Write-Host "GUI 기능 상태:" -ForegroundColor Green
    $GUIFeatures
} else {
    Write-Host "GUI 기능이 설치되지 않았습니다." -ForegroundColor Red
}
