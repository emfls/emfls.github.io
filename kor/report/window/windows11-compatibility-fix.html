Windows 11 호환성 검사 통과 못했다면 이렇게 하세요

⚠️ 중요: Windows 11 호환성 검사 실패는 하드웨어 또는 설정 문제로 발생합니다. 이 가이드를 통해 단계별로 문제를 해결하고 Windows 11 설치 요구사항을 충족시킬 수 있습니다.

🔍 Windows 11 시스템 요구사항 확인

요구사항 최소 사양 권장 사양 확인 방법
프로세서 1GHz 이상, 2코어 이상 64비트 Intel 8세대 이상 / AMD Ryzen 2000 이상 작업 관리자 → 성능 탭
메모리 4GB RAM 8GB RAM 이상 시스템 정보 (msinfo32)

보안 부팅 UEFI 펌웨어 보안 부팅 지원 BIOS/UEFI 설정
그래픽 DirectX 12 호환 WDDM 2.0 드라이버 dxdiag 실행

🚀 단계별 호환성 문제 해결

1단계

PC 상태 검사 도구로 정확한 문제 파악

💡 먼저 해야 할 일: Microsoft 공식 PC 상태 검사 도구를 사용하여 정확히 어떤 요구사항이 충족되지 않는지 확인하세요.

PC 상태 검사 도구 사용법:

  1. Microsoft 공식 웹사이트에서 "PC 상태 검사" 앱 다운로드
  2. 앱 설치 후 실행
  3. "지금 확인" 버튼 클릭
  4. 실패한 항목들을 메모장에 기록

✅ 확인사항




# PowerShell로 시스템 정보 확인
Get-ComputerInfo | Select-Object WindowsProductName, TotalPhysicalMemory, CsProcessors
Get-Tpm
Get-SecureBootUEFI -Name SetupMode
2단계

TPM 2.0 활성화 및 설정

🔒 가장 흔한 문제: TPM 2.0이 비활성화되어 있거나 지원하지 않는 경우가 Windows 11 호환성 검사 실패의 주요 원인입니다.

TPM 상태 확인:

  1. Windows + R → tpm.msc 입력
  2. TPM 관리 콘솔에서 상태 확인
  3. "TPM을 사용할 준비가 되었습니다" 메시지 확인

TPM 2.0 활성화 방법:

메인보드 제조사 BIOS 진입 키 TPM 설정 위치 설정 이름
ASUS F2 또는 Del Advanced → PCH-FW Configuration TPM Device Selection
MSI Del Settings → Security → Trusted Computing Security Device Support
Gigabyte Del Settings → Miscellaneous Intel Platform Trust Technology
ASRock F2 또는 Del Security → Intel Platform Trust Technology Intel PTT

✅ TPM 설정 체크리스트





3단계

보안 부팅 (Secure Boot) 활성화

📋 보안 부팅이란: UEFI 펌웨어의 보안 기능으로, 부팅 시 디지털 서명된 운영체제만 실행할 수 있도록 하는 기술입니다.

보안 부팅 활성화 전 준비사항:

  1. UEFI 모드 확인: 레거시 BIOS가 아닌 UEFI 모드여야 함
  2. 디스크 파티션: GPT 파티션 테이블 사용
  3. CSM 비활성화: Compatibility Support Module 끄기

보안 부팅 설정 방법:

  1. BIOS/UEFI 설정 진입
  2. Boot 또는 Security 메뉴 찾기
  3. Secure Boot 옵션을 "Enabled"로 설정
  4. OS Type을 "Windows UEFI mode"로 설정
  5. CSM (Compatibility Support Module) 비활성화

⚠️ 주의사항

  • 보안 부팅 활성화 전 중요한 데이터 백업
  • 레거시 부팅 모드에서 설치된 OS는 부팅 불가능
  • 일부 오래된 하드웨어나 소프트웨어 호환성 문제 발생 가능
# PowerShell로 보안 부팅 상태 확인
Confirm-SecureBootUEFI
Get-SecureBootUEFI -Name SetupMode
Get-SecureBootPolicy

✅ 보안 부팅 체크리스트





4단계

CPU 호환성 문제 해결

🚫 지원되지 않는 CPU: Windows 11은 Intel 8세대(2017년) 이후, AMD Ryzen 2000 시리즈(2018년) 이후 CPU만 공식 지원합니다.

CPU 호환성 확인:

  1. 작업 관리자 → 성능 탭 → CPU에서 모델명 확인
  2. Microsoft 공식 지원 CPU 목록과 비교
  3. 제조사별 지원 여부 확인

지원되지 않는 CPU 해결 방법:

해결 방법 난이도 위험도 설명
레지스트리 수정 중간 중간 CPU 검사 우회 (비공식)
ISO 파일 수정 높음 중간 설치 파일 직접 수정
CPU 업그레이드 높음 낮음 하드웨어 교체 (권장)
Windows 10 유지 낮음 낮음 2025년까지 지원
# 레지스트리 수정으로 CPU 검사 우회 (비공식)
# 주의: 이 방법은 Microsoft에서 공식 지원하지 않습니다
reg add "HKLM\SYSTEM\Setup\LabConfig" /v "BypassCPUCheck" /t REG_DWORD /d 1 /f
reg add "HKLM\SYSTEM\Setup\LabConfig" /v "BypassTPMCheck" /t REG_DWORD /d 1 /f
reg add "HKLM\SYSTEM\Setup\LabConfig" /v "BypassSecureBootCheck" /t REG_DWORD /d 1 /f

✅ CPU 호환성 체크리스트




5단계

메모리 및 저장소 업그레이드

메모리 요구사항:

  • 최소: 4GB RAM (64비트)
  • 권장: 8GB RAM 이상
  • 최적: 16GB RAM (멀티태스킹 환경)

저장소 요구사항:

  • 최소: 64GB 사용 가능 공간
  • 권장: 256GB SSD
  • 최적: 512GB NVMe SSD
# PowerShell로 메모리 및 저장소 정보 확인
Get-WmiObject -Class Win32_PhysicalMemory | Measure-Object -Property Capacity -Sum
Get-WmiObject -Class Win32_LogicalDisk | Select-Object DeviceID, Size, FreeSpace

업그레이드 가이드:

구성 요소 확인 방법 업그레이드 방법 예상 비용
RAM 작업 관리자 → 성능 → 메모리 메모리 슬롯에 추가 설치 5-15만원
저장소 디스크 관리 또는 설정 → 저장소 SSD 교체 또는 추가 10-30만원
그래픽 dxdiag 실행 그래픽 카드 업그레이드 20-100만원

🔧 문제별 상세 해결 방법

TPM 관련 문제

문제 원인 해결 방법
TPM을 찾을 수 없음 하드웨어 미지원 메인보드 교체 또는 우회 방법 사용
TPM 1.2 버전 구형 TPM 칩 BIOS 업데이트 또는 하드웨어 교체
TPM 비활성화 상태 BIOS 설정 문제 BIOS에서 TPM 또는 Intel PTT 활성화
TPM 초기화 필요 이전 설정 충돌 TPM 관리 콘솔에서 초기화

보안 부팅 관련 문제

문제 원인 해결 방법
보안 부팅 옵션 없음 레거시 BIOS UEFI 지원 메인보드로 교체
보안 부팅 활성화 불가 CSM 활성화 상태 CSM 비활성화 후 재시도
부팅 실패 MBR 파티션 사용 GPT로 변환 또는 재설치
키 관리 오류 보안 키 손상 기본 키로 복원

⚠️ 주의사항 및 대안

비공식 우회 방법 사용 시 주의사항

  • 보안 위험: 시스템 보안 기능이 비활성화됨
  • 업데이트 문제: 향후 Windows Update에서 문제 발생 가능
  • 지원 제외: Microsoft 공식 지원 받을 수 없음
  • 성능 저하: 최적화되지 않은 환경에서 실행
  • 호환성 문제: 일부 앱이나 기능 사용 불가

Windows 10 계속 사용하기

  • 지원 기간: 2025년 10월 14일까지 보안 업데이트 제공
  • 안정성: 검증된 운영체제로 안정적 사용 가능
  • 호환성: 대부분의 소프트웨어와 하드웨어 지원
  • 성능: 구형 하드웨어에서 더 나은 성능
