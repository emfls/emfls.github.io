Windows 0x80240023 WU_E_EULAS_DECLINED 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 사용권 계약(EULA)이 거부되었을 때 발생하는 문제입니다. 사용권 계약 수락 문제, 자동화된 설치 환경, 또는 정책 설정 문제가 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240023 (WU_E_EULAS_DECLINED)는 Windows Update 시스템에서 하나 이상의 업데이트에 대한 사용권 계약(End User License Agreement)이 거부되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 사용자가 업데이트 설치 시 EULA를 거부한 경우
  • 자동 업데이트 환경에서 EULA 수락 설정 누락
  • 그룹 정책에 의한 EULA 자동 수락 비활성화
  • 손상된 업데이트 메타데이터로 인한 EULA 처리 오류
  • 서버 환경에서 대화형 세션 없이 업데이트 시도
  • 타사 업데이트 관리 도구와의 충돌
  • Windows Update 서비스 권한 설정 문제
  • 레지스트리 설정 오류로 인한 EULA 처리 실패
  • 업데이트 캐시 손상으로 인한 EULA 정보 오류
  • WSUS 환경에서 EULA 승인 정책 미설정

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 EULA 관련 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: EULA 자동 수락 설정

1 그룹 정책을 통한 EULA 자동 수락 설정 (Pro/Enterprise):
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • "업데이트를 설치하기 전에 최종 사용자 사용권 계약에 자동으로 동의" 정책 찾기
  • 정책을 "사용"으로 설정
  • "확인" 클릭하여 저장
2 레지스트리를 통한 EULA 자동 수락 설정 (Home 버전):
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v AcceptTrustedPublisherCerts /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v EULAAcceptRequired /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AcceptTrustedPublisherCerts /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v EULAAcceptRequired /t REG_DWORD /d 0 /f
3 그룹 정책 업데이트 적용:
  • 다음 명령어로 정책 강제 적용:
gpupdate /force
gpupdate /target:computer /force
gpupdate /target:user /force

방법 3: 수동으로 EULA 수락 및 업데이트 설치

1 Windows Update 설정에서 수동 확인:
  • 설정 → 업데이트 및 보안 → Windows Update
  • "업데이트 확인" 클릭
  • 업데이트 목록에서 EULA가 필요한 업데이트 확인
  • 각 업데이트를 개별적으로 선택하여 EULA 수락
2 PowerShell을 통한 EULA 수락:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-WindowsUpdate -AcceptAll -Install -AutoReboot
Install-WindowsUpdate -AcceptAll -AutoReboot

방법 4: Windows Update 캐시 및 메타데이터 재구축

1 Windows Update 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
2 EULA 관련 캐시 및 메타데이터 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작 및 메타데이터 재생성:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 5: Microsoft Update Catalog을 통한 수동 설치

1 Microsoft Update Catalog 접속:
  • 웹 브라우저에서 Microsoft Update Catalog 사이트 접속
  • 실패한 업데이트의 KB 번호 검색
  • 시스템에 맞는 정확한 버전 선택
  • 업데이트 파일 다운로드
2 수동 업데이트 설치:
  • 다운로드한 .msu 파일을 더블클릭하여 실행
  • EULA 화면이 나타나면 "동의함" 클릭
  • 또는 명령어로 설치:
wusa.exe [업데이트파일명].msu /quiet /norestart
dism /online /add-package /packagepath:[업데이트파일명].msu /quiet /norestart

방법 6: WSUS 환경에서 EULA 승인 설정

1 WSUS 관리 콘솔에서 EULA 승인:
  • WSUS 관리 콘솔 실행
  • 업데이트 → 모든 업데이트
  • 승인되지 않은 업데이트 중 EULA가 필요한 항목 찾기
  • 업데이트 우클릭 → "승인"
  • 대상 그룹 선택 후 "설치용으로 승인"
  • EULA 수락 확인
2 WSUS 클라이언트 설정 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WSUS 설정 확인:
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUStatusServer
wuauclt /detectnow /reportnow

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 EULA 자동 수락 및 업데이트 강제 설치:
  • 다음 명령어들을 실행:
Set-WUSettings -AcceptTrustedPublisherCerts
Get-WUServiceManager
Get-WindowsUpdate -AcceptAll -Verbose
Install-WindowsUpdate -AcceptAll -AutoReboot -ForceInstall -Verbose
Get-WindowsUpdateLog -LogPath C:\WindowsUpdate.log
Reset-WUComponents -Verbose

  • 중요한 데이터는 백업 후 작업하세요
  • EULA 자동 수락 설정은 보안 정책을 고려하세요
  • 기업 환경에서는 IT 관리자와 상의 후 설정하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • WSUS 환경에서는 서버 관리자와 협의하세요
  • 수동 업데이트 설치 시 정확한 버전을 확인하세요
  • 그룹 정책 변경 시 다른 시스템에 미치는 영향을 고려하세요

언제 전문가의 도움이 필요한가?

  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 특정 업데이트의 EULA가 지속적으로 처리되지 않는 경우
  • 시스템 전체 재설치가 필요한 경우

🔍 관련 오류 코드

유사한 Windows Update EULA 관련 오류들

  • 0x80240022: 모든 업데이트 실패 (WU_E_ALL_UPDATES_FAILED)
  • 0x80240021: 시간 초과 (WU_E_TIME_OUT)
  • 0x80240020: 대화형 사용자 없음 (WU_E_NO_INTERACTIVE_USER)
  • 0x80240024: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)
  • 0x80240025: 업데이트 설치 관리자 없음 (WU_E_NO_UPDATE_INSTALLER)
  • 0x80070005: 액세스 거부됨
  • 0x80070057: 매개 변수가 올바르지 않음

🛠️ 고급 해결 방법

EULA 상태 확인 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 EULA 설정 상태 확인:
$EULASettings = @(
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate",
    "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update"
)
foreach ($Setting in $EULASettings) {
    if (Test-Path $Setting) {
        Write-Host "EULA 설정 확인: $Setting" -ForegroundColor Green
        $AcceptCerts = Get-ItemProperty $Setting -Name "AcceptTrustedPublisherCerts" -ErrorAction SilentlyContinue
        $EULARequired = Get-ItemProperty $Setting -Name "EULAAcceptRequired" -ErrorAction SilentlyContinue
        Write-Host "신뢰할 수 있는 게시자 인증서 수락: $($AcceptCerts.AcceptTrustedPublisherCerts)"
        Write-Host "EULA 수락 필요: $($EULARequired.EULAAcceptRequired)"
    }
}

자동 EULA 수락 설정 스크립트

  • EULA 자동 수락 설정 자동화:
Write-Host "EULA 자동 수락 설정 시작..." -ForegroundColor Green
$PolicyPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate"
$AutoUpdatePath = "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update"
if (!(Test-Path $PolicyPath)) { New-Item -Path $PolicyPath -Force }
Set-ItemProperty -Path $PolicyPath -Name "AcceptTrustedPublisherCerts" -Value 1 -Type DWord
Set-ItemProperty -Path $PolicyPath -Name "EULAAcceptRequired" -Value 0 -Type DWord
Set-ItemProperty -Path $AutoUpdatePath -Name "AcceptTrustedPublisherCerts" -Value 1 -Type DWord
Set-ItemProperty -Path $AutoUpdatePath -Name "EULAAcceptRequired" -Value 0 -Type DWord
gpupdate /force
Write-Host "EULA 자동 수락 설정이 완료되었습니다." -ForegroundColor Green

🔄 EULA 처리 프로세스 이해

Windows Update EULA 처리 단계

  1. 업데이트 검색: Windows Update가 사용 가능한 업데이트 검색
  2. EULA 확인: 각 업데이트에 대한 EULA 요구사항 확인
  3. 사용자 동의: EULA가 필요한 경우 사용자 동의 요청
  4. 자동 처리: 정책 설정에 따른 자동 EULA 수락
  5. 다운로드: EULA 수락 후 업데이트 다운로드
  6. 설치: 업데이트 설치 진행

EULA 관련 주요 구성 요소

  • AcceptTrustedPublisherCerts: 신뢰할 수 있는 게시자 인증서 자동 수락
  • EULAAcceptRequired: EULA 수락 요구 여부
  • 그룹 정책: 도메인 또는 로컬 정책을 통한 EULA 처리
  • WSUS 승인: 기업 환경에서 중앙 집중식 EULA 승인
