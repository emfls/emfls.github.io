Windows 11 0xC1900107 CLEANUP_PENDING 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 업그레이드나 업데이트 과정에서 이전 설치의 정리 작업이 완료되지 않았을 때 발생하는 오류입니다. 미완료된 정리 작업을 완료하거나 초기화해야 합니다.

🔍 오류 개요

0xC1900107 CLEANUP_PENDING는 Windows 11 업그레이드나 주요 업데이트 과정에서 이전 설치의 정리 작업이 아직 대기 중이거나 완료되지 않았을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 이전 Windows 업그레이드의 미완료된 정리 작업
  • Windows.old 폴더 정리 작업이 진행 중
  • 임시 설치 파일들의 정리 작업 대기
  • 시스템 재시작 후 완료되지 않은 정리 프로세스
  • 디스크 공간 부족으로 인한 정리 작업 중단
  • 백그라운드 정리 서비스의 오류
  • 업그레이드 프로세스의 비정상적 중단
  • 파일 시스템 오류로 인한 정리 작업 실패

🛠️ 해결 방법

방법 1: 시스템 재시작 및 대기

1 시스템 완전 재시작:
  • 모든 프로그램 종료
  • 시작 메뉴 → 전원 → 다시 시작
  • 재시작 완료 후 30분 이상 대기
  • 백그라운드 정리 작업이 자동으로 완료되는지 확인
2 정리 작업 진행 상황 확인:
  • 작업 관리자 실행 (Ctrl + Shift + Esc)
  • "프로세스" 탭에서 다음 프로세스들 확인:
    • cleanmgr.exe (디스크 정리)
    • TiWorker.exe (Trusted Installer)
    • SetupHost.exe (설치 호스트)
    • MoUsoCoreWorker.exe (업데이트 작업자)
  • 이러한 프로세스가 실행 중이라면 완료까지 대기

방법 2: 디스크 정리 도구 실행

1 시스템 파일 정리:
  • Windows + R 키를 누르고 cleanmgr 입력
  • 시스템 드라이브(C:) 선택
  • "시스템 파일 정리" 버튼 클릭
  • 다음 항목들을 체크하여 삭제:
    • 이전 Windows 설치 (Windows.old)
    • Windows 업그레이드 로그 파일
    • 임시 Windows 설치 파일
    • Windows Update 정리
    • 시스템 압축 파일
2 명령어를 통한 강제 정리:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
cleanmgr /sageset:1
cleanmgr /sagerun:1
DISM /Online /Cleanup-Image /StartComponentCleanup
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase

방법 3: Windows.old 폴더 수동 삭제

1 Windows.old 폴더 확인:
  • 파일 탐색기에서 C:\ 드라이브 열기
  • Windows.old 폴더 존재 여부 확인
  • 폴더 크기 확인 (우클릭 → 속성)
  • 숨겨진 파일 표시 활성화 (보기 → 숨겨진 항목)
2 폴더 소유권 변경 후 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
takeown /f C:\Windows.old /r /d Y
icacls C:\Windows.old /grant administrators:F /t
rd /s /q C:\Windows.old
  • 삭제 완료까지 기다리기 (시간이 오래 걸릴 수 있음)

방법 4: 임시 파일 및 설치 파일 정리

1 임시 파일 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
del /f /s /q %temp%\*.*
del /f /s /q %windir%\Temp\*.*
del /f /s /q %windir%\Prefetch\*.*
del /f /s /q %windir%\SoftwareDistribution\Download\*.*
2 Windows 업그레이드 관련 파일 정리:
  • 다음 경로의 파일들 삭제:
rd /s /q "C:\$Windows.~BT"
rd /s /q "C:\$Windows.~WS"
rd /s /q "C:\Windows10Upgrade"
del /f /q "C:\hiberfil.sys"
del /f /q "C:\pagefile.sys"
  • 오류가 발생하는 파일은 건너뛰기

방법 5: 저장소 센스를 통한 자동 정리

1 저장소 센스 활성화:
  • 설정 → 시스템 → 저장소
  • "저장소 센스" 활성화
  • "저장소 센스 구성" 클릭
  • "지금 정리" 버튼 클릭
2 고급 정리 옵션 설정:
  • "이전 버전의 Windows 삭제" 옵션 활성화
  • 임시 파일 정리 주기를 "1일"로 설정
  • 다운로드 폴더 정리를 "30일"로 설정
  • 휴지통 정리를 "30일"로 설정
  • "지금 정리" 실행하여 즉시 정리

방법 6: Windows Update 서비스 재설정

1 Windows Update 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop usosvc
2 업데이트 폴더 이름 변경:
  • 다음 명령어들을 실행:
ren C:\Windows\SoftwareDistribution SoftwareDistribution.old
ren C:\Windows\System32\catroot2 catroot2.old
3 서비스 재시작:
  • 다음 명령어들을 실행:
net start cryptSvc
net start bits
net start wuauserv
net start msiserver
net start usosvc

방법 7: 시스템 파일 검사 및 복구

1 SFC 스캔 실행:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
sfc /scannow
  • 스캔 완료까지 기다리기 (30-90분 소요)
  • 손상된 파일 복구 완료 후 재부팅
2 DISM 명령어 실행:
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth

방법 8: 안전 모드에서 정리 작업

1 안전 모드 진입:
  • 설정 → 업데이트 및 보안 → 복구
  • "고급 시작" 섹션에서 "지금 다시 시작" 클릭
  • "문제 해결" → "고급 옵션" → "시작 설정"
  • "다시 시작" 클릭 후 F4 키로 안전 모드 진입
2 안전 모드에서 정리 작업:
  • 안전 모드에서 관리자 권한으로 명령 프롬프트 실행
  • 앞서 언급한 정리 명령어들 실행
  • Windows.old 폴더 강제 삭제
  • 정상 모드로 재부팅 후 업그레이드 재시도

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
시스템 재시작 및 대기 쉬움 30-60분 보통 예
디스크 정리 도구 실행 쉬움 30-90분 높음 아니오
Windows.old 폴더 수동 삭제 보통 30-120분 매우 높음 아니오
임시 파일 정리 보통 20-60분 높음 아니오
저장소 센스 자동 정리 쉬움 30-90분 높음 아니오
Windows Update 서비스 재설정 보통 20-40분 보통 예
시스템 파일 검사 쉬움 60-120분 보통 예
안전 모드 정리 작업 어려움 60-180분 높음 예

🔧 주요 명령어 참조

정리 작업 명령어

  • cleanmgr /sageset:1: 디스크 정리 설정
  • cleanmgr /sagerun:1: 자동 디스크 정리 실행
  • DISM /Online /Cleanup-Image /StartComponentCleanup: 구성 요소 정리
  • rd /s /q C:\Windows.old: Windows.old 폴더 강제 삭제

파일 소유권 관련 명령어

  • takeown /f [경로] /r /d Y: 파일 소유권 가져오기
  • icacls [경로] /grant administrators:F /t: 관리자 권한 부여
  • del /f /s /q [경로]: 강제 파일 삭제
  • rd /s /q [경로]: 강제 폴더 삭제

⚠️ 주의사항

정리 작업 시 주의점

  • 중요한 데이터는 미리 백업하세요
  • Windows.old 폴더 삭제 시 이전 버전으로 롤백할 수 없습니다
  • 정리 작업 중에는 시스템을 강제 종료하지 마세요
  • 디스크 공간이 충분한지 확인 후 작업하세요
  • 시스템 파일 삭제 시 신중하게 진행하세요
  • 정리 작업 완료까지 충분한 시간을 확보하세요

언제 전문가의 도움이 필요한가?

  • 디스크 공간이 극도로 부족한 경우
  • 시스템 파일이 심각하게 손상된 경우
  • 정리 작업이 반복적으로 실패하는 경우
  • 중요한 업무용 시스템에서 문제가 발생한 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 디스크 정리 실행 (월 1회 이상)
  • 저장소 센스 자동 정리 기능 활성화
  • 충분한 디스크 공간 확보 (최소 20GB 여유 공간)
  • 업그레이드 전 불필요한 파일 미리 정리
  • 정기적인 시스템 재시작으로 정리 작업 완료
  • 업그레이드 중 시스템 강제 종료 금지
  • 안정적인 전원 공급 환경 유지

💡 추가 팁

  • 정리 작업 진행 상황을 작업 관리자에서 모니터링
  • 디스크 사용량 분석 도구로 공간 사용 현황 파악
  • 업그레이드 전 시스템 복원 지점 생성
  • 네트워크 연결이 안정적인 환경에서 작업
  • 정리 작업 완료 후 시스템 재시작 권장
  • 대용량 파일은 외부 저장소로 이동 고려

🔍 관련 오류 코드

유사한 Windows 업그레이드 정리 관련 오류들

  • 0xC1900101: 일반적인 설치 실패
  • 0xC1900200: 시스템이 최소 요구 사항을 충족하지 않음
  • 0xC1900208: 호환되지 않는 앱이 차단함
  • 0xC1900210: 디스크 공간이 부족함
  • 0x80070070: 디스크 공간 부족
  • 0x8007000E: 메모리 부족

🛠️ 고급 진단 도구

정리 작업 진단 도구

  • 디스크 정리: cleanmgr - 시스템 파일 정리
  • 저장소 설정: 설정 → 시스템 → 저장소
  • 작업 관리자: taskmgr - 정리 프로세스 모니터링
  • 이벤트 뷰어: eventvwr.msc - 정리 작업 로그

PowerShell 진단 명령어

  • Get-Process | Where-Object {$_.Name -like "*clean*"}: 정리 관련 프로세스
  • Get-ChildItem C:\ -Directory | Where-Object {$_.Name -like "*Windows*"}: Windows 관련 폴더
  • Get-WmiObject -Class Win32_LogicalDisk: 디스크 공간 확인
  • Get-EventLog -LogName System -EntryType Error: 시스템 오류 로그

📊 정리 작업 진행 상황 확인 방법

단계별 확인 방법

  1. 프로세스 확인: 작업 관리자에서 정리 관련 프로세스 실행 여부
  2. 디스크 사용량 확인: 저장소 설정에서 공간 사용 현황
  3. 폴더 존재 확인: Windows.old, $Windows.~BT 등 임시 폴더
  4. 서비스 상태 확인: Windows Update 관련 서비스 실행 상태
  5. 로그 확인: 이벤트 뷰어에서 정리 작업 관련 로그
  6. 재시작 후 재확인: 시스템 재시작 후 상태 변화 확인

🔄 자동화 스크립트

정리 작업 완료 스크립트

@echo off
echo Windows 정리 작업 완료 스크립트 시작...
echo.
echo 1단계: 현재 정리 프로세스 확인 중...
tasklist | findstr /i "cleanmgr TiWorker SetupHost"
echo.
echo 2단계: Windows Update 서비스 중지 중...
net stop wuauserv
net stop usosvc
echo.
echo 3단계: 임시 파일 정리 중...
del /f /s /q %temp%\*.* 2>nul
del /f /s /q %windir%\Temp\*.* 2>nul
del /f /s /q %windir%\SoftwareDistribution\Download\*.* 2>nul
echo.
echo 4단계: Windows.old 폴더 확인 및 삭제 중...
if exist "C:\Windows.old" (
echo Windows.old 폴더 발견. 삭제 중...
takeown /f C:\Windows.old /r /d Y
icacls C:\Windows.old /grant administrators:F /t
rd /s /q C:\Windows.old
echo Windows.old 폴더 삭제 완료
) else (
echo Windows.old 폴더가 없습니다
)
echo.
echo 5단계: 구성 요소 정리 중...
DISM /Online /Cleanup-Image /StartComponentCleanup
echo.
echo 6단계: 서비스 재시작 중...
net start wuauserv
net start usosvc
echo.
echo 정리 작업 완료! 시스템을 재시작하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 시스템 재시작
