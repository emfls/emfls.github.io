Windows 0x80240040 WU_E_NO_SERVER_CORE_SUPPORT 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Server Core 환경에서 Windows Update가 지원되지 않을 때 발생하는 문제입니다. Server Core 설치 유형, GUI 구성 요소 누락, 또는 서버 역할 제한이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240040 (WU_E_NO_SERVER_CORE_SUPPORT)는 Windows Server Core 환경에서 특정 Windows Update 기능이 지원되지 않을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Server Core 설치에서 GUI 기반 업데이트 시도
  • Server Core에서 지원되지 않는 업데이트 패키지
  • Desktop Experience 기능 누락
  • Windows Update GUI 구성 요소 부재
  • Server Core 전용 업데이트 메커니즘 미사용
  • PowerShell 또는 명령줄 도구 미사용
  • WSUS 서버 설정 문제 (Server Core 환경)
  • Server Manager 기능 제한
  • Windows Update Agent 호환성 문제
  • Server Core 역할 및 기능 제한

🛠️ 해결 방법

방법 1: Server Core 환경 확인

1 현재 Windows 설치 유형 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 설치 유형 확인:
Get-WindowsFeature -Name "Server-Gui-Shell"
Get-WindowsFeature -Name "Server-Gui-Mgmt-Infra"
dism /online /get-currentedition
Get-ComputerInfo | Select-Object WindowsInstallationType, WindowsProductName
2 Server Core 상태 진단:
  • 다음 명령어로 상세 정보 확인:
systeminfo | findstr /C:"OS Name" /C:"OS Version" /C:"System Type"
wmic os get Caption,InstallDate,OSArchitecture
Get-WindowsFeature | Where-Object {$_.InstallState -eq "Installed"} | Select-Object Name

방법 2: PowerShell을 이용한 Windows Update 관리

1 PSWindowsUpdate 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-PackageProvider -Name NuGet -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-Command -Module PSWindowsUpdate
2 PowerShell을 통한 업데이트 확인 및 설치:
  • 다음 명령어들을 실행:
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -AutoReboot
Get-WUHistory
Get-WUServiceManager

방법 3: 명령줄을 이용한 Windows Update

1 UsoClient를 이용한 업데이트:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
UsoClient ScanInstallWait
UsoClient StartScan
UsoClient StartDownload
UsoClient StartInstall
UsoClient RestartDevice
2 wuauclt를 이용한 업데이트:
  • 다음 명령어들을 실행:
wuauclt /detectnow
wuauclt /updatenow
wuauclt /resetauthorization /detectnow
wuauclt /reportnow

방법 4: WSUS 서버 구성 (기업 환경)

1 WSUS 클라이언트 설정:
  • PowerShell에서 다음 명령어 실행:
$WSUSServer = "http://wsus-server:8530"
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" -Name "WUServer" -Value $WSUSServer
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" -Name "WUStatusServer" -Value $WSUSServer
Set-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" -Name "UseWUServer" -Value 1 -Type DWord
2 WSUS 연결 테스트:
  • 다음 명령어로 WSUS 서버 연결 확인:
Test-NetConnection -ComputerName "wsus-server" -Port 8530
wuauclt /detectnow /reportnow
Get-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate"

방법 5: Desktop Experience 기능 추가

1 GUI 기능 설치 가능성 확인:
  • PowerShell에서 다음 명령어 실행:
Get-WindowsFeature -Name "Server-Gui-Shell"
Get-WindowsFeature -Name "Server-Gui-Mgmt-Infra"
dism /online /get-features | findstr "ServerCore"
2 Desktop Experience 설치 (가능한 경우):
  • 다음 명령어로 GUI 기능 설치:
Install-WindowsFeature Server-Gui-Mgmt-Infra,Server-Gui-Shell -Restart
# 또는 DISM 사용
dism /online /enable-feature /featurename:ServerCore-FullServer /all
  • ⚠️ 주의: 이 작업은 시스템 재부팅이 필요하며 상당한 시간이 소요됩니다

방법 6: Windows Update 서비스 재구성

1 Windows Update 서비스 상태 확인:
  • PowerShell에서 다음 명령어 실행:
Get-Service -Name "wuauserv", "cryptSvc", "bits", "msiserver" | Format-Table
sc query wuauserv
sc queryex wuauserv
2 서비스 재시작 및 구성:
  • 다음 명령어들을 순서대로 실행:
Stop-Service -Name "wuauserv", "cryptSvc", "bits" -Force
Remove-Item -Path "C:\Windows\SoftwareDistribution" -Recurse -Force
Remove-Item -Path "C:\Windows\System32\catroot2" -Recurse -Force
Start-Service -Name "cryptSvc", "bits", "wuauserv"
Set-Service -Name "wuauserv" -StartupType Automatic

방법 7: 수동 업데이트 다운로드 및 설치

1 Microsoft Update Catalog 사용:
  • 다른 컴퓨터에서 Microsoft Update Catalog 접속
  • Server Core용 업데이트 검색 및 다운로드
  • USB 또는 네트워크를 통해 Server Core로 전송
2 수동 업데이트 설치:
  • PowerShell에서 다음 명령어 실행:
# MSU 파일 설치
wusa.exe "C:\Updates\update.msu" /quiet /norestart
# 또는 DISM 사용
dism /online /add-package /packagepath:"C:\Updates\update.msu"
# CAB 파일 설치
dism /online /add-package /packagepath:"C:\Updates\update.cab"

방법 8: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사:
  • 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
Get-WindowsErrorReporting

⚠️ 주의사항

Server Core 환경에서 주의점

  • Server Core는 GUI가 없는 최소 설치 환경입니다
  • Desktop Experience 설치 시 상당한 시간과 리소스가 필요합니다
  • PowerShell 및 명령줄 도구 사용에 익숙해야 합니다
  • 원격 관리 도구 사용을 권장합니다
  • 서버 역할 변경 시 신중하게 검토하세요
  • 네트워크 설정이 올바른지 확인하세요
  • WSUS 서버 사용 시 정책 설정을 확인하세요
  • 시스템 복구 지점을 미리 생성하세요

언제 전문가의 도움이 필요한가?

  • Server Core 환경 관리 경험이 부족한 경우
  • 기업 환경에서 복잡한 WSUS 정책이 적용된 경우
  • Active Directory 도메인 환경에서 정책 충돌이 있는 경우
  • 하드웨어 호환성 문제로 인한 업데이트 실패
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 서버 역할 변경이 비즈니스에 미치는 영향이 큰 경우
  • 원격 관리 환경 구성이 필요한 경우

유사한 Server Core 관련 오류들

  • 0x8024001E: 서비스 중지 (WU_E_SERVICE_STOP)
  • 0x8024001D: 잘못된 업데이트 (WU_E_INVALID_UPDATE)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x80240041: 서버 선택 실패 (WU_E_SERVER_SELECTION_FAILED)
  • 0x80240042: 프로토콜 오류 (WU_E_PROTOCOL_ERROR)
  • 0x80070005: 액세스 거부됨
  • 0x80070002: 시스템이 지정된 파일을 찾을 수 없음

🛠️ 고급 해결 방법

Server Core 기능 확인 스크립트

  • PowerShell에서 다음 스크립트 실행:
# Server Core 상태 확인
$ServerCore = Get-WindowsFeature -Name "Server-Gui-Shell"
$GuiMgmt = Get-WindowsFeature -Name "Server-Gui-Mgmt-Infra"
Write-Host "Server GUI Shell: $($ServerCore.InstallState)"
Write-Host "GUI Management Infrastructure: $($GuiMgmt.InstallState)"
if ($ServerCore.InstallState -eq "Available") {
    Write-Host "현재 Server Core 모드입니다." -ForegroundColor Yellow
} else {
    Write-Host "Desktop Experience가 설치되어 있습니다." -ForegroundColor Green
}

자동 업데이트 스크립트

  • Server Core용 자동 업데이트 스크립트:
# PSWindowsUpdate 모듈 확인 및 설치
if (!(Get-Module -ListAvailable -Name PSWindowsUpdate)) {
    Install-Module PSWindowsUpdate -Force
}
Import-Module PSWindowsUpdate
# 업데이트 확인 및 설치
$Updates = Get-WindowsUpdate
if ($Updates) {
    Write-Host "발견된 업데이트: $($Updates.Count)개" -ForegroundColor Green
    Install-WindowsUpdate -AcceptAll -AutoReboot
} else {
    Write-Host "사용 가능한 업데이트가 없습니다." -ForegroundColor Yellow
}

🔧 원격 관리 설정

PowerShell 원격 관리 활성화

  • Server Core에서 다음 명령어 실행:
# PowerShell 원격 관리 활성화
Enable-PSRemoting -Force
Set-Item WSMan:\localhost\Client\TrustedHosts -Value "*" -Force
# 방화벽 규칙 확인
Get-NetFirewallRule -DisplayName "*Remote Management*" | Enable-NetFirewallRule
# WinRM 서비스 시작
Start-Service WinRM
Set-Service WinRM -StartupType Automatic

원격 컴퓨터에서 Server Core 관리

  • 관리 컴퓨터에서 다음 명령어 실행:
# 원격 세션 생성
$Session = New-PSSession -ComputerName "ServerCore-IP" -Credential (Get-Credential)
# 원격 명령 실행
Invoke-Command -Session $Session -ScriptBlock { Get-WindowsUpdate }
# 세션 종료
Remove-PSSession $Session
