Windows 11 0x50012 INSUFFICIENT_SPACE_SYSTEM_RESERVED_2 해결 가이드

⚠️ 주의사항: 이 오류는 시스템 예약 파티션(EFI 시스템 파티션)의 공간이 부족할 때 발생하는 오류입니다. 부팅 관련 중요한 파티션이므로 신중하게 작업해야 합니다.

🔍 오류 개요

0x50012 INSUFFICIENT_SPACE_SYSTEM_RESERVED_2는 Windows 11에서 시스템 예약 파티션(일반적으로 EFI 시스템 파티션)의 공간이 부족할 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • EFI 시스템 파티션(ESP) 공간 부족
  • 부팅 파일 및 드라이버 누적으로 인한 공간 부족
  • Windows 업데이트 파일이 시스템 파티션에 누적
  • 복구 환경(WinRE) 파일 크기 증가
  • 다중 부팅 환경에서 부팅 파일 중복
  • 언어팩 및 기능 업데이트로 인한 공간 사용
  • 시스템 예약 파티션 크기가 너무 작음 (100MB 이하)
  • 손상된 부팅 파일로 인한 공간 낭비

🛠️ 해결 방법

방법 1: 시스템 예약 파티션 공간 확인

1 디스크 관리에서 파티션 확인:
  • Windows + X 키를 누르고 "디스크 관리" 선택
  • 시스템 예약 파티션 또는 EFI 시스템 파티션 확인
  • 파티션 크기 및 사용 가능한 공간 확인
  • 일반적으로 100MB~500MB 크기의 파티션
2 명령 프롬프트로 상세 정보 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 파티션 정보 확인:
diskpart
list disk
select disk 0
list partition
select partition 1
detail partition

방법 2: EFI 시스템 파티션 정리

1 EFI 파티션 마운트:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 EFI 파티션을 드라이브 문자에 할당:
diskpart
list disk
select disk 0
list partition
select partition 1
assign letter=Z
exit
2 EFI 파티션 내용 확인 및 정리:
  • 다음 명령어로 EFI 파티션 내용 확인:
dir Z:\ /s
cd Z:\EFI
dir
  • 불필요한 폴더 삭제 (주의: Microsoft, Boot 폴더는 삭제하지 말 것)
  • 오래된 부팅 항목 정리:
bcdedit /enum firmware
bcdedit /delete {불필요한-부팅-항목-GUID}

방법 3: Windows 복구 환경(WinRE) 정리

1 WinRE 상태 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WinRE 정보 확인:
reagentc /info
2 WinRE 비활성화 및 재구성:
  • 다음 명령어로 WinRE 임시 비활성화:
reagentc /disable
reagentc /enable
  • WinRE 이미지 최적화:
dism /cleanup-wim

방법 4: 부팅 파일 정리 및 최적화

1 부팅 구성 데이터(BCD) 정리:
  • 다음 명령어로 BCD 백업 생성:
bcdedit /export C:\BCD_Backup
  • 불필요한 부팅 항목 제거:
bcdedit /enum all
bcdedit /delete {불필요한-항목-GUID}
2 부팅 파일 재구성:
  • 부팅 파일 재생성:
bcdboot C:\Windows /s Z: /f UEFI
bootrec /fixboot
bootrec /rebuildbcd

방법 5: 시스템 예약 파티션 확장

1 인접 파티션 공간 확보:
  • 디스크 관리에서 시스템 예약 파티션 옆 파티션 축소
  • C: 드라이브 우클릭 → "볼륨 축소"
  • 최소 200MB 이상 공간 확보
  • 축소된 공간이 시스템 예약 파티션 바로 옆에 위치해야 함
2 파티션 확장 (고급 사용자용):
  • 타사 파티션 관리 도구 사용 권장
  • 또는 다음 명령어로 확장 (위험):
diskpart
list disk
select disk 0
list partition
select partition 1
extend size=200
  • ⚠️ 주의: 파티션 확장은 데이터 손실 위험이 있습니다

방법 6: 언어팩 및 기능 정리

1 불필요한 언어팩 제거:
  • 설정 → 시간 및 언어 → 언어
  • 사용하지 않는 언어팩 제거
  • 또는 PowerShell로 제거:
Get-WindowsPackage -Online | Where-Object {$_.PackageName -like "*Language*"}
Remove-WindowsPackage -Online -PackageName "불필요한-언어팩-이름"
2 선택적 기능 정리:
  • 설정 → 앱 → 선택적 기능
  • 사용하지 않는 기능 제거
  • PowerShell로 확인 및 제거:
Get-WindowsOptionalFeature -Online
Disable-WindowsOptionalFeature -Online -FeatureName "불필요한-기능-이름"

방법 7: 시스템 파일 정리

1 DISM을 이용한 구성 요소 정리:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 구성 요소 정리:
DISM /Online /Cleanup-Image /StartComponentCleanup
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
DISM /Online /Cleanup-Image /SPSuperseded
2 Windows Update 정리:
  • 디스크 정리 도구 실행:
cleanmgr /sageset:1
  • 다음 항목들을 체크하여 정리:
    • Windows Update 정리
    • 시스템 오류 메모리 덤프 파일
    • 이전 Windows 설치
    • 임시 파일

방법 8: 새로운 시스템 예약 파티션 생성

1 Windows PE 부팅 디스크 생성:
  • 다른 컴퓨터에서 Windows 설치 미디어 생성
  • 문제가 있는 컴퓨터를 설치 미디어로 부팅
  • "컴퓨터 복구" → "문제 해결" → "고급 옵션" → "명령 프롬프트"
2 새로운 EFI 파티션 생성:
  • 다음 명령어로 새로운 EFI 파티션 생성:
diskpart
list disk
select disk 0
list partition
create partition efi size=500
format quick fs=fat32 label="System"
assign letter=S
exit
  • 부팅 파일 복사:
bcdboot C:\Windows /s S: /f UEFI

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 위험도 재부팅 필요
시스템 예약 파티션 공간 확인 쉬움 10-15분 낮음 아니오
EFI 시스템 파티션 정리 보통 20-30분 보통 예
WinRE 정리 보통 15-25분 보통 아니오
부팅 파일 정리 어려움 30-45분 높음 예
파티션 확장 매우 어려움 60-120분 매우 높음 예
언어팩 정리 쉬움 15-30분 낮음 예
시스템 파일 정리 보통 30-60분 낮음 아니오
새로운 파티션 생성 매우 어려움 120-180분 매우 높음 예

🔧 주요 명령어 참조

파티션 관리 명령어

  • diskpart: 디스크 파티션 관리 도구
  • list disk: 디스크 목록 확인
  • list partition: 파티션 목록 확인
  • assign letter=Z: 드라이브 문자 할당

부팅 관련 명령어

  • bcdedit /enum: 부팅 구성 데이터 확인
  • bcdboot C:\Windows /s Z: /f UEFI: 부팅 파일 재생성
  • reagentc /info: Windows 복구 환경 정보
  • bootrec /rebuildbcd: 부팅 구성 데이터 재구성

⚠️ 주의사항

시스템 파티션 작업 시 주의점

  • 시스템 예약 파티션은 부팅에 필수적이므로 신중하게 작업하세요
  • 모든 작업 전에 중요한 데이터를 백업하세요
  • 부팅 구성 데이터(BCD) 백업을 반드시 생성하세요
  • 파티션 확장 작업은 데이터 손실 위험이 매우 높습니다
  • EFI 파티션의 Microsoft, Boot 폴더는 절대 삭제하지 마세요
  • 작업 중 전원이 차단되지 않도록 주의하세요

언제 전문가의 도움이 필요한가?

  • 파티션 구조가 복잡하거나 손상된 경우
  • 다중 부팅 환경에서 문제가 발생한 경우
  • 하드웨어 RAID 환경에서 문제가 발생한 경우
  • 중요한 업무용 시스템에서 문제가 발생한 경우
  • 파티션 확장이 필요하지만 경험이 부족한 경우

✅ 예방 방법

  • 정기적인 시스템 파일 정리 (월 1회)
  • 불필요한 언어팩 및 기능 제거
  • Windows Update 정리 정기적 실행
  • 시스템 예약 파티션 공간 모니터링
  • 다중 부팅 환경 구성 시 충분한 EFI 파티션 크기 확보
  • 정기적인 부팅 구성 데이터 백업
  • 시스템 복원 지점 정기적 생성

💡 추가 팁

  • EFI 파티션 권장 크기는 최소 500MB 이상
  • 다중 부팅 환경에서는 1GB 이상 권장
  • 파티션 작업 전 부팅 가능한 복구 미디어 준비
  • 시스템 예약 파티션 정리는 정기적으로 수행
  • 타사 파티션 관리 도구 사용 시 신뢰할 수 있는 제품 선택
  • UEFI 펌웨어 설정에서 Secure Boot 상태 확인

🔍 관련 오류 코드

유사한 시스템 파티션 관련 오류들

  • 0x50011: INSUFFICIENT_SPACE_SYSTEM_RESERVED
  • 0x80070070: 디스크 공간이 부족합니다
  • 0xC1900101: 설치 실패 - 첫 번째 부팅에서
  • 0x800F0922: 시스템 파일 무결성 오류
  • 0xC000000F: 부팅 구성 데이터 누락
  • 0xC0000225: 부팅 장치에 액세스할 수 없음

🛠️ 고급 진단 도구

파티션 진단 도구

  • 디스크 관리: diskmgmt.msc - 파티션 상태 확인
  • DiskPart: diskpart - 고급 파티션 관리
  • 시스템 정보: msinfo32 - 시스템 구성 확인
  • 이벤트 뷰어: eventvwr.msc - 시스템 오류 로그

부팅 진단 도구

  • BCDEdit: bcdedit - 부팅 구성 관리
  • BootRec: bootrec - 부팅 복구 도구
  • ReAgentC: reagentc - 복구 환경 관리
  • DISM: dism - 시스템 이미지 관리

📊 시스템 예약 파티션 크기 가이드

권장 파티션 크기

시스템 구성 최소 크기 권장 크기 설명
단일 부팅 (UEFI) 100MB 500MB 기본 Windows 설치
다중 부팅 500MB 1GB 여러 OS 부팅 파일
서버 환경 500MB 1GB 추가 복구 도구
개발 환경 1GB 2GB 다양한 부팅 옵션

🔄 자동화 스크립트

시스템 예약 파티션 정리 스크립트

@echo off
echo 시스템 예약 파티션 정리 시작...
echo 현재 파티션 상태 확인 중...
echo list disk g> diskpart_script.txt
echo select disk 0 >> diskpart_script.txt
echo list partition >> diskpart_script.txt
diskpart /s diskpart_script.txt
del diskpart_script.txt
echo EFI 파티션 마운트 중...
echo select disk 0 > mount_efi.txt
echo select partition 1 >> mount_efi.txt
echo assign letter=Z >> mount_efi.txt
diskpart /s mount_efi.txt
del mount_efi.txt
echo 시스템 파일 정리 중...
DISM /Online /Cleanup-Image /StartComponentCleanup
echo Windows Update 정리 중...
cleanmgr /sagerun:1
echo WinRE 최적화 중...
reagentc /disable
reagentc /enable
echo EFI 파티션 마운트 해제 중...
echo select disk 0 > unmount_efi.txt
echo select partition 1 >> unmount_efi.txt
echo remove letter=Z >> unmount_efi.txt
diskpart /s unmount_efi.txt
del unmount_efi.txt
echo 정리 완료! 시스템을 재시작하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 시스템 재시작

이 가이드를 통해 시스템 예약 파티션 공간 부족 문제가 해결되기를 바랍니다. 파티션 관련 문제가 지속되면 전문가의 도움을 받으시기 바랍니다.
