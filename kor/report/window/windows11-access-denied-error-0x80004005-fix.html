Windows 11 0x80004005 General Access Denied 해결 가이드

⚠️ 주의사항: 이 오류는 Windows에서 파일, 폴더, 레지스트리 또는 시스템 리소스에 대한 접근이 거부될 때 발생하는 일반적인 권한 오류입니다. 아래 해결 방법을 순서대로 시도하시기 바랍니다.

🔍 오류 개요

0x80004005 General Access Denied는 Windows 11에서 가장 흔하게 발생하는 권한 관련 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 파일 또는 폴더 접근 권한 부족
  • 관리자 권한 필요한 작업 수행 시
  • 레지스트리 키 접근 권한 문제
  • 시스템 파일 보호 기능
  • 사용자 계정 컨트롤(UAC) 제한
  • 손상된 사용자 권한 설정
  • 바이러스 또는 악성 코드 감염
  • Windows 업데이트 후 권한 변경

🛠️ 해결 방법

방법 1: 관리자 권한으로 실행

1 프로그램을 관리자 권한으로 실행:
  • 문제가 되는 프로그램 또는 파일 우클릭
  • "관리자 권한으로 실행" 선택
  • UAC 창에서 "예" 클릭
  • 작업이 정상적으로 수행되는지 확인
2 명령 프롬프트 관리자 권한 실행:
  • Windows + X 키를 누르고 "Windows Terminal(관리자)" 선택
  • 또는 시작 메뉴에서 "cmd" 검색 후 "관리자 권한으로 실행"
  • 필요한 명령어 실행

방법 2: 파일 및 폴더 권한 수정

1 파일/폴더 권한 확인:
  • 문제가 되는 파일 또는 폴더 우클릭
  • "속성" → "보안" 탭 선택
  • 현재 사용자 계정이 목록에 있는지 확인
  • 권한 수준 확인 (읽기, 쓰기, 모든 권한)
2 권한 추가 및 수정:
  • "편집" 버튼 클릭
  • "추가" 버튼을 클릭하여 사용자 계정 추가
  • "고급" → "지금 찾기"로 사용자 계정 검색
  • 해당 계정 선택 후 "모든 권한" 체크
  • "확인" 클릭하여 변경 사항 적용
3 소유권 변경:
  • "고급" 버튼 클릭
  • "소유자" 탭에서 "변경" 클릭
  • 현재 사용자 계정으로 소유권 변경
  • "하위 컨테이너 및 개체의 소유자 바꾸기" 체크

방법 3: 사용자 계정 컨트롤(UAC) 설정 조정

1 UAC 설정 확인:
  • 제어판 → 사용자 계정 → 사용자 계정 컨트롤 설정 변경
  • 현재 UAC 수준 확인
  • 필요시 "앱이 컴퓨터를 변경하려고 할 때 알림" 수준으로 조정
  • 변경 후 재부팅
2 관리자 계정 활성화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 입력:
net user administrator /active:yes
net user administrator [새비밀번호]
  • 내장 관리자 계정으로 로그인하여 문제 해결

방법 4: 레지스트리 권한 수정

1 레지스트리 편집기 실행:
  • Windows + R 키를 누르고 regedit 입력
  • 관리자 권한으로 실행
  • 문제가 되는 레지스트리 키로 이동
2 레지스트리 키 권한 변경:
  • 해당 키 우클릭 → "사용 권한" 선택
  • 현재 사용자 계정 선택
  • "모든 권한" 체크박스 선택
  • "고급" → "소유자" 탭에서 소유권 변경
  • 변경 사항 적용 후 레지스트리 편집기 종료

방법 5: Windows 서비스 권한 확인

1 서비스 관리자 실행:
  • Windows + R 키를 누르고 services.msc 입력
  • 문제와 관련된 서비스 찾기
  • 서비스 우클릭 → "속성" 선택
2 서비스 로그온 계정 확인:
  • "로그온" 탭 선택
  • "로컬 시스템 계정" 또는 적절한 계정 선택
  • "서비스와 데스크톱 상호 작용 허용" 체크 (필요시)
  • 서비스 재시작

방법 6: 시스템 파일 검사 및 복구

1 SFC 스캔 실행:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 입력:
sfc /scannow
  • 스캔 완료까지 기다리기
  • 손상된 파일 복구 완료 후 재부팅
2 DISM 명령어 실행:
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth

방법 7: 사용자 프로필 권한 재설정

1 사용자 프로필 폴더 권한 확인:
  • C:\Users\[사용자명] 폴더로 이동
  • 폴더 우클릭 → "속성" → "보안" 탭
  • 현재 사용자 계정의 권한 확인
  • 필요시 "모든 권한" 부여
2 임시 사용자 계정 생성:
  • 설정 → 계정 → 가족 및 기타 사용자
  • "다른 사용자를 이 PC에 추가" 클릭
  • 새 관리자 계정 생성
  • 새 계정으로 로그인하여 문제 재현 여부 확인

방법 8: 바이러스 검사 및 악성 코드 제거

1 Windows Defender 전체 검사:
  • Windows 보안 → 바이러스 및 위협 방지
  • "검사 옵션" → "Microsoft Defender 오프라인 검사" 선택
  • "지금 검사" 클릭하여 재부팅 후 검사 실행
2 악성 소프트웨어 제거 도구 실행:
  • Microsoft 공식 사이트에서 MSRT 다운로드
  • 관리자 권한으로 실행
  • 전체 검사 수행
  • 발견된 악성 코드 제거

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
관리자 권한으로 실행 쉬움 1-5분 매우 높음 아니오
파일/폴더 권한 수정 보통 10-20분 높음 아니오
UAC 설정 조정 쉬움 5-10분 보통 예
레지스트리 권한 수정 어려움 15-30분 보통 아니오
Windows 서비스 권한 확인 보통 10-15분 보통 아니오
시스템 파일 검사 쉬움 30-60분 높음 예
사용자 프로필 권한 재설정 보통 20-40분 높음 예
바이러스 검사 쉬움 60-120분 보통 예

🔧 주요 명령어 참조

권한 관련 명령어

  • icacls [파일경로] /grant [사용자]:F: 파일 권한 부여
  • takeown /f [파일경로] /r /d y: 파일 소유권 가져오기
  • net user administrator /active:yes: 관리자 계정 활성화
  • whoami /priv: 현재 사용자 권한 확인

시스템 복구 명령어

  • sfc /scannow: 시스템 파일 검사
  • DISM /Online /Cleanup-Image /RestoreHealth: 시스템 이미지 복구
  • chkdsk C: /f /r: 디스크 오류 검사

⚠️ 주의사항

권한 수정 시 주의점

  • 시스템 파일의 권한을 함부로 변경하지 마세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 관리자 계정 활성화 후 보안에 주의하세요
  • UAC 설정을 너무 낮게 설정하지 마세요
  • 권한 변경 후 시스템 안정성을 확인하세요
  • 중요한 데이터는 미리 백업하세요

언제 전문가의 도움이 필요한가?

  • 시스템 전체에 권한 문제가 발생한 경우
  • 중요한 시스템 파일이 손상된 경우
  • 악성 코드 감염이 의심되는 경우
  • 기업 환경에서 도메인 권한 문제가 발생한 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 시스템 백업 및 복원 지점 생성
  • 신뢰할 수 있는 소프트웨어만 설치
  • 관리자 권한이 필요한 작업 시 주의
  • 정기적인 바이러스 검사 실행
  • Windows 업데이트 정기적 설치
  • 사용자 계정별 적절한 권한 설정
  • 시스템 파일 및 폴더 권한 함부로 변경 금지

💡 추가 팁

  • 권한 오류 발생 시 구체적인 파일 경로나 작업 내용 기록
  • 이벤트 뷰어에서 관련 오류 로그 확인
  • 권한 문제는 대부분 관리자 권한으로 해결 가능
  • 파일 권한 변경 시 상위 폴더부터 확인
  • 네트워크 드라이브 접근 시 자격 증명 확인
  • 그룹 정책으로 인한 제한 사항 확인

🔍 관련 오류 코드

유사한 권한 관련 오류들

  • 0x80070005: 액세스가 거부되었습니다
  • 0x80004003: 잘못된 포인터
  • 0x80070002: 지정된 파일을 찾을 수 없습니다
  • 0x80070003: 지정된 경로를 찾을 수 없습니다
  • 0x8007000E: 메모리가 부족합니다
  • 0x80004001: 구현되지 않음

🛠️ 고급 권한 관리 도구

Windows 내장 도구

  • 로컬 보안 정책: secpol.msc - 보안 정책 설정
  • 로컬 사용자 및 그룹: lusrmgr.msc - 사용자 계정 관리
  • 그룹 정책 편집기: gpedit.msc - 시스템 정책 설정
  • 권한 부여 관리자: azman.msc - 권한 관리

타사 도구

  • Process Monitor: 파일/레지스트리 접근 모니터링
  • AccessChk: 권한 확인 도구
  • SetACL: 고급 권한 설정 도구

📊 권한 문제 진단 체크리스트

단계별 진단 방법

  1. 오류 발생 상황 파악: 어떤 작업 중 오류 발생했는지 확인
  2. 사용자 계정 확인: 현재 로그인한 계정의 권한 수준 확인
  3. 대상 파일/폴더 확인: 접근하려는 리소스의 권한 설정 확인
  4. 관리자 권한 테스트: 관리자 권한으로 동일 작업 시도
  5. 다른 사용자 계정 테스트: 다른 계정으로 로그인하여 테스트
  6. 안전 모드 테스트: 안전 모드에서 동일 작업 시도

이 가이드를 통해 General Access Denied 권한 문제가 해결되기를 바랍니다. 권한 관련 문제가 지속되면 시스템 관리자나 Microsoft 지원팀에 문의하시기 바랍니다.
