Windows 11 0x80240006 WU_E_TOOMANYRANGES 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 다운로드 과정에서 너무 많은 데이터 범위가 요청될 때 발생하는 오류입니다. BITS 큐 오버플로우나 동시 연결 과다가 주요 원인입니다.

🔍 오류 개요

0x80240006 WU_E_TOOMANYRANGES는 Windows 11에서 업데이트 다운로드 중 너무 많은 데이터 범위가 동시에 요청될 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • BITS(Background Intelligent Transfer Service) 큐 오버플로우
  • 동시 다운로드 연결 수 과다
  • 다운로드 범위 분할 과도
  • 네트워크 대역폭 제한으로 인한 큐 누적
  • 배달 최적화 서비스 과부하
  • 메모리 부족으로 인한 범위 관리 실패
  • 다중 업데이트 동시 처리 충돌
  • BITS 작업 관리 오류

🛠️ 해결 방법

방법 1: BITS 큐 완전 정리 및 재설정

1 모든 BITS 작업 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop bits
net stop wuauserv
net stop dosvc
net stop cryptSvc
2 BITS 큐 완전 정리:
  • 다음 명령어로 모든 BITS 작업 제거:
bitsadmin /reset /allusers
bitsadmin /util /purge
bitsadmin /cancel /allusers
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr*.dat
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader\*.log
3 BITS 데이터베이스 재생성:
  • BITS 데이터베이스 파일 삭제 및 재생성:
del /f /q "%ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr.db"
del /f /q "%ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr0.dat"
del /f /q "%ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr1.dat"

방법 2: 동시 연결 수 제한 설정

1 레지스트리를 통한 BITS 연결 제한:
  • Windows + R 키를 누르고 regedit 입력
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\BITS
  • 키가 없다면 새로 생성
  • 다음 DWORD 값들을 생성:
    • "MaxInternetBandwidthBytesPerSecond": 1048576 (1MB/s)
    • "MaxBandwidthValidFrom": 0
    • "MaxBandwidthValidTo": 24
    • "MaxFilesPerJob": 10
2 Windows Update 동시 연결 제한:
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\DeliveryOptimization\Config
  • 다음 DWORD 값들을 생성 또는 수정:
    • "DOMaxCacheSize": 1024 (1GB)
    • "DOMaxDownloadBandwidth": 50 (50%)
    • "DOMaxUploadBandwidth": 10 (10%)
    • "DODownloadMode": 0 (HTTP만 사용)

방법 3: 배달 최적화 서비스 최적화

1 배달 최적화 설정 조정:
  • 설정 → 업데이트 및 보안 → 배달 최적화
  • "다른 PC에서 다운로드 허용" 비활성화
  • "고급 옵션" → "다운로드 설정"
  • "절대값" 선택 후 대역폭을 50% 이하로 제한
  • "업로드 설정"에서 업로드 대역폭을 10% 이하로 제한
2 배달 최적화 캐시 정리:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
net stop dosvc
rd /s /q "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization"
md "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization"
net start dosvc

방법 4: Windows Update 순차 처리 설정

1 업데이트 개별 설치:
  • 설정 → 업데이트 및 보안 → Windows Update
  • "업데이트 확인" 클릭
  • 여러 업데이트가 표시되면 하나씩 개별 설치
  • 각 업데이트 설치 완료 후 재부팅
  • 다음 업데이트 진행
2 PowerShell을 통한 순차 업데이트:
  • 관리자 권한으로 PowerShell 실행
  • PSWindowsUpdate 모듈 설치:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
  • 업데이트 목록 확인 후 개별 설치:
Get-WindowsUpdate
Install-WindowsUpdate -KBArticleID "KB번호" -AcceptAll -AutoReboot

방법 5: 시스템 리소스 최적화

1 메모리 사용량 최적화:
  • 작업 관리자에서 불필요한 프로세스 종료
  • 시작 프로그램 최소화
  • 백그라운드 앱 비활성화
  • 가상 메모리 크기 증가:
제어판 → 시스템 → 고급 시스템 설정 → 성능 설정 → 고급 → 가상 메모리 변경
초기 크기: RAM 크기 × 1.5
최대 크기: RAM 크기 × 3
2 디스크 공간 확보:
  • 디스크 정리 도구 실행:
cleanmgr /sageset:1
cleanmgr /sagerun:1
  • 임시 파일 삭제:
del /f /s /q %temp%\*.*
del /f /s /q %windir%\Temp\*.*
del /f /s /q %windir%\Prefetch\*.*

방법 6: 네트워크 연결 안정화

1 네트워크 어댑터 최적화:
  • 장치 관리자 → 네트워크 어댑터
  • 네트워크 어댑터 우클릭 → "속성"
  • "고급" 탭에서 다음 설정 조정:
    • "Receive Buffers": 512
    • "Transmit Buffers": 256
    • "Interrupt Moderation": 활성화
2 TCP/IP 설정 최적화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
netsh int tcp set global autotuninglevel=normal
netsh int tcp set global chimney=enabled
netsh int tcp set global rss=enabled
netsh int tcp set global netdma=enabled

방법 7: Windows Update 서비스 재구성

1 서비스 의존성 재설정:
  • 다음 명령어로 서비스 의존성 확인 및 재설정:
sc config wuauserv depend= rpcss
sc config bits depend= rpcss/eventlog
sc config cryptsvc depend= rpcss
sc config dosvc depend= rpcss
2 서비스 시작 순서 조정:
  • 서비스를 올바른 순서로 시작:
net start rpcss
net start eventlog
net start cryptsvc
net start bits
net start wuauserv
net start dosvc

방법 8: 수동 업데이트 및 오프라인 설치

1 Microsoft Update 카탈로그 사용:
  • 웹 브라우저에서 Microsoft Update 카탈로그 접속
  • 실패한 업데이트의 KB 번호 검색
  • 시스템 아키텍처에 맞는 업데이트 개별 다운로드
  • 다운로드한 파일을 관리자 권한으로 순차 설치
2 Windows Update Assistant 사용:
  • Microsoft 공식 사이트에서 Windows 11 Update Assistant 다운로드
  • 관리자 권한으로 프로그램 실행
  • "지금 업데이트" 클릭하여 단일 프로세스로 업데이트
  • 업데이트 완료 후 시스템 재시작

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
BITS 큐 완전 정리 보통 20-30분 매우 높음 예
동시 연결 수 제한 어려움 15-25분 높음 예
배달 최적화 서비스 최적화 쉬움 10-20분 높음 아니오
순차 처리 설정 보통 30-60분 높음 예
시스템 리소스 최적화 쉬움 20-40분 보통 예
네트워크 연결 안정화 보통 15-30분 보통 예
서비스 재구성 어려움 25-35분 높음 예
수동 업데이트 설치 보통 60-120분 매우 높음 예

🔧 주요 명령어 참조

BITS 관리 명령어

  • bitsadmin /reset /allusers: 모든 BITS 작업 재설정
  • bitsadmin /util /purge: BITS 큐 정리
  • bitsadmin /list /allusers /verbose: BITS 작업 상세 목록
  • bitsadmin /cancel /allusers: 모든 BITS 작업 취소

시스템 최적화 명령어

  • cleanmgr /sageset:1: 디스크 정리 설정
  • cleanmgr /sagerun:1: 자동 디스크 정리 실행
  • netsh int tcp set global autotuninglevel=normal: TCP 자동 조정
  • sc config wuauserv depend= rpcss: 서비스 의존성 설정

⚠️ 주의사항

시스템 설정 변경 시 주의점

  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • BITS 작업 삭제 시 다른 다운로드에 영향을 줄 수 있습니다
  • 네트워크 설정 변경 시 연결 상태를 확인하세요
  • 가상 메모리 크기 변경 시 충분한 디스크 공간을 확보하세요
  • 서비스 의존성 변경 시 시스템 안정성을 확인하세요
  • 업데이트 순차 설치 시 충분한 시간을 확보하세요

언제 전문가의 도움이 필요한가?

  • 시스템 리소스가 심각하게 부족한 경우
  • 네트워크 인프라 문제가 의심되는 경우
  • 하드웨어 성능 문제가 의심되는 경우
  • 기업 환경에서 네트워크 정책 문제가 발생한 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 BITS 큐 정리 (월 1회)
  • 시스템 리소스 모니터링 및 최적화
  • 네트워크 대역폭 적절한 제한 설정
  • 동시 다운로드 프로그램 사용 자제
  • 충분한 시스템 메모리 확보
  • 정기적인 시스템 재시작
  • 업데이트 시간대 분산

💡 추가 팁

  • 대용량 업데이트는 네트워크 트래픽이 적은 시간에 설치
  • 여러 업데이트가 있을 때는 개별적으로 순차 설치
  • BITS 큐 상태를 정기적으로 모니터링
  • 시스템 성능 모니터로 리소스 사용량 확인
  • 네트워크 QoS 설정으로 업데이트 우선순위 조정
  • 배달 최적화 설정을 네트워크 환경에 맞게 조정

🔍 관련 오류 코드

  • 0x80240001: WU_E_NO_SERVICE

  • 0x80240003: WU_E_UNKNOWN_ID
  • 0x80240004: WU_E_NOT_INITIALIZED
  • 0x80240005: WU_E_RANGEOVERLAP
  • 0x80240007: WU_E_INVALIDINDEX

🛠️ 고급 진단 도구

BITS 및 다운로드 진단 도구

  • BITS 관리 콘솔: bitsadmin - BITS 작업 관리
  • 리소스 모니터: resmon - 네트워크 및 메모리 사용량
  • 성능 모니터: perfmon - 시스템 성능 모니터링
  • 이벤트 뷰어: eventvwr - BITS 관련 오류 로그

PowerShell 진단 명령어

  • Get-BitsTransfer -AllUsers: 모든 BITS 전송 작업 확인
  • Get-DeliveryOptimizationStatus: 배달 최적화 상태 확인
  • Get-Process | Sort-Object WorkingSet -Descending: 메모리 사용량 확인
  • Get-Counter "\Memory\Available MBytes": 사용 가능한 메모리 확인

🔄 자동화 스크립트

BITS 큐 최적화 스크립트

@echo off
echo BITS 큐 최적화 및 범위 제한 설정 시작...
net stop bits
net stop wuauserv
net stop dosvc
echo BITS 큐 완전 정리 중...
bitsadmin /reset /allusers
bitsadmin /util /purge
bitsadmin /cancel /allusers
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr*.dat
echo 배달 최적화 캐시 정리 중...
rd /s /q "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
md "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
echo 서비스 재시작 중...
net start bits
net start wuauserv
net start dosvc
echo 최적화 완료! 시스템을 재시작하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 시스템 재시작
