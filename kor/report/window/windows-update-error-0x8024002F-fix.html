Windows 0x8024002F WU_E_CALL_CANCELLED_BY_POLICY 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 정책에 의해 호출이 취소되었을 때 발생하는 문제입니다. 그룹 정책 제한, 보안 정책 설정, 또는 기업 환경의 업데이트 정책이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x8024002F (WU_E_CALL_CANCELLED_BY_POLICY)는 Windows Update 시스템에서 정책 설정에 의해 업데이트 호출이 취소되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 그룹 정책에 의한 Windows Update 호출 제한
  • 기업 환경의 WSUS 정책으로 인한 업데이트 차단
  • 도메인 정책에 의한 업데이트 프로세스 중단
  • 보안 정책 설정으로 인한 업데이트 호출 취소
  • 사용자 권한 부족으로 인한 정책 제한
  • 시간 제한 정책으로 인한 업데이트 중단
  • 네트워크 정책으로 인한 서버 접근 차단
  • 업데이트 스케줄 정책으로 인한 호출 취소
  • 타사 보안 소프트웨어의 정책 간섭
  • 레지스트리 정책 설정으로 인한 기능 제한

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 정책 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 그룹 정책 설정 확인 및 수정

1 로컬 그룹 정책 편집기 실행 (Pro/Enterprise):
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들을 확인 및 수정:
  • - "사용자가 Windows Update를 일시 중지하지 못하도록 방지" → "구성되지 않음"
  • - "Windows Update에 대한 액세스 제거" → "구성되지 않음"
  • - "자동 업데이트 사용 안 함" → "구성되지 않음"
  • - "업데이트 설치 일정 설정" → 적절히 구성
2 사용자 권한 및 정책 확인:
  • 사용자 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 사용자별 Windows Update 정책 확인
  • 정책 변경 후 다음 명령어 실행:
gpupdate /force
gpupdate /target:computer /force
gpupdate /target:user /force
secedit /refreshpolicy machine_policy /enforce
secedit /refreshpolicy user_policy /enforce

방법 3: 도메인 정책 확인 및 조정

1 현재 적용된 그룹 정책 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 적용된 정책 확인:
gpresult /r
gpresult /h C:\GPResult.html
gpresult /scope:computer /v
gpresult /scope:user /v
2 도메인 정책 우선순위 확인:
  • 도메인 환경에서 정책 우선순위 분석
  • IT 관리자와 협의하여 정책 조정
  • 임시로 로컬 정책 우선 적용 (테스트 목적):
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v NoAutoUpdate /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v DisableWindowsUpdateAccess /t REG_DWORD /d 0 /f
gpupdate /force
  • ⚠️ 주의: 기업 환경에서는 IT 관리자와 반드시 상의

방법 4: WSUS 정책 설정 확인

1 WSUS 서버 설정 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WSUS 설정 확인:
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUStatusServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v AUOptions
2 WSUS 정책 임시 비활성화 (테스트 목적):
  • WSUS 서버 연결에 문제가 있는 경우:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer /t REG_DWORD /d 0 /f
net stop wuauserv
net start wuauserv
wuauclt /detectnow
  • 테스트 완료 후 원래 설정으로 복원
  • ⚠️ 주의: 기업 환경에서는 IT 관리자 승인 필요

방법 5: 레지스트리 정책 설정 수정

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 정책 제한 레지스트리 키 수정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v DisableWindowsUpdateAccess /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v SetDisableUXWUAccess /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v NoAutoUpdate /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v AUPowerManagement /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
3 기본 Windows Update 정책 복원:
  • 다음 명령어로 기본 설정 복원:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallDay /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v ScheduledInstallTime /t REG_DWORD /d 3 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /v AcceptTrustedPublisherCerts /t REG_DWORD /d 1 /f

방법 6: Windows Update 서비스 권한 복구

1 Windows Update 서비스 권한 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 서비스 권한 확인:
sc query wuauserv
sc sdshow wuauserv
sc queryex wuauserv
whoami /priv
2 서비스 권한 복구:
  • 다음 명령어로 서비스 권한 복구:
sc config wuauserv start= auto
sc config wuauserv type= own
sc config wuauserv error= normal
net stop wuauserv
net start wuauserv
wuauclt /resetauthorization /detectnow

방법 7: 타사 보안 소프트웨어 설정 확인

1 보안 소프트웨어 정책 확인:
  • 설치된 보안 소프트웨어의 Windows Update 관련 설정 확인
  • 다음 항목들을 확인:
  • - Windows Update 차단 설정
  • - 업데이트 스케줄 제한
  • - 시스템 정책 간섭
  • - 방화벽 규칙
2 보안 소프트웨어 설정 조정:
  • Windows Update 관련 제한 해제
  • 필요시 보안 소프트웨어 임시 비활성화
  • 업데이트 완료 후 보안 소프트웨어 재활성화
  • 보안 소프트웨어 예외 목록에 Windows Update 추가

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 정책 상태 확인 및 업데이트 강제 실행:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Get-WUSettings
Set-WUSettings -AcceptTrustedPublisherCerts
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -IgnoreReboot -Verbose
Get-WindowsUpdateLog -LogPath C:\WindowsUpdate.log
Reset-WUComponents -Verbose

  • 0x80240025: 사용자 액세스 비활성화 (WU_E_USER_ACCESS_DISABLED)
  • 0x8024001A: 정책 미설정 (WU_E_POLICY_NOT_SET)
  • 0x80240016: 설치 허용되지 않음 (WU_E_INSTALL_NOT_ALLOWED)
  • 0x80070005: 액세스 거부됨
  • 0x80240030: 정책 값 누락 (WU_E_POLICY_VALUE_MISSING)
  • 0x80240031: 정책 소스 누락 (WU_E_POLICY_SOURCE_MISSING)

🛠️ 고급 해결 방법

정책 상태 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 정책 상태 상세 분석:
$WUPolicies = @(
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate",
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU",
    "HKCU:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate"
)
Write-Host "=== Windows Update 정책 상태 분석 ===" -ForegroundColor Green
foreach ($PolicyPath in $WUPolicies) {
    if (Test-Path $PolicyPath) {
        Write-Host "정책 경로: $PolicyPath" -ForegroundColor Yellow
        Get-ItemProperty $PolicyPath | Format-List
    } else {
        Write-Host "정책 경로 없음: $PolicyPath" -ForegroundColor Green
    }
}

그룹 정책 결과 분석

  • 그룹 정책 적용 결과 상세 분석:
$GPResult = gpresult /r /scope:computer
$WUGPResult = $GPResult | Where-Object {$_ -like "*Windows Update*" -or $_ -like "*WindowsUpdate*"}
Write-Host "Windows Update 관련 그룹 정책:" -ForegroundColor Cyan
$WUGPResult
$Services = Get-Service -Name "wuauserv", "cryptSvc", "bits"
Write-Host "Windows Update 서비스 상태:" -ForegroundColor Cyan
$Services | Format-Table Name, Status, StartType
$CurrentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
$Principal = New-Object Security.Principal.WindowsPrincipal($CurrentUser)
$IsAdmin = $Principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
Write-Host "관리자 권한: $IsAdmin" -ForegroundColor $(if($IsAdmin){"Green"}else{"Red"})

🔄 정책 관리 가이드

Windows Update 정책 유형

  • 컴퓨터 정책: 시스템 전체에 적용되는 정책
  • 사용자 정책: 특정 사용자에게 적용되는 정책
  • 도메인 정책: Active Directory 도메인 정책
  • 로컬 정책: 로컬 컴퓨터에만 적용되는 정책
  • WSUS 정책: Windows Server Update Services 정책

정책 우선순위

우선순위 정책 유형 적용 범위 관리 방법
1 (최고) 도메인 정책 도메인 전체 Active Directory
2 사이트 정책 AD 사이트 Active Directory
3 OU 정책 조직 단위 Active Directory
4 (최저) 로컬 정책 로컬 컴퓨터 gpedit.msc

📊 정책 호출 취소 문제 진단 매트릭스

증상 가능한 원인 진단 방법 해결 우선순위
업데이트 검색 중단 그룹 정책 제한 gpresult 확인 매우 높음
업데이트 설치 차단 WSUS 정책 레지스트리 확인 높음
서비스 시작 실패 서비스 권한 문제 서비스 권한 확인 높음
시간 제한 초과 정책 시간 제한 정책 설정 확인 보통
사용자 권한 오류 UAC 설정 사용자 권한 확인 보통

🏢 기업 환경 특별 고려사항

도메인 환경에서의 정책 관리

  • Active Directory 정책: 도메인 컨트롤러에서 중앙 관리
  • OU별 정책: 조직 단위별 차별화된 정책 적용
  • 보안 필터링: 특정 그룹에만 정책 적용
  • WMI 필터: 하드웨어/소프트웨어 조건별 정책 적용
  • 정책 상속: 상위 OU 정책 상속 및 차단

WSUS 환경에서의 정책 관리

  • WSUS 서버 승인 정책
  • 클라이언트 그룹 관리
  • 업데이트 배포 일정
  • 자동 승인 규칙
  • 보고 및 모니터링

IT 관리자와 상의해야 할 사항

  • 도메인 그룹 정책 변경 권한
  • WSUS 서버 설정 및 정책
  • 보안 정책 및 규정 준수
  • 사용자 권한 상승 정책
  • 업데이트 배포 일정 및 승인

🔧 정책 관리 도구

Windows 내장 도구

  • gpedit.msc: 로컬 그룹 정책 편집기
  • gpmc.msc: 그룹 정책 관리 콘솔 (서버 환경)
  • rsop.msc: 정책 결과 집합
  • secpol.msc: 로컬 보안 정책

명령줄 도구

  • gpupdate: 그룹 정책 업데이트
  • gpresult: 그룹 정책 결과 확인
  • secedit: 보안 정책 관리
  • wuauclt: Windows Update 클라이언트

PowerShell 모듈

  • GroupPolicy: 그룹 정책 관리
  • PSWindowsUpdate: Windows Update 관리
  • UpdateServices: WSUS 서버 관리
  • ActiveDirectory: AD 환경 관리
