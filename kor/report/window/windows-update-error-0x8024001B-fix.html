Windows 0x8024001B WU_E_SELFUPDATE_IN_PROGRESS 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 Windows Update 클라이언트 자체의 업데이트가 진행 중일 때 발생하는 문제입니다. 자체 업데이트 프로세스 충돌이나 업데이트 클라이언트 손상이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x8024001B (WU_E_SELFUPDATE_IN_PROGRESS)는 Windows Update 클라이언트가 자체 업데이트를 수행하는 동안 다른 업데이트 작업이 시도될 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 클라이언트 자체 업데이트 진행 중
  • Windows Update Agent (WUA) 업데이트 프로세스 충돌
  • 동시에 실행되는 여러 업데이트 프로세스
  • 손상된 Windows Update 클라이언트
  • 업데이트 서비스 간의 리소스 경합
  • 백그라운드에서 진행 중인 자동 업데이트
  • Windows Update 서비스 잠금 상태
  • 시스템 재부팅이 필요한 상태에서의 업데이트 시도
  • 업데이트 클라이언트 버전 불일치
  • Windows Update 구성 요소 초기화 실패

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 자체 업데이트 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 진행 중인 업데이트 프로세스 대기 및 확인

1 현재 실행 중인 업데이트 프로세스 확인:
  • 작업 관리자 (Ctrl + Shift + Esc) 실행
  • 다음 프로세스들이 실행 중인지 확인:
  • - wuauclt.exe (Windows Update 클라이언트)
  • - TrustedInstaller.exe (신뢰할 수 있는 설치 관리자)
  • - svchost.exe (Windows Update 서비스)
  • - UsoClient.exe (업데이트 세션 오케스트레이터)
2 명령어를 통한 업데이트 상태 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 업데이트 상태 확인:
wuauclt /detectnow
usoclient ScanInstallWait
powercfg /requests
tasklist | findstr /i "wuauclt TrustedInstaller UsoClient"
sc query wuauserv
3 자체 업데이트 완료 대기:
  • 진행 중인 자체 업데이트가 있다면 완료될 때까지 대기 (30-60분)
  • 시스템 재부팅 후 다시 시도
  • 업데이트 완료 후 Windows Update 재실행

방법 3: Windows Update 서비스 재시작

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 클라이언트 프로세스 종료:
  • 다음 명령어로 관련 프로세스 확인 및 종료:
taskkill /f /im wuauclt.exe
taskkill /f /im UsoClient.exe
taskkill /f /im usocoreworker.exe
taskkill /f /im WaaSMedicAgent.exe
  • ⚠️ 주의: TrustedInstaller.exe는 강제 종료하지 마세요
3 Windows Update 캐시 및 임시 파일 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
4 서비스 재시작 및 자체 업데이트 재초기화:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 4: Windows Update Agent 수동 업데이트

1 현재 Windows Update Agent 버전 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 현재 버전 확인:
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" -Name "Version"
wmic datafile where name="C:\\Windows\\System32\\wuaueng.dll" get Version
[System.Environment]::OSVersion.Version
2 최신 Windows Update Agent 다운로드:
  • Microsoft 다운로드 센터에서 최신 WUA 다운로드
  • 시스템 아키텍처에 맞는 버전 선택 (x86, x64)
  • WindowsUpdateAgent30-x86.exe 또는 WindowsUpdateAgent30-x64.exe
3 Windows Update Agent 수동 설치:
  • 다운로드한 파일을 관리자 권한으로 실행
  • 또는 명령 프롬프트에서 다음 명령어 실행:
WindowsUpdateAgent30-x64.exe /quiet /norestart
shutdown /r /t 60

방법 5: Windows Update 구성 요소 재등록

1 Windows Update 핵심 구성 요소 재등록:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
regsvr32 /s wuapi.dll

regsvr32 /s wuweb.dll
regsvr32 /s qmgr.dll
regsvr32 /s qmgrprxy.dll
regsvr32 /s wucltux.dll
regsvr32 /s muweb.dll
regsvr32 /s wuwebv.dll
regsvr32 /s wudriver.dll
2 자체 업데이트 관련 구성 요소 재등록:
  • 다음 명령어들을 실행:
regsvr32 /s atl.dll
regsvr32 /s urlmon.dll
regsvr32 /s mshtml.dll

방법 6: 시스템 재부팅 및 단계적 업데이트

1 시스템 완전 재부팅:
  • 모든 프로그램 종료
  • 시스템 완전 종료 후 재시작
  • 재부팅 후 15-20분 대기 (백그라운드 프로세스 완료)
2 단계적 업데이트 접근:
  • Windows Update 설정에서 "업데이트 확인" 클릭
  • 한 번에 하나의 업데이트만 설치
  • 각 업데이트 설치 후 재부팅
  • 자체 업데이트가 먼저 완료되도록 허용

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
sfc /scanfile=C:\Windows\System32\wucltux.dll
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 자체 업데이트 상태 확인 및 강제 완료:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Get-WUHistory | Where-Object {$_.Title -like "*Windows Update*"}
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -IgnoreReboot -Verbose
Reset-WUComponents -Verbose

업데이트 프로세스 대기/확인 쉬움 30-90분 매우 높음 아니오
Windows Update 서비스 재시작 보통 30-45분 높음 예

구성 요소 재등록 보통 25-35분 높음 예

⚠️ 주의사항

자체 업데이트 진행 중 오류 해결 시 주의점

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 진행 중인 자체 업데이트를 강제로 중단하지 마세요
  • TrustedInstaller.exe 프로세스는 강제 종료하지 마세요
  • 자체 업데이트 완료까지 충분한 시간을 허용하세요
  • 시스템 재부팅 후 백그라운드 프로세스 완료를 기다리세요
  • 한 번에 하나의 업데이트만 설치하세요
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • Windows Update Agent 업데이트가 반복적으로 실패하는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 자체 업데이트가 무한 루프에 빠진 경우
  • 기업 환경에서 WSUS 서버와의 충돌이 있는 경우
  • 하드웨어 호환성 문제로 인한 지속적인 오류
  • 시스템 전체 재설치가 필요한 경우

✅ 예방 방법

  • 정기적인 시스템 재부팅 (주 1-2회)
  • Windows Update 자동 설치 설정
  • 충분한 시스템 리소스 확보
  • 업데이트 설치 중 다른 작업 최소화
  • 시스템 복구 지점 정기 생성
  • 안정적인 인터넷 연결 유지
  • 디스크 공간 충분히 확보 (최소 20GB 이상)
  • 정기적인 시스템 파일 검사 (월 1회)
  • 백그라운드 업데이트 완료 시간 확보

💡 추가 팁

  • 자체 업데이트 진행 중 오류는 주로 프로세스 충돌로 발생합니다
  • Windows Update Agent 자체 업데이트는 다른 업데이트보다 우선됩니다
  • 시스템 재부팅은 가장 간단하고 효과적인 해결 방법입니다
  • 자체 업데이트는 보통 30-60분 정도 소요됩니다
  • 업데이트 중 시스템을 강제 종료하지 마세요
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 Windows Update Agent 버전 확인을 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

🔍 관련 오류 코드

유사한 Windows Update 자체 업데이트 관련 오류들

  • 0x8024001A: 정책 미설정 (WU_E_POLICY_NOT_SET)
  • 0x80240019: 독점 설치 충돌 (WU_E_EXCLUSIVE_INSTALL_CONFLICT)
  • 0x80240018: 사용자 토큰 없음 (WU_E_NO_USERTOKEN)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x8024001D: 업데이트 설치 관리자 없음 (WU_E_NO_UPDATE_INSTALLER)
  • 0x8024001E: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)
  • 0x80070005: 액세스 거부됨

🛠️ 고급 해결 방법

Windows Update Agent 상태 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • WUA 상태 상세 분석:
$WUAVersion = (Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update").Version
Write-Host "Windows Update Agent 버전: $WUAVersion"
$WUService = Get-Service -Name "wuauserv"
Write-Host "Windows Update 서비스 상태: $($WUService.Status)"
$WUProcesses = Get-Process | Where-Object {$_.ProcessName -like "*wuau*" -or $_.ProcessName -like "*uso*"}
if ($WUProcesses) {
    Write-Host "실행 중인 Windows Update 프로세스:"
    $WUProcesses | Select-Object ProcessName, Id, StartTime | Format-Table
} else {
    Write-Host "실행 중인 Windows Update 프로세스가 없습니다."
}

자체 업데이트 진행 상황 모니터링

  • 자체 업데이트 진행 상황 실시간 모니터링:
while ($true) {
    $WUProcess = Get-Process -Name "wuauclt" -ErrorAction SilentlyContinue
    $TIProcess = Get-Process -Name "TrustedInstaller" -ErrorAction SilentlyContinue
    if ($WUProcess -or $TIProcess) {
        Write-Host "$(Get-Date): Windows Update 자체 업데이트 진행 중..." -ForegroundColor Yellow
    } else {
        Write-Host "$(Get-Date): 자체 업데이트 완료" -ForegroundColor Green
        break
    }
    Start-Sleep -Seconds 30
}

🔄 자체 업데이트 프로세스 이해

Windows Update Agent 자체 업데이트 단계

  1. 버전 확인: 현재 WUA 버전과 서버 버전 비교
  2. 다운로드: 새로운 WUA 구성 요소 다운로드
  3. 백업: 기존 WUA 구성 요소 백업
  4. 설치: 새로운 WUA 구성 요소 설치
  5. 등록: 새로운 구성 요소 시스템 등록
  6. 재시작: Windows Update 서비스 재시작
  7. 검증: 업데이트 성공 여부 확인

자체 업데이트 중 발생할 수 있는 문제

  • 프로세스 충돌: 동시 실행되는 업데이트 프로세스
  • 파일 잠금: 사용 중인 시스템 파일
  • 권한 문제: 관리자 권한 부족
