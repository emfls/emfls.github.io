Windows 11 0x80200056 UPGRADE_INTERRUPTED 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 업그레이드 과정이 중단되었을 때 발생하는 오류입니다. 전원 차단, 시스템 오류, 또는 사용자 중단으로 인해 발생할 수 있으며, 신중한 복구 작업이 필요합니다.

🔍 오류 개요

0x80200056 UPGRADE_INTERRUPTED는 Windows 11 업그레이드 과정이 예기치 않게 중단되었을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 업그레이드 중 전원 차단 또는 정전
  • 시스템 강제 종료 또는 재시작
  • 하드웨어 오류로 인한 시스템 중단
  • 디스크 공간 부족으로 인한 설치 중단
  • 메모리 오류로 인한 프로세스 중단
  • 네트워크 연결 중단으로 인한 다운로드 실패
  • 바이러스 백신 프로그램의 간섭
  • 시스템 파일 손상으로 인한 설치 실패

🛠️ 해결 방법

방법 1: 시스템 상태 확인 및 재시작

1 현재 시스템 상태 확인:
  • 시스템이 정상적으로 부팅되는지 확인
  • Windows 버전 확인 (Windows + R → winver)
  • 업그레이드가 어느 단계에서 중단되었는지 파악
  • 오류 메시지나 로그 파일 확인
2 시스템 완전 재시작:
  • 모든 프로그램 종료
  • 시작 메뉴 → 전원 → 다시 시작
  • 재시작 후 자동 복구 프로세스 실행 여부 확인
  • Windows Update에서 업그레이드 재시도 가능 여부 확인

방법 2: Windows Update 문제 해결사 실행

1 자동 문제 해결사 실행:
  • 설정 → 시스템 → 문제 해결 → 기타 문제 해결사
  • "Windows Update" 항목에서 "실행" 클릭
  • 문제 해결사가 중단된 업그레이드 관련 문제 자동 진단
  • 발견된 문제에 대한 수정 사항 적용
2 고급 문제 해결사 실행:
  • Windows + R 키를 누르고 다음 명령어 입력:
msdt.exe -id WindowsUpdateDiagnostic
  • "고급" 옵션 선택
  • "관리자 권한으로 실행" 체크
  • "자동으로 복구 적용" 체크
  • 모든 권장 수정 사항 적용

방법 3: Windows Update 구성 요소 재설정

1 Windows Update 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop dosvc
net stop usosvc
2 업그레이드 관련 폴더 정리:
  • 다음 명령어들을 실행하여 중단된 업그레이드 파일 정리:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q "C:\$Windows.~BT"
rd /s /q "C:\$Windows.~WS"
rd /s /q "C:\Windows10Upgrade"
del /f /q "C:\Windows\WindowsUpdate.log"
3 서비스 재시작:
  • 다음 명령어들을 순서대로 실행:
net start cryptSvc
net start bits
net start wuauserv
net start msiserver
net start dosvc
net start usosvc

방법 4: 시스템 파일 검사 및 복구

1 SFC 스캔 실행:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
sfc /scannow
  • 스캔 완료까지 기다리기 (30-90분 소요)
  • 손상된 파일 복구 완료 후 재부팅
2 DISM 명령어 실행:
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth
DISM /Online /Cleanup-Image /StartComponentCleanup

방법 5: 디스크 공간 확보 및 오류 검사

1 디스크 공간 확보:
  • Windows + R 키를 누르고 cleanmgr 입력
  • 시스템 드라이브(C:) 선택
  • "시스템 파일 정리" 클릭
  • 다음 항목들을 체크하여 삭제:
    • 이전 Windows 설치 (Windows.old)
    • Windows 업그레이드 로그 파일
    • 임시 Windows 설치 파일
    • Windows Update 정리
    • 임시 파일
2 디스크 오류 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
chkdsk C: /f /r
  • 재부팅 시 디스크 검사 실행 여부 묻는 메시지에서 "Y" 입력
  • 재부팅 후 자동으로 디스크 검사 실행

방법 6: 시스템 복원 실행

1 시스템 복원 지점 확인:
  • 제어판 → 시스템 및 보안 → 시스템 → 시스템 보호
  • "시스템 복원" 클릭
  • 업그레이드 시작 이전 날짜의 복원 지점 확인
2 시스템 복원 실행:
  • 적절한 복원 지점 선택
  • "영향을 받는 프로그램 검색" 클릭하여 영향 확인
  • "다음" 클릭하여 복원 시작
  • 복원 완료 후 시스템 재시작

방법 7: Windows 11 설치 미디어를 이용한 복구

1 Windows 11 설치 미디어 준비:
  • Microsoft 공식 사이트에서 Windows 11 ISO 다운로드
  • Media Creation Tool 사용하여 USB 설치 미디어 생성
  • 또는 ISO 파일을 DVD에 굽기
2 설치 미디어로 부팅:
  • 설치 미디어를 연결하고 시스템 재시작
  • BIOS/UEFI에서 부팅 순서 변경 (필요시)
  • Windows 설치 화면에서 "컴퓨터 복구" 선택
  • "문제 해결" → "고급 옵션" 선택
3 시작 복구 실행:
  • "시작 복구" 선택
  • 자동 진단 및 복구 과정 완료까지 대기
  • 복구 완료 후 정상 부팅 시도

방법 8: 인플레이스 업그레이드 재시도

1 시스템 안정화 후 재시도:
  • 앞의 모든 복구 작업 완료 후 시스템 안정성 확인
  • 불필요한 프로그램 및 서비스 중지
  • 바이러스 백신 프로그램 임시 비활성화
  • 충분한 디스크 공간 확보 (최소 20GB)
2 Windows 11 업그레이드 재실행:
  • Windows 11 설치 미디어에서 setup.exe 실행
  • "업데이트 다운로드 및 설치" 선택
  • "개인 파일 및 앱 유지" 옵션 선택
  • 안정적인 전원 공급 환경에서 업그레이드 진행
  • 업그레이드 중 시스템 사용 최소화

  • net start wuauserv: Windows Update 서비스 시작
  • sc query wuauserv: 서비스 상태 확인
  • rstrui.exe: 시스템 복원 실행

  • 시스템이 부팅되지 않는 경우

  • 데이터 손실 위험이 높은 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 업그레이드 전 시스템 백업 생성
  • 충분한 디스크 공간 확보 (최소 30GB 권장)
  • 안정적인 전원 공급 환경 구축 (UPS 사용 권장)
  • 업그레이드 중 다른 프로그램 실행 금지
  • 바이러스 백신 프로그램 임시 비활성화
  • 네트워크 연결 안정성 확인
  • 시스템 복원 지점 미리 생성

  • 이벤트 뷰어에서 상세한 오류 정보 확인
  • 업그레이드 로그 파일 분석으로 중단 원인 파악
  • 하드웨어 호환성 재확인 후 업그레이드 재시도
  • 업그레이드 전 시스템 성능 최적화
  • Windows Update Assistant 사용 고려

  • 0x80200055: UPGRADE_CANCELLED
  • 0x80200057: UPGRADE_FAILED
  • 0xC1900101: 일반적인 설치 실패
  • 0xC1900107: CLEANUP_PENDING
  • 0x80070070: 디스크 공간 부족
  • 0x8007000E: 메모리 부족

🛠️ 고급 진단 도구

업그레이드 중단 진단 도구

  • 이벤트 뷰어: eventvwr.msc - 업그레이드 관련 오류 로그
  • 시스템 정보: msinfo32 - 시스템 상태 확인
  • 리소스 모니터: resmon - 시스템 리소스 모니터링
  • 성능 모니터: perfmon - 시스템 성능 추적

PowerShell 진단 명령어

  • Get-WindowsUpdateLog: Windows Update 로그 생성
  • Get-WinEvent -LogName System | Where-Object {$_.Id -eq 1074}: 시스템 종료 로그
  • Get-ComputerInfo: 상세 시스템 정보
  • Test-ComputerSecureChannel: 시스템 무결성 확인

📊 업그레이드 중단 원인 분석

단계별 원인 확인 방법

  1. 중단 시점 확인: 업그레이드가 어느 단계에서 중단되었는지 파악
  2. 오류 로그 분석: 이벤트 뷰어에서 중단 시점의 오류 메시지 확인
  3. 하드웨어 상태 확인: 메모리, 디스크, 전원 공급 상태 점검
  4. 소프트웨어 충돌 확인: 백신 프로그램이나 타사 소프트웨어 간섭 여부
  5. 시스템 리소스 확인: 디스크 공간, 메모리 사용량 점검
  6. 네트워크 상태 확인: 인터넷 연결 안정성 및 속도 확인
