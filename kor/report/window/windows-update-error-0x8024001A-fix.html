Windows 0x8024001A WU_E_POLICY_NOT_SET 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 필요한 정책이 설정되지 않았을 때 발생하는 문제입니다. 그룹 정책 설정, 레지스트리 구성, 또는 시스템 정책 누락이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x8024001A (WU_E_POLICY_NOT_SET)는 Windows Update 시스템에서 필요한 정책이 설정되지 않았을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 관련 그룹 정책 미설정
  • 레지스트리의 Windows Update 정책 키 누락
  • WSUS 서버 정책 설정 오류
  • 자동 업데이트 정책 구성 문제
  • 업데이트 서버 위치 정책 미지정
  • 시스템 관리자에 의한 정책 제거
  • 도메인 정책과 로컬 정책 충돌
  • 손상된 정책 설정 파일
  • 그룹 정책 업데이트 실패
  • 기업 환경에서의 정책 배포 오류

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 정책 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 그룹 정책 설정 확인 및 구성

1 로컬 그룹 정책 편집기 실행 (Pro/Enterprise):
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들을 확인 및 설정:
  • - "자동 업데이트 구성" → "사용" 설정
  • - "인트라넷 Microsoft 업데이트 서비스 위치 지정" (필요시)
  • - "자동 업데이트를 즉시 설치할 수 있도록 허용"
2 자동 업데이트 정책 구성:
  • "자동 업데이트 구성" 정책을 더블클릭
  • "사용" 선택
  • 자동 업데이트 구성 옵션 선택:
  • - 옵션 2: 다운로드 및 설치 알림
  • - 옵션 3: 자동 다운로드 및 설치 알림
  • - 옵션 4: 자동 다운로드 및 설치 예약
  • "확인" 클릭하여 저장

방법 3: 레지스트리를 통한 정책 설정 (Home 버전)

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 정책 레지스트리 키 생성:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v AUOptions /t REG_DWORD /d 4 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v NoAutoUpdate /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v ScheduledInstallDay /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v ScheduledInstallTime /t REG_DWORD /d 3 /f
3 추가 Windows Update 정책 설정:
  • 다음 명령어들을 실행:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v AcceptTrustedPublisherCerts /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v DisableWindowsUpdateAccess /t REG_DWORD /d 0 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v ElevateNonAdmins /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v TargetGroup /t REG_SZ /d "WorkstationGroup" /f

방법 4: 그룹 정책 강제 업데이트

1 그룹 정책 캐시 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
rd /s /q "%WinDir%\System32\GroupPolicyUsers"
rd /s /q "%WinDir%\System32\GroupPolicy"
secedit /configure /cfg %windir%\inf\defltbase.inf /db defltbase.sdb /verbose
2 그룹 정책 강제 업데이트 및 적용:
  • 다음 명령어들을 실행:
gpupdate /force
gpupdate /target:computer /force
gpupdate /target:user /force
secedit /refreshpolicy machine_policy /enforce
secedit /refreshpolicy user_policy /enforce

방법 5: Windows Update 서비스 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 캐시 및 정책 데이터 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작 및 정책 적용:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow
gpupdate /force

방법 6: WSUS 서버 정책 설정 (기업 환경)

1 WSUS 서버 설정 확인:
  • 그룹 정책 편집기에서 다음 경로로 이동:
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • "인트라넷 Microsoft 업데이트 서비스 위치 지정" 정책 확인
  • WSUS 서버 URL이 올바르게 설정되어 있는지 확인
2 WSUS 서버 연결 테스트:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WSUS 서버 연결 테스트:
ping [WSUS서버주소]
telnet [WSUS서버주소] 8530
nslookup [WSUS서버주소]
wuauclt /detectnow /reportnow

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 정책 확인 및 업데이트 강제 실행:
  • 다음 명령어들을 실행:
Get-WUServiceManager
Get-WUSettings
Set-WUSettings -AcceptTrustedPublisherCerts
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -Verbose
Reset-WUComponents -Verbose

  • reg delete "키경로" /f: 레지스트리 키 삭제

정책 미설정 오류 해결 시 주의점

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 그룹 정책 변경 시 기업 환경에서는 IT 관리자와 상의하세요
  • WSUS 서버 설정 변경 시 네트워크 관리자와 협의하세요
  • 정책 설정 후 반드시 그룹 정책 업데이트를 실행하세요
  • 도메인 환경에서는 도메인 정책이 우선 적용됩니다
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 기업 환경에서 복잡한 도메인 정책이 적용된 경우
  • WSUS 서버 설정이 복잡하거나 문제가 있는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 그룹 정책 설정이 다른 시스템 기능에 영향을 주는 경우
  • Active Directory 환경에서 정책 배포 문제가 있는 경우
  • 시스템 전체 재설치가 필요한 경우

  • WSUS 서버 사용 시 서버 연결 상태를 정기적으로 확인하세요

  • 레지스트리 직접 수정은 Home 버전에서 유용합니다
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 정책 설정 검토를 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

🔍 관련 오류 코드

유사한 Windows Update 정책 관련 오류들

  • 0x80240019: 독점 설치 충돌 (WU_E_EXCLUSIVE_INSTALL_CONFLICT)
  • 0x80240018: 사용자 토큰 없음 (WU_E_NO_USERTOKEN)
  • 0x80240017: 적용 불가 (WU_E_NOT_APPLICABLE)
  • 0x80240016: 설치 허용되지 않음 (WU_E_INSTALL_NOT_ALLOWED)
  • 0x8024001B: 업데이트 유형 없음 (WU_E_NO_UPDATE_TYPE)
  • 0x8024001C: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)
  • 0x80070005: 액세스 거부됨

🛠️ 고급 해결 방법

그룹 정책 상태 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 적용된 정책 확인:
$WUPolicies = @(
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate",
    "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU"
)
foreach ($Policy in $WUPolicies) {
    if (Test-Path $Policy) {
        Write-Host "정책 발견: $Policy" -ForegroundColor Green
        Get-ItemProperty $Policy | Format-List
    } else {
        Write-Host "정책 누락: $Policy" -ForegroundColor Red
    }
}

Windows Update 정책 자동 설정 스크립트

  • 기본 정책 자동 설정:
$PolicyPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate"
$AUPath = "$PolicyPath\AU"
if (!(Test-Path $PolicyPath)) { New-Item -Path $PolicyPath -Force }
if (!(Test-Path $AUPath)) { New-Item -Path $AUPath -Force }
Set-ItemProperty -Path $AUPath -Name "AUOptions" -Value 4 -Type DWord
Set-ItemProperty -Path $AUPath -Name "NoAutoUpdate" -Value 0 -Type DWord
Set-ItemProperty -Path $AUPath -Name "ScheduledInstallDay" -Value 0 -Type DWord
Set-ItemProperty -Path $AUPath -Name "ScheduledInstallTime" -Value 3 -Type DWord
Write-Host "Windows Update 정책이 설정되었습니다." -ForegroundColor Green

🔄 정책 설정 가이드

Windows Update 정책 옵션 설명

AUOptions 값 설명 동작 권장 사용
1 자동 업데이트 비활성화 수동으로만 업데이트 테스트 환경
2 다운로드 및 설치 알림 사용자 확인 후 다운로드 개인 사용자
3 자동 다운로드, 설치 알림 자동 다운로드, 설치 확인 일반 사용자
4 자동 다운로드 및 설치 예약 완전 자동화 기업 환경
5 로컬 관리자 선택 허용 관리자가 설정 선택 혼합 환경

필수 정책 설정 항목

  • AUOptions: 자동 업데이트 동작 방식
  • NoAutoUpdate: 자동 업데이트 활성화/비활성화
  • ScheduledInstallDay: 예약 설치 요일 (0=매일)
  • ScheduledInstallTime: 예약 설치 시간 (24시간 형식)
  • AcceptTrustedPublisherCerts: 신뢰할 수 있는 게시자 인증서 허용

환경 권장 AUOptions WSUS 사용 추가 설정
