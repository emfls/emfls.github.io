Windows 11 0x80240002 WU_E_MAX_CAPACITY_REACHED 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 시스템에서 최대 용량에 도달했을 때 발생하는 오류입니다. 디스크 공간 부족이나 업데이트 캐시 오버플로우가 주요 원인입니다.

🔍 오류 개요

0x80240002 WU_E_MAX_CAPACITY_REACHED는 Windows 11에서 업데이트 시스템이 최대 용량 한계에 도달했을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 시스템 드라이브(C:) 디스크 공간 부족
  • Windows Update 캐시 폴더 오버플로우
  • 임시 파일 누적으로 인한 공간 부족
  • 업데이트 다운로드 폴더 용량 한계
  • 시스템 복원 지점 과다 생성
  • 페이지 파일 크기 부족
  • 배달 최적화 캐시 오버플로우
  • Windows.old 폴더 누적

🛠️ 해결 방법

방법 1: 디스크 공간 확보

1 디스크 정리 도구 실행:
  • Windows + R 키를 누르고 cleanmgr 입력
  • 시스템 드라이브(C:) 선택
  • 다음 항목들을 체크하여 삭제:
    • 임시 파일
    • 시스템 오류 메모리 덤프 파일
    • Windows Update 정리
    • 이전 Windows 설치 (Windows.old)
    • 휴지통
    • 임시 인터넷 파일
2 저장소 센스 활성화:
  • 설정 → 시스템 → 저장소
  • "저장소 센스" 활성화
  • "저장소 센스 구성" 클릭
  • 자동 정리 주기를 "매주" 또는 "매월"로 설정
  • "지금 정리" 클릭하여 즉시 정리

방법 2: Windows Update 캐시 정리

1 Windows Update 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop dosvc
2 업데이트 캐시 폴더 정리:
  • 다음 명령어로 캐시 폴더 내용 삭제:
del /f /s /q %windir%\SoftwareDistribution\Download\*.*
del /f /s /q %windir%\SoftwareDistribution\DataStore\*.*
rd /s /q %windir%\SoftwareDistribution\Download
rd /s /q %windir%\SoftwareDistribution\DataStore
3 서비스 재시작:
  • 다음 명령어들을 실행하여 서비스 재시작:
net start wuauserv
net start cryptSvc
net start bits
net start dosvc

방법 3: 배달 최적화 캐시 정리

1 배달 최적화 설정 확인:
  • 설정 → 업데이트 및 보안 → 배달 최적화
  • "고급 옵션" 클릭
  • 현재 캐시 사용량 확인
  • 필요시 캐시 크기 제한 설정
2 배달 최적화 캐시 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
net stop dosvc
del /f /s /q %windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache\*.*
net start dosvc

방법 4: 임시 파일 정리

1 시스템 임시 파일 삭제:
  • 다음 명령어들을 실행하여 임시 파일 삭제:
del /f /s /q %temp%\*.*
del /f /s /q %windir%\Temp\*.*
del /f /s /q %windir%\Prefetch\*.*
del /f /s /q %windir%\Logs\*.*
2 사용자 임시 파일 정리:
  • Windows + R 키를 누르고 %temp% 입력
  • 임시 파일 폴더에서 모든 파일 선택 후 삭제
  • 삭제할 수 없는 파일은 건너뛰기

방법 5: 시스템 복원 지점 정리

1 시스템 복원 설정 확인:
  • 제어판 → 시스템 및 보안 → 시스템 → 시스템 보호
  • "구성" 버튼 클릭
  • 현재 디스크 사용량 확인
  • 최대 사용량을 적절히 조정 (5-10%)
2 이전 복원 지점 삭제:
  • "삭제" 버튼 클릭
  • "가장 오래된 복원 지점을 제외한 모든 복원 지점 삭제" 선택
  • 또는 명령 프롬프트에서 다음 명령어 실행:
vssadmin delete shadows /all /quiet
wmic shadowcopy delete

방법 6: 페이지 파일 크기 조정

1 가상 메모리 설정 확인:
  • 제어판 → 시스템 및 보안 → 시스템 → 고급 시스템 설정
  • "고급" 탭 → "성능" 섹션의 "설정" 클릭
  • "고급" 탭 → "가상 메모리" 섹션의 "변경" 클릭
2 페이지 파일 크기 조정:
  • "모든 드라이브에 대한 페이징 파일 크기 자동 관리" 체크 해제
  • 시스템 드라이브 선택 후 "사용자 지정 크기" 선택
  • 초기 크기: RAM 크기 × 1.5
  • 최대 크기: RAM 크기 × 3
  • "설정" → "확인" 클릭 후 재부팅

방법 7: 디스크 압축 및 정리

1 시스템 파일 압축:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 시스템 파일 압축:
compact /c /s:C:\ /i /a
compact /c /s:C:\Windows /i /a
  • 압축 완료까지 기다리기 (시간 소요)
2 WinSxS 폴더 정리:
  • 다음 명령어로 구성 요소 저장소 정리:
DISM /Online /Cleanup-Image /StartComponentCleanup
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
DISM /Online /Cleanup-Image /SPSuperseded

방법 8: 외부 저장소 활용

1 업데이트 다운로드 위치 변경:
  • 외부 USB 드라이브 또는 추가 하드 드라이브 연결
  • 충분한 여유 공간 확보 (최소 20GB)
  • 임시로 업데이트 파일을 외부 저장소에 다운로드
2 사용자 폴더 이동:
  • 문서, 사진, 비디오 등 사용자 폴더를 외부 드라이브로 이동
  • 각 폴더 우클릭 → "속성" → "위치" 탭
  • "이동" 버튼 클릭하여 새 위치 지정
  • 시스템 드라이브 공간 확보

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 공간 확보량 재부팅 필요
디스크 공간 확보 쉬움 20-40분 5-20GB 아니오
Windows Update 캐시 정리 보통 15-30분 1-5GB 아니오
배달 최적화 캐시 정리 보통 10-20분 1-10GB 아니오
임시 파일 정리 쉬움 10-15분 1-3GB 아니오
시스템 복원 지점 정리 쉬움 5-10분 2-10GB 아니오
페이지 파일 크기 조정 보통 10-15분 가변적 예
디스크 압축 및 정리 어려움 60-180분 5-15GB 아니오
외부 저장소 활용 보통 30-60분 대량 아니오

🔧 주요 명령어 참조

디스크 정리 명령어

  • cleanmgr: 디스크 정리 도구
  • cleanmgr /sageset:1: 디스크 정리 설정
  • cleanmgr /sagerun:1: 자동 디스크 정리 실행
  • del /f /s /q %temp%\*.*: 임시 파일 삭제

시스템 정리 명령어

  • DISM /Online /Cleanup-Image /StartComponentCleanup: 구성 요소 정리
  • vssadmin delete shadows /all: 섀도우 복사본 삭제
  • compact /c /s:C:\ /i: 파일 압축
  • sfc /scannow: 시스템 파일 검사

⚠️ 주의사항

디스크 정리 시 주의점

  • 중요한 파일은 미리 백업하세요
  • 시스템 복원 지점 삭제 시 신중하게 결정하세요
  • 페이지 파일 크기 변경 후 시스템 성능을 모니터링하세요
  • 디스크 압축은 시스템 성능에 영향을 줄 수 있습니다
  • 외부 저장소 사용 시 안정적인 연결을 유지하세요
  • 정리 작업 중에는 다른 프로그램 사용을 자제하세요

언제 전문가의 도움이 필요한가?

  • 시스템 드라이브 용량이 극도로 부족한 경우
  • 하드웨어 업그레이드가 필요한 경우
  • 디스크 오류가 발생한 경우
  • 시스템 파일이 심각하게 손상된 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 디스크 정리 실행 (월 1회 이상)
  • 저장소 센스 자동 정리 기능 활성화
  • 시스템 드라이브에 최소 20% 여유 공간 유지
  • 불필요한 프로그램 정기적 제거
  • 대용량 파일은 외부 저장소에 보관
  • 시스템 복원 지점 개수 제한 설정
  • 임시 파일 자동 정리 설정

💡 추가 팁

  • 디스크 사용량 분석 도구로 공간 사용 현황 파악
  • 클라우드 저장소 활용으로 로컬 공간 절약
  • 중복 파일 제거 도구 사용
  • 휴면 파일(hiberfil.sys) 크기 조정 고려
  • 시스템 로그 파일 정기적 정리
  • 브라우저 캐시 및 쿠키 정기적 삭제

🔍 관련 오류 코드

유사한 용량 관련 Windows Update 오류들

  • 0x80070070: 디스크 공간이 부족합니다
  • 0x8007000E: 메모리가 부족합니다
  • 0x80240003: WU_E_UNKNOWN_ID
  • 0x80240004: WU_E_NOT_INITIALIZED
  • 0x80240005: WU_E_RANGEOVERLAP
  • 0x80240006: WU_E_TOOMANYRANGES

🛠️ 디스크 공간 분석 도구

Windows 내장 도구

  • 저장소 설정: 설정 → 시스템 → 저장소
  • 디스크 정리: cleanmgr
  • 리소스 모니터: resmon
  • 성능 모니터: perfmon

타사 도구

  • TreeSize: 폴더별 용량 분석
  • WinDirStat: 시각적 디스크 사용량 분석
  • CCleaner: 종합 시스템 정리
  • Disk Cleanup: 고급 디스크 정리

🔄 자동화 스크립트

배치 파일로 자동 정리

@echo off
echo Windows Update 캐시 정리 시작...
net stop wuauserv
net stop cryptSvc
net stop bits
del /f /s /q %windir%\SoftwareDistribution\Download\*.*
del /f /s /q %temp%\*.*
del /f /s /q %windir%\Temp\*.*
net start wuauserv
net start cryptSvc
net start bits
echo 정리 완료!
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 정기적으로 실행하여 공간 확보

이 가이드를 통해 Windows Update 용량 한계 문제가 해결되기를 바랍니다. 디스크 공간 관련 문제가 지속되면 하드웨어 업그레이드를 고려하시기 바랍니다.
