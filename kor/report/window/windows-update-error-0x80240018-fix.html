Windows 0x80240018 WU_E_NO_USERTOKEN 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 사용자 인증 토큰이 없거나 손상되었을 때 발생하는 문제입니다. 사용자 계정 권한, 인증 서비스, 또는 토큰 생성 과정에 문제가 있을 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240018 (WU_E_NO_USERTOKEN)는 Windows Update 시스템에서 사용자 인증 토큰을 찾을 수 없거나 유효하지 않을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 사용자 계정 인증 토큰 손상 또는 누락
  • Windows 로그온 서비스 문제
  • 사용자 계정 권한 설정 오류
  • 인증 관련 시스템 서비스 중단
  • 사용자 프로필 손상
  • 보안 토큰 서비스 (STS) 오류
  • 레지스트리 권한 설정 문제
  • Windows 로그온 세션 문제
  • 도메인 인증 문제 (기업 환경)
  • 시스템 파일 손상으로 인한 토큰 생성 실패

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 인증 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 사용자 계정 및 권한 확인

1 현재 사용자 계정 상태 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 사용자 정보 확인:
whoami /all
net user %username%
query user
wmic useraccount where name="%username%" get *
2 사용자 권한 및 그룹 멤버십 확인:
  • 다음 명령어로 권한 확인:
net localgroup administrators
net localgroup users
whoami /groups
whoami /priv

방법 3: 인증 관련 서비스 재시작

1 인증 및 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
net stop seclogon
net stop winlogon
2 사용자 토큰 관련 캐시 삭제:
  • 다음 명령어들을 실행하여 토큰 캐시 삭제:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %LOCALAPPDATA%\Microsoft\Windows\INetCache\*
del /f /s /q %APPDATA%\Microsoft\Windows\Cookies\*
3 서비스 재시작 및 토큰 재생성:
  • 다음 명령어들을 실행:
net start seclogon
net start winlogon
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 4: 사용자 프로필 복구

1 사용자 프로필 상태 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 프로필 상태 확인:
Get-WmiObject -Class Win32_UserProfile | Select-Object LocalPath, Loaded, Special
Get-ItemProperty "HKLM:\SOFTWARE\Microsoft\Windows NT\CurrentVersion\ProfileList\*" | Select-Object PSChildName, ProfileImagePath
whoami /logonid
2 사용자 프로필 권한 복구:
  • 다음 명령어로 프로필 권한 복구:
icacls %USERPROFILE% /reset /T /C
icacls %USERPROFILE% /grant %USERNAME%:(OI)(CI)F /T
icacls %APPDATA% /grant %USERNAME%:(OI)(CI)F /T
icacls %LOCALAPPDATA% /grant %USERNAME%:(OI)(CI)F /T

방법 5: 레지스트리 권한 및 토큰 설정 복구

1 레지스트리 백업:
  • Windows + R 키를 누르고 regedit 입력
  • 파일 → 내보내기로 전체 레지스트리 백업
  • 안전한 위치에 백업 파일 저장
2 Windows Update 및 인증 관련 레지스트리 키 재설정:
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v AUOptions /t REG_DWORD /d 4 /f
3 사용자 토큰 관련 레지스트리 권한 복구:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
$acl = Get-Acl "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate"
$accessRule = New-Object System.Security.AccessControl.RegistryAccessRule("Everyone","FullControl","Allow")
$acl.SetAccessRule($accessRule)
Set-Acl "HKLM:\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate" $acl

방법 6: 새 사용자 계정으로 테스트

1 임시 관리자 계정 생성:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 테스트 계정 생성:
net user TestAdmin Password123! /add
net localgroup administrators TestAdmin /add
net user TestAdmin /active:yes
2 새 계정으로 로그인 테스트:
  • 현재 세션에서 로그아웃
  • TestAdmin 계정으로 로그인
  • Windows Update 실행 테스트
  • 문제 해결 후 테스트 계정 삭제:
net user TestAdmin /delete

방법 7: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\seclogon.dll
sfc /scanfile=C:\Windows\System32\winlogon.exe
  • 스캔 완료 후 시스템 재부팅

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 사용자 토큰 확인 및 업데이트 강제 실행:
  • 다음 명령어들을 실행:
[Security.Principal.WindowsIdentity]::GetCurrent()
Get-WUServiceManager
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -ForceInstall -Verbose
Reset-WUComponents -Verbose

🔍 관련 오류 코드

유사한 Windows Update 인증 관련 오류들

  • 0x80070005: 액세스 거부됨
  • 0x80240017: 적용 불가 (WU_E_NOT_APPLICABLE)
  • 0x80240016: 설치 허용되지 않음 (WU_E_INSTALL_NOT_ALLOWED)
  • 0x80240019: 업데이트 소스 없음 (WU_E_NO_UPDATE_SOURCE)
  • 0x8024001A: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)
  • 0x8024001B: 업데이트 유형 없음 (WU_E_NO_UPDATE_TYPE)
  • 0x80240020: 업데이트 다운로더 없음 (WU_E_NO_UPDATE_DOWNLOADER)

🛠️ 고급 해결 방법

사용자 토큰 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 사용자 토큰 정보 확인:
$currentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
Write-Host "사용자 이름: $($currentUser.Name)"
Write-Host "인증 유형: $($currentUser.AuthenticationType)"
Write-Host "토큰: $($currentUser.Token)"
Write-Host "익명 여부: $($currentUser.IsAnonymous)"
Write-Host "인증 여부: $($currentUser.IsAuthenticated)"
Write-Host "게스트 여부: $($currentUser.IsGuest)"
Write-Host "시스템 여부: $($currentUser.IsSystem)"

사용자 권한 분석 스크립트

  • 사용자 권한 상세 분석:
$principal = New-Object Security.Principal.WindowsPrincipal($currentUser)
$isAdmin = $principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
Write-Host "관리자 권한: $isAdmin"
$groups = $currentUser.Groups | ForEach-Object {
    $_.Translate([Security.Principal.NTAccount])
}
Write-Host "그룹 멤버십:"
$groups | ForEach-Object { Write-Host "- $($_.Value)" }

🔄 사용자 인증 프로세스 이해

Windows 사용자 인증 단계

  1. 로그온 요청: 사용자가 로그인 정보 입력
  2. 자격 증명 검증: 시스템이 사용자 정보 확인
  3. 토큰 생성: 인증 성공 시 사용자 토큰 생성
  4. 세션 설정: 사용자 세션 및 환경 설정
  5. 권한 부여: 사용자 권한 및 그룹 멤버십 적용
  6. 프로필 로드: 사용자 프로필 및 설정 로드

토큰 관련 주요 구성 요소

  • SID (Security Identifier): 사용자 고유 식별자
  • 그룹 멤버십: 사용자가 속한 그룹 정보
  • 권한 (Privileges): 사용자에게 부여된 시스템 권한
  • ACL (Access Control List): 리소스 접근 권한

ping %LOGONSERVER%
