Windows 0x80240024 WU_E_NO_UPDATE 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 업데이트를 찾을 수 없거나 사용 가능한 업데이트가 없을 때 발생하는 문제입니다. 업데이트 검색 실패, 서버 연결 문제, 또는 시스템 상태 문제가 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240024 (WU_E_NO_UPDATE)는 Windows Update 시스템에서 사용 가능한 업데이트를 찾을 수 없을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • Windows Update 서버에서 업데이트를 찾을 수 없음
  • 시스템이 이미 최신 상태임
  • 업데이트 검색 프로세스 실패
  • Windows Update 서비스 연결 문제
  • 손상된 업데이트 메타데이터
  • 잘못된 시스템 날짜/시간 설정
  • 그룹 정책에 의한 업데이트 차단
  • WSUS 서버 설정 문제
  • 네트워크 연결 문제
  • Windows Update 캐시 손상

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 업데이트 검색 문제 진단:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id NetworkDiagnosticsWeb
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 시스템 날짜 및 시간 확인

1 시스템 날짜/시간 자동 설정:
  • 설정 → 시간 및 언어 → 날짜 및 시간
  • "시간 자동 설정" 활성화
  • "표준 시간대 자동 설정" 활성화
  • "지금 동기화" 클릭
2 명령어를 통한 시간 동기화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
w32tm /config /manualpeerlist:"time.windows.com,0x1" /syncfromflags:manual
w32tm /config /reliable:yes
w32tm /resync
net time \\time.windows.com /set /yes

방법 3: Windows Update 서비스 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 캐시 및 메타데이터 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작 및 업데이트 검색 재시도:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow
usoclient startscan

방법 4: 네트워크 연결 확인 및 최적화

1 Windows Update 서버 연결 테스트:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 서버 연결 확인:
ping windowsupdate.microsoft.com
ping update.microsoft.com
ping download.windowsupdate.com
nslookup windowsupdate.microsoft.com
telnet windowsupdate.microsoft.com 80
2 네트워크 설정 재설정:
  • 다음 명령어들을 순서대로 실행:
ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew
netsh winsock reset
netsh int ip reset
netsh winhttp reset proxy
  • 명령어 실행 후 시스템 재부팅

방법 5: 그룹 정책 설정 확인

1 로컬 그룹 정책 확인 (Pro/Enterprise):
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들이 "구성되지 않음" 또는 적절히 설정되어 있는지 확인:
  • - "자동 업데이트 구성"
  • - "인트라넷 Microsoft 업데이트 서비스 위치 지정"
  • - "자동 업데이트 사용 안 함"
2 레지스트리를 통한 정책 확인 (Home 버전):
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 정책 설정 확인:
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /s
reg query "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /s
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
gpupdate /force

방법 6: WSUS 설정 확인 및 수정

1 WSUS 서버 설정 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 WSUS 설정 확인:
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v WUStatusServer
reg query "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer
2 WSUS 설정 임시 비활성화:
  • WSUS 서버 연결에 문제가 있는 경우 임시 비활성화:
reg add "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate\AU" /v UseWUServer /t REG_DWORD /d 0 /f
net stop wuauserv
net start wuauserv
wuauclt /detectnow
  • ⚠️ 주의: 기업 환경에서는 IT 관리자와 상의 후 실행

방법 7: Microsoft Update 서비스 추가

1 Microsoft Update 서비스 활성화:
  • 설정 → 업데이트 및 보안 → Windows Update
  • "고급 옵션" 클릭
  • "Windows를 업데이트할 때 다른 Microsoft 제품에 대한 업데이트 받기" 활성화
2 PowerShell을 통한 Microsoft Update 서비스 추가:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어 실행:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
Get-WUServiceManager

방법 8: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
  • 스캔 완료 후 시스템 재부팅

⚠️ 주의사항

업데이트 없음 오류 해결 시 주의점

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 시스템 날짜/시간이 정확한지 확인하세요
  • 네트워크 연결이 안정적인지 확인하세요
  • WSUS 설정 변경 시 기업 환경에서는 IT 관리자와 상의하세요
  • 그룹 정책 변경 시 다른 시스템에 미치는 영향을 고려하세요
  • 실제로 업데이트가 필요한지 확인하세요
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 기업 환경에서 복잡한 WSUS 설정이 필요한 경우
  • 도메인 환경에서 그룹 정책 충돌이 있는 경우
  • 네트워크 인프라 문제가 의심되는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • Windows 버전이 지원 종료된 경우
  • 하드웨어 호환성 문제가 있는 경우

✅ 예방 방법

  • 정확한 시스템 날짜/시간 설정 유지
  • 안정적인 인터넷 연결 환경 유지
  • Windows Update 자동 설치 설정
  • 정기적인 수동 업데이트 확인
  • 시스템 복구 지점 정기 생성
  • 네트워크 설정 정기 확인
  • 그룹 정책 설정 정기 검토
  • 정기적인 시스템 파일 검사 (월 1회)
  • WSUS 서버 연결 상태 모니터링 (기업 환경)

💡 추가 팁

  • 업데이트 없음 오류는 실제로 시스템이 최신 상태일 수도 있습니다
  • 시스템 날짜/시간 오류가 가장 흔한 원인 중 하나입니다
  • 네트워크 연결 문제로 인해 업데이트 서버에 접근하지 못할 수 있습니다
  • 그룹 정책이나 WSUS 설정이 업데이트를 차단할 수 있습니다
  • Microsoft Update 서비스를 추가하면 더 많은 업데이트를 찾을 수 있습니다
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 Windows Update 상태 확인을 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

🔍 관련 오류 코드

유사한 Windows Update 검색 관련 오류들

  • 0x80240023: EULA 거부 (WU_E_EULAS_DECLINED)
  • 0x80240022: 모든 업데이트 실패 (WU_E_ALL_UPDATES_FAILED)
  • 0x80240021: 시간 초과 (WU_E_TIME_OUT)
  • 0x8024001F: 연결 없음 (WU_E_NO_CONNECTION)
  • 0x80240025: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)
  • 0x8024402C: 업데이트 서버에 연결할 수 없음
  • 0x80072EE2: 연결 시간 초과

🛠️ 고급 해결 방법

업데이트 검색 상태 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 업데이트 검색 상태 분석:
$UpdateSession = New-Object -ComObject Microsoft.Update.Session
$UpdateSearcher = $UpdateSession.CreateUpdateSearcher()
Write-Host "업데이트 검색 시작..." -ForegroundColor Yellow
try {
    $SearchResult = $UpdateSearcher.Search("IsInstalled=0")
    Write-Host "검색된 업데이트 수: $($SearchResult.Updates.Count)" -ForegroundColor Green
    foreach ($Update in $SearchResult.Updates) {
        Write-Host "- $($Update.Title)" -ForegroundColor Cyan
    }
} catch {
    Write-Host "업데이트 검색 실패: $($_.Exception.Message)" -ForegroundColor Red
}

시스템 상태 종합 진단 스크립트

  • 시스템 전반적인 상태 진단:
Write-Host "=== 시스템 상태 진단 ===" -ForegroundColor Green
$OSInfo = Get-ComputerInfo
Write-Host "OS: $($OSInfo.WindowsProductName)"
Write-Host "버전: $($OSInfo.WindowsVersion)"
Write-Host "빌드: $($OSInfo.WindowsBuildLabEx)"
$DateTime = Get-Date
Write-Host "현재 시간: $DateTime"
$Services = Get-Service -Name "wuauserv", "cryptSvc", "bits"
Write-Host "서비스 상태:"
$Services | Format-Table Name, Status, StartType
Test-NetConnection -ComputerName "windowsupdate.microsoft.com" -Port 80

🔄 업데이트 검색 프로세스 이해

Windows Update 검색 단계

  1. 서비스 시작: Windows Update 서비스 활성화
  2. 서버 연결: Microsoft Update 서버 또는 WSUS 서버 연결
  3. 시스템 정보 수집: 현재 시스템 상태 및 설치된 업데이트 확인
  4. 업데이트 검색: 사용 가능한 업데이트 목록 요청
  5. 호환성 확인: 시스템과의 호환성 검증
  6. 결과 반환: 적용 가능한 업데이트 목록 제공

업데이트 검색에 영향을 주는 요소

  • 시스템 날짜/시간: 정확한 날짜/시간 필요
  • 네트워크 연결: 안정적인 인터넷 연결
  • 서버 설정: Windows Update 또는 WSUS 서버 설정
  • 그룹 정책: 업데이트 관련 정책 설정
  • 시스템 상태: 현재 설치된 업데이트 및 시스템 버전

📊 업데이트 검색 문제 진단 매트릭스

네트워크 연결 문제 서버 연결 실패 인터넷 연결 불안정 네트워크 재설정
