Windows 11 위젯 완전히 제거하는 방법

🗑️ 위젯 완전 제거 보장: 이 가이드를 통해 Windows 11의 위젯 기능을 완전히 제거할 수 있습니다. 작업 표시줄 버튼 숨기기부터 서비스 비활성화, 시스템 파일 제거까지 단계별로 진행하여 시스템 성능 향상과 깔끔한 인터페이스를 구현할 수 있습니다.

📊 위젯 제거 방법별 비교

제거 방법 난이도 제거 수준 복구 가능성 성능 개선 안전성
작업 표시줄 버튼 숨기기 초급 UI만 숨김 쉬움 미미함 매우 안전
위젯 서비스 비활성화 중급 기능 중지 보통 중간 안전
레지스트리 편집 고급 시스템 차단 어려움 높음 주의 필요
PowerShell 스크립트 고급 완전 제거 어려움 최고 주의 필요
시스템 파일 제거 전문가 물리적 삭제 매우 어려움 최고 위험

🚀 단계별 위젯 제거 과정

1단계

기본 위젯 버튼 숨기기 (초급)

🎯 가장 안전한 방법: 위젯 기능을 완전히 제거하지 않고 작업 표시줄에서 버튼만 숨기는 방법입니다. 언제든지 쉽게 되돌릴 수 있어 초보자에게 권장합니다.

설정을 통한 위젯 버튼 숨기기:

  1. 작업 표시줄 빈 공간에서 우클릭
  2. "작업 표시줄 설정" 선택
  3. "위젯" 토글을 "끄기"로 변경
  4. 즉시 적용되어 위젯 버튼이 사라짐

대체 방법 - 설정 앱 사용:

  1. Windows + I 키로 설정 열기
  2. 개인 설정 → "작업 표시줄"
  3. "위젯" 항목을 끄기로 설정

PowerShell로 위젯 버튼 숨기기:

# PowerShell 관리자 권한으로 실행
# 위젯 버튼 숨기기
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 0

# 탐색기 재시작으로 즉시 적용
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

# 설정 확인
Get-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa"

위젯 버튼 다시 표시하기:

# 위젯 버튼 다시 표시
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 1

# 탐색기 재시작
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

이 방법의 장단점:

장점 단점
• 매우 안전함
• 쉽게 되돌릴 수 있음
• 시스템 안정성 유지		 • 위젯 서비스는 여전히 실행됨
• 메모리 사용량 변화 없음
• 완전한 제거가 아님

✅ 1단계 기본 숨기기 체크리스트




2단계

위젯 서비스 비활성화 (중급)

⚙️ 서비스 레벨 제거: 위젯 관련 서비스를 비활성화하여 시스템 리소스 사용을 줄이고 더 확실한 제거 효과를 얻을 수 있습니다. 메모리 사용량과 CPU 사용률이 개선됩니다.

위젯 관련 서비스 목록:

서비스명 표시명 기능 제거 안전도
cbdhsvc Clipboard User Service 위젯 데이터 처리 안전
WpnService Windows Push Notifications 위젯 알림 주의
lfsvc Geolocation Service 위치 기반 위젯 안전
WebClient WebClient 웹 콘텐츠 로딩 위험

서비스 관리자를 통한 비활성화:

  1. Windows + R → services.msc 입력
  2. "cbdhsvc" 서비스 찾기
  3. 서비스 더블클릭 → "중지" 버튼 클릭
  4. "시작 유형"을 "사용 안 함"으로 변경
  5. "적용" → "확인" 클릭

PowerShell로 서비스 일괄 비활성화:

# PowerShell 관리자 권한으로 실행
# 위젯 관련 서비스 비활성화

# Clipboard User Service 비활성화
Set-Service -Name "cbdhsvc" -StartupType Disabled
Stop-Service -Name "cbdhsvc" -Force -ErrorAction SilentlyContinue

# Geolocation Service 비활성화 (위젯 전용이 아니므로 주의)
Set-Service -Name "lfsvc" -StartupType Disabled
Stop-Service -Name "lfsvc" -Force -ErrorAction SilentlyContinue

# 서비스 상태 확인
Get-Service -Name "cbdhsvc", "lfsvc" | Select-Object Name, Status, StartType

위젯 프로세스 강제 종료:

# 위젯 관련 프로세스 확인
Get-Process | Where-Object {$_.ProcessName -like "*widget*" -or $_.ProcessName -like "*WebView*"}

# 위젯 프로세스 강제 종료
Stop-Process -Name "WidgetService" -Force -ErrorAction SilentlyContinue
Stop-Process -Name "WebViewHost" -Force -ErrorAction SilentlyContinue

# 메모리 사용량 확인
Get-Process | Sort-Object WorkingSet -Descending | Select-Object -First 10 Name, @{Name="Memory(MB)";Expression={[math]::Round($_.WorkingSet/1MB,2)}}

서비스 복원 방법:

# 서비스 다시 활성화
Set-Service -Name "cbdhsvc" -StartupType Manual
Start-Service -Name "cbdhsvc"

Set-Service -Name "lfsvc" -StartupType Manual
Start-Service -Name "lfsvc"

# 서비스 상태 확인
Get-Service -Name "cbdhsvc", "lfsvc"

⚠️ 서비스 비활성화 시 주의사항

  • 일부 서비스는 다른 기능에도 영향을 줄 수 있음
  • 시스템 복원 지점 생성 후 진행 권장
  • 변경 후 시스템 안정성 확인 필요
  • Windows 업데이트 시 서비스가 재활성화될 수 있음

✅ 2단계 서비스 비활성화 체크리스트





3단계

레지스트리 편집으로 완전 차단 (고급)

🚨 고급 사용자 전용: 레지스트리 편집은 시스템에 영구적인 변화를 가할 수 있습니다. 반드시 백업을 생성하고 충분한 지식이 있는 경우에만 진행하세요. 잘못된 편집은 시스템 불안정을 야기할 수 있습니다.

레지스트리 백업 생성:

  1. Windows + R → regedit 입력
  2. 파일 → "내보내기" 선택
  3. 전체 레지스트리 백업 생성
  4. 안전한 위치에 백업 파일 저장

위젯 관련 레지스트리 키:

레지스트리 경로 키 이름 값 효과
HKCU\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced TaskbarDa 0 위젯 버튼 숨김
HKLM\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds EnableFeeds 0 위젯 기능 차단
HKCU\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds EnableFeeds 0 사용자별 차단
HKLM\SOFTWARE\Microsoft\PolicyManager\current\device\NewsAndInterests AllowNewsAndInterests 0 정책 차단

PowerShell로 레지스트리 일괄 편집:

# PowerShell 관리자 권한으로 실행
# 레지스트리 백업 생성
$backupPath = "C:\Backup\Registry_Widgets_$(Get-Date -Format 'yyyyMMdd_HHmmss').reg"
reg export "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" $backupPath

# 위젯 완전 비활성화
# 작업 표시줄 위젯 버튼 제거
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 0

# 시스템 전체 위젯 비활성화
$policyPath = "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds"
if (!(Test-Path $policyPath)) {
New-Item -Path $policyPath -Force
}
Set-ItemProperty -Path $policyPath -Name "EnableFeeds" -Value 0

# 사용자별 위젯 비활성화
$userPolicyPath = "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds"
if (!(Test-Path $userPolicyPath)) {
New-Item -Path $userPolicyPath -Force
}
Set-ItemProperty -Path $userPolicyPath -Name "EnableFeeds" -Value 0

# 탐색기 재시작
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

그룹 정책을 통한 위젯 비활성화:

  1. Windows + R → gpedit.msc 입력
  2. 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소
  3. "뉴스 및 관심사" 폴더 찾기
  4. "작업 표시줄에서 뉴스 및 관심사 사용 안 함" 정책 활성화

레지스트리 복원 방법:

# 레지스트리 복원
# 백업 파일을 사용하여 복원
reg import "C:\Backup\Registry_Widgets_백업파일.reg"

# 또는 PowerShell로 개별 복원
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 1
Remove-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds" -Name "EnableFeeds" -ErrorAction SilentlyContinue
Remove-ItemProperty -Path "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds" -Name "EnableFeeds" -ErrorAction SilentlyContinue

# 탐색기 재시작
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

✅ 3단계 레지스트리 편집 체크리스트





4단계

PowerShell 스크립트로 완전 제거 (고급)

🔧 자동화된 완전 제거: PowerShell 스크립트를 사용하여 위젯 관련 모든 구성 요소를 자동으로 제거할 수 있습니다. 이 방법은 가장 효과적이지만 신중하게 사용해야 합니다.

종합 위젯 제거 스크립트:

# Windows 11 위젯 완전 제거 스크립트
# 관리자 권한 필요

Write-Host "Windows 11 위젯 완전 제거 시작..." -ForegroundColor Green

# 1. 백업 생성
$backupDir = "C:\WidgetRemoval_Backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"
New-Item -ItemType Directory -Path $backupDir -Force

# 레지스트리 백업
reg export "HKEY_CURRENT_USER\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" "$backupDir\TaskbarSettings.reg"
reg export "HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows" "$backupDir\WindowsPolicies.reg"

# 2. 위젯 서비스 중지 및 비활성화
$services = @("cbdhsvc", "lfsvc")
foreach ($service in $services) {
try {
Stop-Service -Name $service -Force -ErrorAction SilentlyContinue
Set-Service -Name $service -StartupType Disabled -ErrorAction SilentlyContinue
Write-Host "서비스 $service 비활성화 완료" -ForegroundColor Yellow
} catch {
Write-Host "서비스 $service 처리 실패: $($_.Exception.Message)" -ForegroundColor Red
}
}

# 3. 위젯 프로세스 종료
$processes = @("WidgetService", "WebViewHost", "msedgewebview2")
foreach ($process in $processes) {
Get-Process -Name $process -ErrorAction SilentlyContinue | Stop-Process -Force
}

# 4. 레지스트리 설정
# 작업 표시줄 위젯 버튼 제거
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 0

# 시스템 정책으로 위젯 비활성화
$policyPaths = @(
"HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds",
"HKCU:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds"
)

foreach ($path in $policyPaths) {
if (!(Test-Path $path)) {
New-Item -Path $path -Force | Out-Null
}
Set-ItemProperty -Path $path -Name "EnableFeeds" -Value 0
}

# 5. 위젯 관련 예약 작업 비활성화
$tasks = Get-ScheduledTask | Where-Object {$_.TaskName -like "*Widget*" -or $_.TaskName -like "*Feed*"}
foreach ($task in $tasks) {
Disable-ScheduledTask -TaskName $task.TaskName -ErrorAction SilentlyContinue
Write-Host "예약 작업 $($task.TaskName) 비활성화" -ForegroundColor Yellow
}

# 6. 탐색기 재시작
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

Write-Host "위젯 제거 완료! 백업 위치: $backupDir" -ForegroundColor Green
Write-Host "시스템 재부팅을 권장합니다." -ForegroundColor Cyan

위젯 제거 확인 스크립트:

# 위젯 제거 상태 확인 스크립트
Write-Host "=== Windows 11 위젯 제거 상태 확인 ===" -ForegroundColor Green

# 1. 작업 표시줄 설정 확인
$taskbarDa = Get-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -ErrorAction SilentlyContinue
if ($taskbarDa.TaskbarDa -eq 0) {
Write-Host "✓ 작업 표시줄 위젯 버튼: 숨김" -ForegroundColor Green
} else {
Write-Host "✗ 작업 표시줄 위젯 버튼: 표시됨" -ForegroundColor Red
}

# 2. 서비스 상태 확인
$services = @("cbdhsvc", "lfsvc")
foreach ($service in $services) {
$svc = Get-Service -Name $service -ErrorAction SilentlyContinue
if ($svc -and $svc.StartType -eq "Disabled") {
Write-Host "✓ 서비스 $service : 비활성화됨" -ForegroundColor Green
} else {
Write-Host "✗ 서비스 $service : 활성화됨" -ForegroundColor Red
}
}

# 3. 정책 설정 확인
$feedPolicy = Get-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds" -Name "EnableFeeds" -ErrorAction SilentlyContinue
if ($feedPolicy.EnableFeeds -eq 0) {
Write-Host "✓ 시스템 정책: 위젯 비활성화됨" -ForegroundColor Green
} else {
Write-Host "✗ 시스템 정책: 위젯 활성화됨" -ForegroundColor Red
}

# 4. 실행 중인 위젯 프로세스 확인
$widgetProcesses = Get-Process | Where-Object {$_.ProcessName -like "*widget*" -or $_.ProcessName -like "*WebView*"}
if ($widgetProcesses.Count -eq 0) {
Write-Host "✓ 위젯 프로세스: 실행되지 않음" -ForegroundColor Green
} else {
Write-Host "✗ 위젯 프로세스: $($widgetProcesses.Count)개 실행 중" -ForegroundColor Red
}

위젯 복원 스크립트:

# 위젯 기능 복원 스크립트
Write-Host "Windows 11 위젯 기능 복원 시작..." -ForegroundColor Green

# 1. 작업 표시줄 위젯 버튼 복원
Set-ItemProperty -Path "HKCU:\Software\Microsoft\Windows\CurrentVersion\Explorer\Advanced" -Name "TaskbarDa" -Value 1

# 2. 서비스 재활성화
$services = @("cbdhsvc", "lfsvc")
foreach ($service in $services) {
Set-Service -Name $service -StartupType Manual -ErrorAction SilentlyContinue
Start-Service -Name $service -ErrorAction SilentlyContinue
}

# 3. 정책 제거
Remove-ItemProperty -Path "HKLM:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds" -Name "EnableFeeds" -ErrorAction SilentlyContinue
Remove-ItemProperty -Path "HKCU:\SOFTWARE\Policies\Microsoft\Windows\Windows Feeds" -Name "EnableFeeds" -ErrorAction SilentlyContinue

# 4. 예약 작업 재활성화
$tasks = Get-ScheduledTask | Where-Object {$_.TaskName -like "*Widget*" -or $_.TaskName -like "*Feed*"}
foreach ($task in $tasks) {
Enable-ScheduledTask -TaskName $task.TaskName -ErrorAction SilentlyContinue
}

# 5. 탐색기 재시작
Stop-Process -Name "explorer" -Force
Start-Process "explorer"

Write-Host "위젯 기능 복원 완료!" -ForegroundColor Green

✅ 4단계 PowerShell 완전 제거 체크리스트





5단계

시스템 파일 제거 (전문가)

⚠️ 극도로 위험한 방법: 이 방법은 시스템 파일을 직접 수정하므로 매우 위험합니다. 시스템 불안정, 부팅 실패, 데이터 손실 등이 발생할 수 있으므로 전문가가 아니라면 절대 시도하지 마세요.

위젯 관련 시스템 파일 위치:

파일/폴더 위치 기능 제거 위험도
WebView2 C:\Program Files (x86)\Microsoft\EdgeWebView 위젯 렌더링 매우 높음
WidgetService.exe C:\Windows\System32 위젯 서비스 매우 높음
Windows.CloudStore.dll C:\Windows\System32 클라우드 데이터 매우 높음

안전한 파일 제거 절차:

  1. 완전한 시스템 백업: 전체 시스템 이미지 백업
  2. 복구 디스크 생성: Windows 복구 미디어 준비
  3. 안전 모드 부팅: 최소한의 서비스로 부팅
  4. 파일 소유권 변경: TrustedInstaller에서 사용자로
  5. 파일 이름 변경: 삭제 대신 .bak 확장자로 변경

파일 소유권 변경 및 제거:

# 극도로 위험한 작업 - 전문가만 실행
# 시스템 백업 필수!

# 1. 소유권 변경
takeown /f "C:\Windows\System32\WidgetService.exe" /a
icacls "C:\Windows\System32\WidgetService.exe" /grant administrators:F

# 2. 파일 이름 변경 (삭제 대신)
ren "C:\Windows\System32\WidgetService.exe" "WidgetService.exe.bak"

# 3. WebView2 폴더 이름 변경
takeown /f "C:\Program Files (x86)\Microsoft\EdgeWebView" /r /d y
icacls "C:\Program Files (x86)\Microsoft\EdgeWebView" /grant administrators:F /t
ren "C:\Program Files (x86)\Microsoft\EdgeWebView" "EdgeWebView.bak"

복구 방법:

# 파일 복구
ren "C:\Windows\System32\WidgetService.exe.bak" "WidgetService.exe"
ren "C:\Program Files (x86)\Microsoft\EdgeWebView.bak" "EdgeWebView"

# 또는 시스템 파일 검사기 사용
sfc /scannow

# DISM으로 시스템 이미지 복구
