Windows 11 0x80300024 DISK_OPERATION_NOT_SUPPORTED 해결 가이드

⚠️ 주의사항: 이 오류는 Windows 설치·복구·파티션 작업 중 디스크에서 지원하지 않는 연산을 시도할 때 발생합니다. 아래 해결 방법을 순서대로 시도하세요.

🔍 오류 개요

0x80300024 DISK_OPERATION_NOT_SUPPORTED 오류는 Windows 11 설치·복구 과정에서 디스크 파티션 형식이나 컨트롤러가 해당 작업을 지원하지 않을 때 발생합니다. 주요 원인은 다음과 같습니다:

  • MBR ↔ GPT 형식 불일치
  • UEFI/Legacy BIOS 모드 충돌
  • 드라이브가 동적 디스크(Dynamic Disk)로 설정됨
  • SATA 컨트롤러 모드(IDE·AHCI) 미호환
  • RAID·NVMe·USB 외장 드라이브 제한
  • 디스크 파티션 테이블 손상
  • 설치 미디어와 디스크 용량·섹터 크기 불일치
  • 스토리지 드라이버 누락

🛠️ 해결 방법

방법 1: BIOS/UEFI 부팅 모드 확인 및 일치

1 부팅 모드 확인:
  • 설치 USB 부팅 후 Shift + F10 → msinfo32 입력
  • "BIOS 모드"가 UEFI인지 Legacy인지 확인
2 디스크 파티션 형식 일치:
  • UEFI → GPT, Legacy → MBR 여야 설치가 진행됩니다.
  • 형식이 다르면 diskpart로 변환 (모든 데이터 삭제됨):
diskpart
list disk
select disk 0
clean
convert gpt (또는 convert mbr)
exit

방법 2: 동적 디스크 → 기본 디스크 변환

1 디스크 확인:
  • 설치 USB → Shift + F10 → diskpart
  • list disk에서 "Dyn" 표기된 디스크 확인
2 변환 절차:
  • 데이터 백업 필수 → 다시 clean → convert gpt/mbr
  • 또는 Windows 부팅 후 diskmgmt.msc → 동적 → 기본 디스크로 변환

방법 3: SATA 컨트롤러 모드 변경 (IDE ↔ AHCI)

1 BIOS 진입:
  • 재부팅 → F2, Del, F10 등으로 BIOS/UEFI 진입
2 스토리지 설정:
  • SATA Mode 를 AHCI 또는 IDE로 전환 후 저장
  • 전환 시 기존 OS 부팅 불가 가능성 → 레지스트리 사전 수정 필요

방법 4: 스토리지 전용 드라이버 로드

1 제조사 드라이버 준비:
  • Intel RST·AMD RAID·NVMe 드라이버를 USB에 복사
2 설치 화면에서 로드:
  • "Windows 설치 위치" 단계 → 드라이버 로드 클릭
  • USB 경로 선택 → 적용 → 디스크 인식 확인

방법 5: 디스크 오류 검사 및 복구

1 chkdsk 실행:
  • Shift + F10 → chkdsk C: /f /r
  • 배드 섹터 복구 후 재부팅
2 SMART 상태 확인:
  • BIOS 또는 CrystalDiskInfo 등으로 디스크 건강 확인

방법 6: 디스크 공간 확보 및 파티션 재구성

1 파티션 삭제·정리:
  • 설치 USB → 파티션 모두 삭제 → "할당되지 않은 공간" 선택
2 우회 설치:
  • 설치가 안 될 경우 다른 디스크·USB에 임시 설치 후 마이그레이션

방법 7: 디스크 초기화 & 새 파티션 테이블 작성

1 diskpart 초기화:
diskpart
list disk
select disk 0
clean
convert gpt (또는 convert mbr)
create partition primary size=100000
format fs=ntfs quick
exit
2 설치 진행:
  • 초기화 후 설치 진행 → 오류 해결 여부 확인

방법 8: 하드웨어 점검 및 교체

1 케이블·슬롯 점검:
  • SATA·전원 케이블 재장착
  • M.2 NVMe 방열판·슬롯 변경
2 디스크 교체 테스트:
  • 다른 SSD/HDD로 설치 진행 → 성공 시 기존 디스크 불량 의심

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 데이터 손실 위험
BIOS/UEFI 부팅 모드 일치 쉬움 10–15분 높음 낮음
동적 → 기본 디스크 변환 보통 15–25분 높음 높음
SATA 모드 변경 보통 10–20분 보통 중간
전용 드라이버 로드 쉬움 10–15분 높음 낮음
디스크 오류 검사 쉬움 30–60분 보통 낮음
파티션 재구성 보통 20–40분 높음 높음
디스크 초기화 어려움 20–30분 매우 높음 높음
하드웨어 교체 어려움 30–60분 매우 높음 중간

🔧 주요 명령어 참조

디스크 관리 명령어

  • diskpart → list disk / clean / convert gpt
  • chkdsk C: /f /r – 디스크 오류 검사
  • wmic diskdrive get status – SMART 상태 확인
  • mbr2gpt /convert /allowfullos – 부팅 가능 상태에서 MBR → GPT 변환

설치 환경 명령어

  • msinfo32 – BIOS 모드·부팅 정보 확인
  • bcdedit /enum all – 부트 구성 점검
  • wpeutil updatebootinfo – WinPE 부트 정보 갱신

⚠️ 주의사항

  • 파티션·디스크 초기화 전 중요한 데이터 반드시 백업
  • UEFI/Legacy 부팅 모드 변경 시 기존 OS가 부팅되지 않을 수 있음
  • SATA 모드 전환 전 레지스트리 AHCI 드라이버 활성화 필요
  • 동적 디스크를 기본 디스크로 변환하면 모든 볼륨 삭제
  • 드라이버 로드 시 제조사 호환 버전 사용

전문가 도움이 필요한 상황

  • 디스크 SMART 오류·배드 섹터 다수 발견
  • RAID·NVMe 복잡한 스토리지 구성
  • 파티션 테이블이 완전히 손상된 경우
  • 기업 환경에서 데이터 복구가 필요한 경우

✅ 예방 방법

  • 정기적인 디스크 건강 점검(SMART 모니터링)
  • UEFI·GPT 환경 유지(Windows 11 권장)
  • 스토리지 드라이버 최신 상태 유지
  • OS 설치·업그레이드 전 전체 백업
  • 정전 방지를 위한 UPS 사용
  • 설치 USB · ISO 최신 이미지 사용
