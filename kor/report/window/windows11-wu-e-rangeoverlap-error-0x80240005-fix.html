Windows 11 0x80240005 WU_E_RANGEOVERLAP 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 다운로드 과정에서 데이터 범위가 중복될 때 발생하는 오류입니다. 네트워크 불안정이나 다운로드 충돌이 주요 원인입니다.

🔍 오류 개요

0x80240005 WU_E_RANGEOVERLAP는 Windows 11에서 업데이트 다운로드 중 데이터 범위가 중복되거나 충돌할 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 네트워크 연결 불안정으로 인한 패킷 중복
  • 다운로드 범위 설정 오류
  • BITS(Background Intelligent Transfer Service) 충돌
  • 배달 최적화 서비스 오류
  • 부분 다운로드 파일 충돌
  • 프록시 서버 설정 문제
  • 방화벽 또는 보안 프로그램 간섭
  • 동시 다운로드 프로세스 충돌

🛠️ 해결 방법

방법 1: BITS 서비스 재시작 및 캐시 정리

1 BITS 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop bits
net stop wuauserv
net stop dosvc
net stop cryptSvc
2 BITS 작업 큐 정리:
  • 다음 명령어로 BITS 작업 큐 완전 정리:
bitsadmin /reset /allusers
bitsadmin /util /purge
del /f /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader\qmgr*.dat
3 서비스 재시작:
  • 다음 명령어들을 순서대로 실행:
net start cryptSvc
net start bits
net start wuauserv
net start dosvc

방법 2: 배달 최적화 서비스 재설정

1 배달 최적화 설정 확인:
  • 설정 → 업데이트 및 보안 → 배달 최적화
  • "다른 PC에서 다운로드 허용" 임시 비활성화
  • "고급 옵션" → "다운로드 설정" 확인
  • 대역폭 제한을 "무제한"으로 설정
2 배달 최적화 캐시 삭제:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
net stop dosvc
rd /s /q "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
md "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
net start dosvc

방법 3: Windows Update 다운로드 폴더 재설정

1 다운로드 폴더 정리:
  • 서비스 중지 후 다음 명령어 실행:
del /f /s /q %windir%\SoftwareDistribution\Download\*.*
rd /s /q %windir%\SoftwareDistribution\Download
md %windir%\SoftwareDistribution\Download
2 다운로드 폴더 권한 재설정:
  • 다음 명령어로 적절한 권한 설정:
icacls %windir%\SoftwareDistribution\Download /grant "NT AUTHORITY\SYSTEM":(OI)(CI)F
icacls %windir%\SoftwareDistribution\Download /grant "NT AUTHORITY\LOCAL SERVICE":(OI)(CI)F
icacls %windir%\SoftwareDistribution\Download /grant "NT AUTHORITY\NETWORK SERVICE":(OI)(CI)F

방법 4: 네트워크 연결 최적화

1 네트워크 어댑터 재설정:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew
netsh winsock reset
netsh int ip reset
2 네트워크 연결 안정화:
  • Wi-Fi 사용 시 유선 연결로 변경 (가능한 경우)
  • 네트워크 드라이버 최신 버전으로 업데이트
  • 라우터/모뎀 재시작
  • 다른 네트워크 활동 최소화

방법 5: 프록시 및 방화벽 설정 확인

1 프록시 설정 해제:
  • 설정 → 네트워크 및 인터넷 → 프록시
  • "자동으로 프록시 설정 검색" 끄기
  • "설정 스크립트 사용" 끄기
  • "수동 프록시 설정" 끄기
  • 또는 명령어로 프록시 설정 초기화:
netsh winhttp reset proxy
netsh winhttp import proxy source=ie
2 방화벽 예외 설정:
  • Windows 보안 → 방화벽 및 네트워크 보호
  • "앱이 방화벽을 통과하도록 허용" 클릭
  • 다음 항목들이 체크되어 있는지 확인:
    • Windows Update
    • Background Intelligent Transfer Service
    • Delivery Optimization

방법 6: 단일 스레드 다운로드 강제 설정

1 레지스트리 수정으로 다운로드 최적화:
  • Windows + R 키를 누르고 regedit 입력
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\DeliveryOptimization\Config
  • 새 DWORD(32비트) 값 생성: "DODownloadMode"
  • 값을 "0"으로 설정 (HTTP만 사용)
2 BITS 동시 연결 제한:
  • 다음 경로로 이동:
HKEY_LOCAL_MACHINE\SOFTWARE\Policies\Microsoft\Windows\BITS
  • 새 DWORD 값 생성: "MaxBandwidthValidFrom"
  • 값을 "0"으로 설정
  • 새 DWORD 값 생성: "MaxBandwidthValidTo"
  • 값을 "24"로 설정

방법 7: Windows Update 문제 해결사 실행

1 자동 문제 해결사 실행:
  • 설정 → 시스템 → 문제 해결 → 기타 문제 해결사
  • "Windows Update" 항목에서 "실행" 클릭
  • 문제 해결사가 범위 중복 문제 자동 진단
  • 발견된 문제에 대한 수정 사항 적용
2 네트워크 문제 해결사 실행:
  • 설정 → 네트워크 및 인터넷 → 상태
  • "네트워크 문제 해결사" 실행
  • 네트워크 연결 문제 자동 진단 및 수정

방법 8: 수동 업데이트 다운로드

1 Microsoft Update 카탈로그 사용:
  • 웹 브라우저에서 Microsoft Update 카탈로그 접속
  • 실패한 업데이트의 KB 번호 검색
  • 시스템 아키텍처에 맞는 업데이트 다운로드
  • 다운로드한 파일을 관리자 권한으로 실행
2 오프라인 업데이트 설치:
  • 안정적인 네트워크 환경에서 업데이트 다운로드
  • USB 드라이브 등을 통해 문제 시스템으로 전송
  • 오프라인 상태에서 업데이트 설치
  • 설치 완료 후 온라인 연결하여 확인

📋 문제 해결 체크리스트

해결 방법 난이도 소요 시간 성공률 재부팅 필요
BITS 서비스 재시작 보통 15-25분 매우 높음 예
배달 최적화 재설정 쉬움 10-20분 높음 아니오
다운로드 폴더 재설정 보통 15-20분 높음 아니오
네트워크 연결 최적화 보통 20-30분 보통 예
프록시/방화벽 설정 쉬움 10-15분 보통 아니오
단일 스레드 다운로드 설정 어려움 15-25분 높음 예
Windows Update 문제 해결사 쉬움 15-20분 보통 예
수동 업데이트 다운로드 보통 30-60분 매우 높음 예

🔧 주요 명령어 참조

BITS 관련 명령어

  • bitsadmin /reset /allusers: 모든 BITS 작업 재설정
  • bitsadmin /util /purge: BITS 큐 정리
  • bitsadmin /list /allusers /verbose: BITS 작업 목록 확인
  • net stop bits && net start bits: BITS 서비스 재시작

네트워크 관련 명령어

  • netsh winsock reset: Winsock 재설정
  • netsh int ip reset: IP 설정 재설정
  • netsh winhttp reset proxy: 프록시 설정 재설정
  • ipconfig /flushdns: DNS 캐시 삭제

⚠️ 주의사항

네트워크 설정 수정 시 주의점

  • 네트워크 설정 변경 전 현재 설정을 기록해두세요
  • 레지스트리 수정 전 반드시 백업을 생성하세요
  • 프록시 설정 해제 시 회사 네트워크 정책을 확인하세요
  • 방화벽 설정 변경 시 보안에 주의하세요
  • BITS 작업 삭제 시 다른 다운로드에 영향을 줄 수 있습니다
  • 네트워크 재설정 후 연결 상태를 확인하세요

언제 전문가의 도움이 필요한가?

  • 네트워크 인프라 문제가 의심되는 경우
  • 기업 환경에서 프록시 서버 문제가 발생한 경우
  • 하드웨어 네트워크 어댑터 문제가 의심되는 경우
  • ISP(인터넷 서비스 제공업체) 문제가 의심되는 경우
  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우

✅ 예방 방법

  • 정기적인 BITS 서비스 상태 점검
  • 배달 최적화 설정 최적화
  • 네트워크 드라이버 최신 상태 유지
  • 동시 다운로드 프로그램 사용 자제
  • 충분한 네트워크 대역폭 확보
  • 정기적인 네트워크 설정 점검

💡 추가 팁

  • 업데이트 다운로드 시 다른 네트워크 활동 최소화
  • 네트워크 트래픽이 적은 시간대에 업데이트 실행
  • Wi-Fi 연결보다 유선 연결 사용 권장
  • 라우터 QoS 설정으로 업데이트 우선순위 조정
  • 네트워크 모니터링 도구로 연결 상태 확인
  • 배달 최적화 설정을 네트워크 환경에 맞게 조정

🛠️ 고급 진단 도구

네트워크 진단 도구

  • 네트워크 문제 해결사: 설정 → 네트워크 및 인터넷 → 상태
  • 리소스 모니터: resmon - 네트워크 활동 모니터링
  • 성능 모니터: perfmon - 네트워크 성능 확인
  • 이벤트 뷰어: eventvwr - BITS 관련 오류 로그

PowerShell 진단 명령어

  • Get-BitsTransfer: 현재 BITS 전송 작업 확인
  • Test-NetConnection update.microsoft.com -Port 443: 서버 연결 테스트
  • Get-NetAdapter: 네트워크 어댑터 상태 확인
  • Get-DeliveryOptimizationStatus: 배달 최적화 상태 확인

📊 네트워크 최적화 설정

권장 네트워크 설정

설정 항목 권장 값 설명
배달 최적화 모드 HTTP만 P2P 비활성화로 충돌 방지
BITS 최대 대역폭 무제한 다운로드 속도 최적화
동시 연결 수 2-4개 범위 중복 방지
프록시 설정 비활성화 직접 연결로 충돌 방지

BITS 및 다운로드 재설정 스크립트

@echo off
echo BITS 서비스 및 다운로드 재설정 시작...
net stop bits

net stop dosvc

bitsadmin /reset /allusers
bitsadmin /util /purge

del /f /s /q %windir%\SoftwareDistribution\Download\*.*
rd /s /q "%windir%\ServiceProfiles\NetworkService\AppData\Local\Microsoft\Windows\DeliveryOptimization\Cache"
echo 서비스 재시작 중...
net start bits
net start wuauserv
net start dosvc
echo 네트워크 재설정 중...
ipconfig /flushdns
netsh winsock reset
echo 재설정 완료! 시스템을 재시작하세요.
pause
  • 위 내용을 메모장에 복사하여 .bat 파일로 저장
  • 관리자 권한으로 실행
  • 스크립트 완료 후 시스템 재시작
