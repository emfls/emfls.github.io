Windows 0x80240028 WU_E_UNINSTALL_NOT_ALLOWED 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 특정 업데이트의 제거가 허용되지 않을 때 발생하는 문제입니다. 시스템 정책, 보안 제한, 또는 중요한 시스템 업데이트 보호로 인해 제거가 차단될 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240028 (WU_E_UNINSTALL_NOT_ALLOWED)는 Windows Update 시스템에서 특정 업데이트의 제거가 허용되지 않을 때 발생하는 오류입니다. 이 오류는 주로 다음과 같은 원인으로 발생합니다:

  • 시스템 보안 정책에 의한 업데이트 제거 제한
  • 중요한 보안 업데이트 보호 메커니즘
  • 그룹 정책에 의한 업데이트 제거 금지
  • 관리자 권한 부족
  • 기업 환경의 WSUS 정책 제한
  • Windows 시스템 파일 보호 기능
  • 업데이트 종속성으로 인한 제거 불가
  • 손상된 업데이트 메타데이터
  • 레지스트리 권한 설정 문제
  • 타사 보안 소프트웨어에 의한 차단

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 고급 문제 해결사 및 권한 검사:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어 실행:
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic
msdt.exe -id WindowsUpdateDiagnostic -advanced

방법 2: 관리자 권한 확인 및 설정

1 현재 사용자 권한 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 권한 확인:
whoami /all
net user %username%
net localgroup administrators
whoami /priv
2 UAC 설정 확인 및 조정:
  • 제어판 → 사용자 계정 → 사용자 계정 제어 설정 변경
  • UAC 슬라이더를 적절한 수준으로 조정
  • 또는 관리자 권한으로 명령 프롬프트에서 다음 명령어 실행:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v EnableLUA /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System" /v ConsentPromptBehaviorAdmin /t REG_DWORD /d 2 /f

방법 3: 그룹 정책 설정 확인 및 수정

1 로컬 그룹 정책 편집기 실행 (Pro/Enterprise):
  • Windows + R 키를 누르고 gpedit.msc 입력
  • 컴퓨터 구성 → 관리 템플릿 → Windows 구성 요소 → Windows Update
  • 다음 정책들을 확인 및 설정:
  • - "사용자가 업데이트를 제거하지 못하도록 방지" → "구성되지 않음"
  • - "업데이트 제거 액세스 제거" → "구성되지 않음"
  • - "자동 업데이트 구성" → 적절히 설정
2 레지스트리를 통한 정책 재설정 (Home 버전):
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v DisableWindowsUpdateAccess /f
reg delete "HKLM\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /v SetDisableUXWUAccess /f
reg delete "HKCU\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" /f
gpupdate /force

방법 4: 업데이트 제거 권한 복구

1 Windows Update 서비스 권한 확인:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어로 서비스 권한 확인:
sc query wuauserv
sc qc wuauserv
sc sdshow wuauserv
icacls C:\Windows\SoftwareDistribution
2 업데이트 폴더 권한 복구:
  • 다음 명령어로 폴더 권한 복구:
takeown /f C:\Windows\SoftwareDistribution /r /d y
icacls C:\Windows\SoftwareDistribution /grant administrators:F /t
icacls C:\Windows\SoftwareDistribution /grant %username%:F /t
icacls C:\Windows\System32\catroot2 /grant administrators:F /t

방법 5: 특정 업데이트 강제 제거

1 설치된 업데이트 목록 확인:
  • 관리자 권한으로 PowerShell 실행
  • 다음 명령어로 설치된 업데이트 확인:
Get-HotFix | Sort-Object InstalledOn -Descending | Select-Object -First 20
dism /online /get-packages | findstr "Install"
wmic qfe list brief /format:table
2 DISM을 이용한 업데이트 강제 제거:
  • 다음 명령어로 특정 업데이트 제거:
dism /online /get-packages
dism /online /remove-package /packagename:[패키지명] /quiet
wusa /uninstall /kb:[KB번호] /quiet /norestart
shutdown /r /t 60
3 PowerShell을 이용한 업데이트 제거:
  • 다음 명령어로 PowerShell 모듈 사용:
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Get-WUHistory | Where-Object {$_.Title -like "*[검색어]*"}
Remove-WindowsUpdate -KBArticleID "KB[번호]" -Confirm:$false

방법 6: 안전 모드에서 업데이트 제거

1 안전 모드로 부팅:
  • 시작 메뉴 → 전원 → Shift 키를 누른 상태에서 "다시 시작" 클릭
  • 문제 해결 → 고급 옵션 → 시작 설정
  • "다시 시작" 클릭 후 F4 키 눌러 안전 모드 진입
  • 또는 명령어 사용:
bcdedit /set {default} safeboot minimal
shutdown /r /t 0
2 안전 모드에서 업데이트 제거:
  • 안전 모드에서 제어판 → 프로그램 → 설치된 업데이트 보기
  • 제거하려는 업데이트 선택 후 제거
  • 또는 명령어 사용:
wusa /uninstall /kb:[KB번호] /quiet /norestart
dism /online /remove-package /packagename:[패키지명]
  • 완료 후 정상 모드로 복귀:
bcdedit /deletevalue {default} safeboot
shutdown /r /t 0

방법 7: 시스템 복원 지점 사용

1 시스템 복원 지점 확인:
  • 제어판 → 시스템 → 시스템 보호
  • "시스템 복원" 클릭
  • 문제가 발생하기 전 시점의 복원 지점 선택
  • 또는 명령어로 확인:
vssadmin list shadows
wmic shadowcopy list brief
rstrui.exe
2 시스템 복원 실행:
  • 선택한 복원 지점으로 시스템 복원
  • 복원 과정에서 영향받을 프로그램 확인
  • 복원 완료 후 시스템 재부팅
  • 복원 후 업데이트 상태 확인

방법 8: DISM 및 SFC 시스템 복구

1 DISM 시스템 이미지 복구:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
DISM /Online /Cleanup-Image /CheckHealth
DISM /Online /Cleanup-Image /ScanHealth
DISM /Online /Cleanup-Image /RestoreHealth /Source:C:\Windows\WinSxS /LimitAccess
DISM /Online /Cleanup-Image /StartComponentCleanup /ResetBase
2 SFC 시스템 파일 검사 및 복구:
  • DISM 명령 완료 후 다음 명령어 실행:
sfc /scannow
sfc /verifyonly
sfc /scanfile=C:\Windows\System32\wuaueng.dll
sfc /scanfile=C:\Windows\System32\wuapi.dll
sfc /scanfile=C:\Windows\System32\wups.dll
  • 스캔 완료 후 시스템 재부팅

🔧 주요 명령어 참조

권한 확인 관련 명령어

  • whoami /all: 현재 사용자 정보 및 권한 확인
  • net localgroup administrators: 관리자 그룹 멤버 확인
  • whoami /priv: 사용자 권한 목록 확인
  • icacls [경로]: 파일/폴더 권한 확인

업데이트 제거 관련 명령어

  • wusa /uninstall /kb:[KB번호]: 특정 업데이트 제거
  • dism /online /remove-package: DISM을 이용한 패키지 제거
  • Get-HotFix: 설치된 업데이트 목록 확인
  • Remove-WindowsUpdate: PowerShell 업데이트 제거

정책 관리 관련 명령어

  • gpedit.msc: 로컬 그룹 정책 편집기
  • gpupdate /force: 그룹 정책 강제 업데이트
  • reg delete "키경로" /f: 레지스트리 키 삭제
  • secedit /configure: 보안 정책 재설정

⚠️ 주의사항

업데이트 제거 허용되지 않음 오류 해결 시 주의점

  • 시스템 복구 지점을 미리 생성하세요
  • 중요한 데이터는 백업 후 작업하세요
  • 보안 업데이트 제거 시 시스템 보안 위험을 고려하세요
  • 그룹 정책 변경 시 기업 환경에서는 IT 관리자와 상의하세요
  • 관리자 권한으로 모든 작업을 수행하세요
  • 안전 모드 작업 시 네트워크 연결을 확인하세요
  • 시스템 복원 시 최근 설치된 프로그램이 제거될 수 있습니다
  • DISM 및 SFC 스캔은 시간이 오래 걸릴 수 있습니다

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 기업 환경에서 복잡한 도메인 정책이 적용된 경우
  • 중요한 보안 업데이트 제거로 인한 보안 위험이 있는 경우
  • 시스템 파일 손상이 심각하여 복구가 불가능한 경우
  • 업데이트 제거 후 시스템 불안정이 발생하는 경우
  • WSUS 서버 정책과 충돌하는 경우
  • 시스템 전체 재설치가 필요한 경우

✅ 예방 방법

  • 정기적인 시스템 복구 지점 생성
  • 업데이트 설치 전 시스템 백업
  • 적절한 관리자 권한 관리
  • 그룹 정책 설정 정기 검토
  • Windows Update 자동 설치 설정
  • 보안 소프트웨어 정기 업데이트
  • 정기적인 시스템 파일 검사 (월 1회)
  • 업데이트 설치 후 시스템 안정성 확인
  • 중요한 업데이트는 신중히 검토 후 제거

💡 추가 팁

  • 중요한 보안 업데이트는 시스템 보호를 위해 제거가 제한될 수 있습니다
  • 기업 환경에서는 IT 정책에 따라 제거가 제한될 수 있습니다
  • 안전 모드에서는 일부 보안 제한이 우회될 수 있습니다
  • 시스템 복원은 업데이트 제거의 안전한 대안입니다
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 권한 설정 검토를 수행하세요
  • 업데이트 제거 시 시스템 안정성을 모니터링하세요

🛠️ 고급 해결 방법

업데이트 제거 권한 검증 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 현재 제거 권한 상태 확인:
$currentUser = [Security.Principal.WindowsIdentity]::GetCurrent()
$principal = New-Object Security.Principal.WindowsPrincipal($currentUser)
$isAdmin = $principal.IsInRole([Security.Principal.WindowsBuiltInRole]::Administrator)
Write-Host "관리자 권한: $isAdmin"
$policies = Get-ItemProperty "HKLM:\SOFTWARE\Policies\Microsoft\Windows\WindowsUpdate" -ErrorAction SilentlyContinue
if ($policies) {
    Write-Host "Windows Update 정책이 적용되어 있습니다."
    $policies | Format-List
} else {
    Write-Host "Windows Update 정책이 설정되지 않았습니다."
}

제거 가능한 업데이트 목록 확인

  • 제거 가능한 업데이트 필터링:
$updates = Get-HotFix | Sort-Object InstalledOn -Descending
$removableUpdates = @()
foreach ($update in $updates) {
    try {
        $result = wusa /uninstall /kb:$($update.HotFixID.Replace("KB","")) /quiet /norestart /log:C:\temp\test.log
        if ($LASTEXITCODE -eq 0) {
            $removableUpdates += $update
        }
    } catch {
        Write-Host "제거 불가: $($update.HotFixID)"
    }
}
$removableUpdates | Format-Table HotFixID, Description, InstalledOn

🔄 업데이트 제거 정책 이해

Windows Update 제거 제한 정책

  • 시스템 보안 정책: 중요한 보안 업데이트 보호
  • 그룹 정책: 조직 차원의 업데이트 관리
  • WSUS 정책: 기업 환경의 중앙 관리
  • 사용자 권한: 관리자 권한 요구사항
  • 종속성 보호: 다른 구성 요소에 필요한 업데이트

제거 가능 여부 판단 기준

  • 보안 등급: 중요/긴급 보안 업데이트는 제거 제한
  • 시스템 안정성: 시스템 핵심 구성 요소 업데이트
  • 종속성: 다른 업데이트나 프로그램의 종속성
  • 정책 설정: 관리자 또는 조직 정책
  • 업데이트 유형: 서비스 팩, 누적 업데이트 등

📊 업데이트 제거 권한 매트릭스

업데이트 유형 제거 가능성 필요 권한 주의사항
선택적 업데이트 높음 사용자 기능 손실 가능
일반 업데이트 보통 관리자 시스템 안정성 확인
보안 업데이트 낮음 관리자 보안 위험 증가
중요 보안 업데이트 매우 낮음 시스템 관리자 제거 권장하지 않음
서비스 팩 불가 해당없음 시스템 복원 필요

도메인 환경에서의 추가 확인사항

  • 도메인 정책: Active Directory 그룹 정책 확인

안전한 업데이트 제거 절차

  1. 시스템 백업: 복구 지점 생성 및 데이터 백업
  2. 영향 분석: 제거할 업데이트의 영향 범위 확인
  3. 종속성 확인: 다른 업데이트와의 종속성 검토
  4. 권한 확인: 필요한 관리자 권한 확보
  5. 정책 검토: 조직 정책 및 보안 정책 확인
  6. 제거 실행: 단계별 제거 수행
  7. 시스템 검증: 제거 후 시스템 안정성 확인
  8. 모니터링: 제거 후 시스템 상태 지속 모니터링

제거 후 확인사항

  • 시스템 부팅 및 기본 기능 정상 작동
  • 보안 소프트웨어 정상 작동
  • 네트워크 연결 및 인터넷 접속
  • 주요 애플리케이션 정상 실행
  • 시스템 성능 및 안정성
