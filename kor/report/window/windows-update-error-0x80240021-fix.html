Windows 0x80240021 WU_E_TIME_OUT 오류 해결 가이드

⚠️ 주의사항: 이 오류는 Windows Update 과정에서 시간 초과가 발생했을 때 나타나는 문제입니다. 네트워크 연결 속도, 서버 응답 지연, 또는 시스템 리소스 부족이 원인일 수 있으므로 신중하게 해결 방법을 적용하시기 바랍니다.

🔍 오류 개요

0x80240021 (WU_E_TIME_OUT)는 Windows Update 작업이 지정된 시간 내에 완료되지 않아 시간 초과가 발생했을 때 나타나는 오류입니다. 이 오류는 주로 다음과 같한 원인으로 발생합니다:

  • 느린 인터넷 연결 속도
  • Windows Update 서버 응답 지연
  • 대용량 업데이트 다운로드 시간 초과
  • 네트워크 연결 불안정
  • 시스템 리소스 부족 (CPU, 메모리, 디스크)
  • 방화벽 또는 보안 소프트웨어에 의한 연결 지연
  • 프록시 서버 설정으로 인한 지연
  • DNS 해석 지연
  • 백그라운드에서 실행되는 다른 프로그램의 간섭
  • Windows Update 서비스 설정 문제

🛠️ 해결 방법

방법 1: Windows Update 문제 해결사 실행

1 Windows Update 문제 해결사 실행:
  • 설정 → 업데이트 및 보안 → 문제 해결
  • "Windows Update" 선택
  • "문제 해결사 실행" 클릭
  • 자동 진단 및 수정 과정 완료
2 네트워크 문제 해결사 실행:
  • 설정 → 네트워크 및 인터넷 → 상태
  • "네트워크 문제 해결사" 클릭
  • 또는 관리자 권한으로 명령 프롬프트에서 실행:
msdt.exe -id NetworkDiagnosticsWeb
msdt.exe -id WindowsUpdateDiagnostic
msdt.exe -id SystemMaintenanceDiagnostic

방법 2: 네트워크 연결 최적화

1 인터넷 연결 속도 확인:
  • 온라인 속도 테스트 사이트에서 연결 속도 측정
  • 다운로드 속도가 10Mbps 이상인지 확인
  • 관리자 권한으로 명령 프롬프트에서 연결 테스트:
ping google.com -t
ping windowsupdate.microsoft.com
tracert windowsupdate.microsoft.com
netstat -an | findstr :80
netstat -an | findstr :443
2 네트워크 설정 최적화:
  • 다음 명령어들을 순서대로 실행:
ipconfig /flushdns
ipconfig /registerdns
ipconfig /release
ipconfig /renew
netsh winsock reset
netsh int ip reset
netsh winhttp reset proxy
netsh int tcp set global autotuninglevel=normal
  • 명령어 실행 후 시스템 재부팅

방법 3: Windows Update 시간 초과 설정 조정

1 레지스트리를 통한 시간 초과 설정 증가:
  • Windows + R 키를 누르고 regedit 입력
  • 다음 경로로 이동:
  • HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update
  • 또는 명령어로 설정:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v DetectionFrequency /t REG_DWORD /d 22 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v DetectionFrequencyEnabled /t REG_DWORD /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v RebootRelaunchTimeout /t REG_DWORD /d 60 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\WindowsUpdate\Auto Update" /v RebootRelaunchTimeoutEnabled /t REG_DWORD /d 1 /f
2 BITS 서비스 시간 초과 설정:
  • 다음 명령어로 BITS 서비스 시간 초과 설정:
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\BITS" /v JobInactivityTimeout /t REG_DWORD /d 7200 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\BITS" /v MaxDownloadTime /t REG_DWORD /d 86400 /f
bitsadmin /SetMaxDownloadTime 86400

방법 4: 시스템 리소스 최적화

1 불필요한 프로그램 종료:
  • 작업 관리자 (Ctrl + Shift + Esc) 실행
  • CPU 및 메모리 사용량이 높은 프로그램 종료
  • 백그라운드 앱 비활성화:
  • 설정 → 개인 정보 → 백그라운드 앱
  • 불필요한 앱들 비활성화
2 시스템 성능 최적화:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 실행:
cleanmgr /sagerun:1
sfc /purgecache
dism /online /cleanup-image /startcomponentcleanup
defrag C: /O
powercfg /h off
3 가상 메모리 설정 최적화:
  • 시스템 속성 → 고급 → 성능 설정 → 고급 → 가상 메모리
  • 또는 명령어로 설정:
wmic computersystem where name="%computername%" set AutomaticManagedPagefile=False
wmic pagefileset where name="C:\\pagefile.sys" set InitialSize=4096,MaximumSize=8192

방법 5: Windows Update 서비스 재설정

1 모든 Windows Update 관련 서비스 중지:
  • 관리자 권한으로 명령 프롬프트 실행
  • 다음 명령어들을 순서대로 실행:
net stop wuauserv
net stop cryptSvc
net stop bits
net stop msiserver
net stop appidsvc
net stop dosvc
net stop UsoSvc
net stop WaaSMedicSvc
net stop TrustedInstaller
2 Windows Update 캐시 및 로그 삭제:
  • 다음 명령어들을 실행:
rd /s /q C:\Windows\SoftwareDistribution
rd /s /q C:\Windows\System32\catroot2
rd /s /q %ALLUSERSPROFILE%\Microsoft\Network\Downloader
del /f /s /q %windir%\Logs\WindowsUpdate\*
del /f /s /q %temp%\*
del /f /s /q %windir%\Temp\*
3 서비스 재시작 및 시간 초과 설정 적용:
  • 다음 명령어들을 실행:
net start wuauserv
net start cryptSvc
net start bits
net start msiserver
net start appidsvc
net start dosvc
net start UsoSvc
net start WaaSMedicSvc
wuauclt /resetauthorization /detectnow

방법 6: 방화벽 및 보안 소프트웨어 최적화

1 Windows Defender 방화벽 설정:
  • 제어판 → 시스템 및 보안 → Windows Defender 방화벽
  • "Windows Defender 방화벽을 통해 앱 또는 기능 허용" 클릭
  • 다음 항목들이 허용되어 있는지 확인:
  • - Windows Update
  • - Background Intelligent Transfer Service
  • - Automatic Updates
2 보안 소프트웨어 설정 최적화:
  • 타사 보안 소프트웨어의 실시간 보호 기능 임시 비활성화
  • Windows Update 관련 프로세스를 예외 목록에 추가:
  • - wuauclt.exe
  • - svchost.exe
  • - TrustedInstaller.exe
  • - UsoClient.exe
  • 업데이트 완료 후 보안 기능 재활성화

방법 7: DNS 서버 최적화

1 빠른 DNS 서버로 변경:
  • 제어판 → 네트워크 및 인터넷 → 네트워크 및 공유 센터
  • "어댑터 설정 변경" 클릭
  • 활성 네트워크 어댑터 우클릭 → 속성
  • "인터넷 프로토콜 버전 4 (TCP/IPv4)" 선택 → 속성
  • "다음 DNS 서버 주소 사용" 선택
2 권장 DNS 서버 설정:
  • 다음 중 하나를 선택하여 설정:
  • Google DNS: 8.8.8.8, 8.8.4.4
  • Cloudflare DNS: 1.1.1.1, 1.0.0.1
  • OpenDNS: 208.67.222.222, 208.67.220.220
  • 또는 명령어로 설정:
netsh interface ip set dns "로컬 영역 연결" static 8.8.8.8
netsh interface ip add dns "로컬 영역 연결" 8.8.4.4 index=2
ipconfig /flushdns

방법 8: PowerShell을 이용한 고급 복구

1 PowerShell Windows Update 모듈 설치:
  • PowerShell을 관리자 권한으로 실행
  • 다음 명령어들을 실행:
Set-ExecutionPolicy RemoteSigned -Force
Install-Module PSWindowsUpdate -Force
Import-Module PSWindowsUpdate
Add-WUServiceManager -ServiceID 7971f918-a847-4430-9279-4a52d1efe18d -Confirm:$false
2 시간 초과 설정 및 업데이트 실행:
  • 다음 명령어들을 실행:
Set-WUSettings -AcceptTrustedPublisherCerts
Get-WUServiceManager
Get-WindowsUpdate -Verbose
Install-WindowsUpdate -AcceptAll -IgnoreReboot -Verbose -Timeout 7200
Get-WindowsUpdateLog -LogPath C:\WindowsUpdate.log
Reset-WUComponents -Verbose

언제 전문가의 도움이 필요한가?

  • 모든 해결 방법을 시도했지만 문제가 지속되는 경우
  • 인터넷 연결 속도가 지속적으로 느린 경우
  • 시스템 하드웨어 성능이 부족한 경우
  • 기업 환경에서 복잡한 네트워크 설정이 있는 경우
  • ISP 레벨에서 연결 문제가 있는 경우
  • 시스템 파일 손상이 심각한 경우
  • 하드웨어 교체나 업그레이드가 필요한 경우

  • 시스템 성능 정기 점검
  • Windows Update 자동 설치 설정
  • 정기적인 시스템 재부팅
  • 보안 소프트웨어 설정 최적화

💡 추가 팁

  • 시간 초과 오류는 주로 네트워크나 시스템 성능 문제로 발생합니다
  • 대용량 업데이트는 밤 시간대에 설치하는 것이 좋습니다
  • 유선 연결이 무선 연결보다 안정적입니다
  • 시스템 리소스 사용량이 80% 이상일 때 업데이트를 피하세요
  • DNS 서버 변경만으로도 속도 개선 효과를 볼 수 있습니다
  • 이벤트 뷰어에서 상세한 오류 로그를 확인하세요
  • 정기적인 네트워크 속도 테스트를 수행하세요
  • 업데이트 실패 시 Windows Update 로그를 분석하세요

  • 0x8024001F: 연결 없음 (WU_E_NO_CONNECTION)
  • 0x80072EE2: 연결 시간 초과
  • 0x80072EFD: 서버에 연결할 수 없음

  • 0x8024402F: 업데이트 서버 연결 실패
  • 0x80240022: 업데이트 핸들러 없음 (WU_E_NO_UPDATE_HANDLER)

🛠️ 고급 해결 방법

네트워크 성능 분석 스크립트

  • PowerShell을 관리자 권한으로 실행
  • 네트워크 성능 및 시간 초과 분석:
$UpdateServers = @("windowsupdate.microsoft.com", "update.microsoft.com", "download.windowsupdate.com")
foreach ($Server in $UpdateServers) {
    Write-Host "테스트 중: $Server" -ForegroundColor Yellow
    $StartTime = Get-Date
    $Result = Test-NetConnection -ComputerName $Server -Port 80 -WarningAction SilentlyContinue
    $EndTime = Get-Date
    $ResponseTime = ($EndTime - $StartTime).TotalMilliseconds
    if ($Result.TcpTestSucceeded) {
        Write-Host "연결 성공: $Server (응답시간: $ResponseTime ms)" -ForegroundColor Green
    } else {
        Write-Host "연결 실패: $Server" -ForegroundColor Red
    }
}

시스템 리소스 모니터링 스크립트

  • 시스템 리소스 사용량 실시간 모니터링:
while ($true) {
    $CPU = Get-WmiObject -Class Win32_Processor | Measure-Object -Property LoadPercentage -Average | Select-Object -ExpandProperty Average
    $Memory = Get-WmiObject -Class Win32_OperatingSystem
    $MemoryUsed = [math]::Round((($Memory.TotalVisibleMemorySize - $Memory.FreePhysicalMemory) / $Memory.TotalVisibleMemorySize) * 100, 2)
    $Disk = Get-WmiObject -Class Win32_LogicalDisk -Filter "DeviceID='C:'"
    $DiskUsed = [math]::Round((($Disk.Size - $Disk.FreeSpace) / $Disk.Size) * 100, 2)
    Clear-Host
    Write-Host "=== 시스템 리소스 모니터링 ===" -ForegroundColor Green
    Write-Host "CPU 사용률: $CPU%" -ForegroundColor $(if($CPU -gt 80){"Red"}else{"Green"})
    Write-Host "메모리 사용률: $MemoryUsed%" -ForegroundColor $(if($MemoryUsed -gt 80){"Red"}else{"Green"})
    Write-Host "디스크 사용률: $DiskUsed%" -ForegroundColor $(if($DiskUsed -gt 90){"Red"}else{"Green"})
    Start-Sleep -Seconds 5
}

연결 속도별 권장 설정

  • 저속 연결 (10Mbps 미만): 시간 초과 값 5배 증가
  • 불안정한 연결: 최대 시간 초과 값 사용

🔧 네트워크 최적화 도구

Windows 내장 네트워크 도구

  • ping: 기본 연결 테스트 및 응답 시간 측정
  • tracert: 네트워크 경로 추적 및 지연 지점 확인
  • netstat: 네트워크 연결 상태 및 포트 사용 확인
  • nslookup: DNS 해석 속도 및 정확성 테스트
  • pathping: 경로별 패킷 손실 및 지연 분석

PowerShell 네트워크 명령어

  • Test-NetConnection: 고급 네트워크 연결 테스트
  • Get-NetAdapter: 네트워크 어댑터 상태 확인
  • Test-Connection: 향상된 ping 기능
  • Resolve-DnsName: DNS 해석 테스트

성능 모니터링 도구

  • 리소스 모니터 (resmon.exe)
  • 성능 모니터 (perfmon.exe)
  • 작업 관리자 성능 탭
  • 네트워크 및 공유 센터
