🔧 GX Works2란?

GX Works2는 미쓰비시 전기에서 개발한 통합 PLC 프로그래밍 소프트웨어입니다. 래더 다이어그램, 함수 블록 다이어그램, 구조화 텍스트 등 다양한 프로그래밍 언어를 지원하며, 산업 자동화 시스템 개발에 필수적인 도구입니다.

🎯 다양한 프로그래밍 언어
  • 래더 다이어그램 (LD)
  • 함수 블록 다이어그램 (FBD)
  • 구조화 텍스트 (ST)
  • 순차 기능 차트 (SFC)
⚡ 강력한 기능
  • 실시간 모니터링
  • 온라인 디버깅
  • 시뮬레이션 기능
  • 프로젝트 관리
🔧 호환성
  • MELSEC-Q 시리즈
  • MELSEC-L 시리즈
  • MELSEC-FX 시리즈
  • MELSEC iQ-R 시리즈

✨ GX Works2의 주요 특징

  • 🎨 직관적인 인터페이스: 사용자 친화적인 GUI로 쉬운 프로그래밍
  • 🔄 통합 개발 환경: 프로그래밍부터 디버깅까지 한 번에
  • 📊 실시간 모니터링: 프로그램 실행 상태를 실시간으로 확인
  • 🛡️ 안전 기능: 안전 관련 프로그래밍 지원

📥 설치 가이드

설치 파일 다운로드

미쓰비시 전기 공식 웹사이트에서 GX Works2 설치 파일을 다운로드합니다. 라이선스 키가 필요하므로 정품 구매 후 진행하세요.

관리자 권한으로 실행

다운로드한 설치 파일을 마우스 우클릭하여 "관리자 권한으로 실행"을 선택합니다.

설치 마법사 진행

설치 마법사의 안내에 따라 진행합니다. 설치 경로는 기본값을 사용하는 것을 권장합니다.

라이선스 등록

설치 완료 후 라이선스 키를 입력하여 제품을 활성화합니다.

USB 드라이버 설치

PLC와 연결하기 위한 USB 드라이버를 별도로 설치합니다.

💡 설치 팁

  • 설치 전 바이러스 백신 프로그램을 일시적으로 비활성화하세요
  • Windows 방화벽 설정을 확인하세요
  • 설치 후 재부팅을 권장합니다
  • 설치 로그를 확인하여 오류가 없는지 점검하세요

🖥️ 인터페이스 이해

메인 화면 구성

🎛️ 주요 구성 요소

  • 메뉴바: 파일, 편집, 보기, 프로젝트 등 주요 기능
  • 툴바: 자주 사용하는 기능들의 빠른 접근
  • 프로젝트 트리: 프로젝트 구조 및 파일 관리
  • 편집 영역: 프로그램 작성 및 편집
  • 출력 창: 컴파일 결과 및 오류 메시지
  • 상태바: 현재 상태 정보 표시

프로젝트 트리 구조

📁 프로젝트 구성

  • Parameter: PLC 파라미터 설정
  • Program: 메인 프로그램 및 서브루틴
  • Label: 변수 및 레이블 정의
  • Device Comment: 디바이스 주석
  • Library: 사용자 라이브러리

편집기 종류

📊 래더 편집기

가장 일반적인 PLC 프로그래밍 방식으로, 전기 회로도와 유사한 형태로 프로그램을 작성합니다.

🔧 함수 블록 편집기

함수 블록을 연결하여 프로그램을 작성하는 방식으로, 복잡한 로직을 모듈화할 수 있습니다.

📝 구조화 텍스트 편집기

고급 프로그래밍 언어와 유사한 텍스트 기반 프로그래밍 방식입니다.

🚀 첫 번째 프로젝트 만들기

새 프로젝트 생성

File → New Project를 선택하여 새 프로젝트를 생성합니다.

PLC 타입 선택

사용할 PLC 시리즈와 모델을 선택합니다. (예: MELSEC-Q, MELSEC-L 등)

프로젝트 이름 설정

프로젝트 이름과 저장 위치를 지정합니다.

프로젝트 설정

프로젝트 타입(Simple Project 또는 Structured Project)을 선택합니다.

💡 프로젝트 타입 선택 가이드

  • Simple Project: 초보자에게 적합, 간단한 구조
  • Structured Project: 대규모 프로젝트에 적합, 모듈화 가능

기본 설정

⚠️ 필수 설정 항목

  • CPU 타입: 사용할 CPU 모델 정확히 선택
  • 메모리 용량: 프로그램 메모리 크기 설정
  • I/O 구성: 입출력 모듈 구성 설정
  • 네트워크 설정: 통신 파라미터 설정

📝 기본 프로그래밍

래더 프로그래밍 기초

🔧 기본 명령어

  • LD (Load): 접점을 읽어오는 명령
  • OUT: 코일을 출력하는 명령
  • AND: 논리곱 연산
  • OR: 논리합 연산
  • NOT: 논리부정 연산

간단한 예제

LED 점멸 프로그램

Network 1: LED 점멸 회로
LD    X0      // 입력 X0 읽기
OUT   Y0      // 출력 Y0에 출력

Network 2: 타이머를 이용한 점멸
LD    M0      // 내부 릴레이 M0
OUT   T0 K10  // 타이머 T0, 1초 설정
LD    T0      // 타이머 T0 접점
OUT   M0      // 내부 릴레이 M0 출력
LD    M0      // 내부 릴레이 M0
OUT   Y1      // 출력 Y1에 출력

  2. I/O 할당 계획
  3. 프로그램 작성
  4. 시뮬레이션 테스트
  5. 실제 하드웨어 테스트

변수 및 레이블 사용

💡 변수 명명 규칙

  • 의미 있는 이름 사용 (예: Motor_Start, Sensor_Input)
  • 일관된 명명 규칙 적용
  • 예약어 사용 금지
  • 주석을 활용한 설명 추가

🔗 PLC 연결 및 다운로드

통신 설정

Online → Connection Setting에서 통신 방식을 설정합니다. (USB, Ethernet, Serial 등)

PLC 연결 확인

Online → PLC Read를 실행하여 PLC와의 연결을 확인합니다.

프로그램 컴파일

Build → Build All을 실행하여 프로그램을 컴파일합니다.

프로그램 다운로드

Online → PLC Write를 실행하여 컴파일된 프로그램을 PLC에 다운로드합니다.

⚠️ 연결 시 주의사항

  • PLC 전원이 켜져 있는지 확인
  • 통신 케이블이 올바르게 연결되었는지 확인
  • PLC 모드가 STOP 상태인지 확인
  • 기존 프로그램 백업 후 진행

모니터링 및 디버깅

🔍 모니터링 기능

  • Device Monitor: 개별 디바이스 값 실시간 확인
  • Ladder Monitor: 래더 프로그램 실행 상태 확인
  • Entry Data Monitor: 데이터 테이블 형태로 모니터링
  • Sampling Trace: 시간에 따른 데이터 변화 추적
